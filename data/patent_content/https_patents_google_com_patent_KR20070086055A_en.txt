본 발명은 일반적으로 컴퓨터 네트워크(예를 들어, 컴퓨터 시스템들의 분산된 시스템)에서 문서들의 위치를 찾아내기 위한 검색 엔진 분야에 관한 것이며, 특히, 비-표음 기호들을 포함하는 언어들에서의 사용자의 요청을 예측함으로써 원하는 검색에 속도를 빠르게 하기 위한 시스템 및 방법에 관한 것이다.FIELD OF THE INVENTION The present invention generally relates to the field of search engines for locating documents in a computer network (e.g., a distributed system of computer systems), and in particular, a request from a user in languages that include non-phonetic symbols The present invention relates to a system and method for speeding up a desired search by predicting.
검색 엔진들은 월드 와이드 웹(WWW)상의 문서들 또는 인트라넷의 컴퓨터들상에 저장된 문서들과 같은, 큰 문서 데이터베이스에서 문서들의 위치를 찾아내기 위한 강력한 툴을 제공한다. 문서들은 사용자에 의해 제시된 검색 쿼리(search query)에 응답하여 위치가 발견된다. 검색 쿼리는 하나 이상의 검색어들로 구성될 수 있다.Search engines provide a powerful tool for locating documents in a large document database, such as documents on the World Wide Web (WWW) or documents stored on computers on an intranet. Documents are located in response to a search query presented by the user. The search query may consist of one or more search terms.
쿼리를 입력하는 하나의 접근법에 있어서, 사용자는 모든 검색어들이 입력될 때까지 연속되는 검색어들의 부가에 의하여 쿼리를 입력한다. 일단 모든 쿼리의 검색어들이 입력되었다는 신호를 보내면, 쿼리는 검색 엔진으로 전송된다. 사용자는 예를 들어, 리턴 문자를 입력함으로써, 키보드상의 엔터 키를 누름으로써, 또는 그래픽 사용자 인터페이스상에 "검색" 버튼을 클릭함으로써 쿼리의 완성을 신호로 보내는 대안적인 방법들을 가질 수 있다. 일단 검색 엔진에 의해 쿼리가 수신되면, 그것은 검색 쿼리를 처리하고, 검색 쿼리에 응답하는 문서들을 검색하며, 사용자에게 문서들의 리스트를 리턴한다.In one approach to entering a query, the user enters the query by adding successive search terms until all search terms have been entered. Once you signal that all of the queries in the query have been entered, the query is sent to the search engine. The user may have alternative ways to signal the completion of the query, for example, by entering a return character, by pressing the Enter key on the keyboard, or by clicking on the “search” button on the graphical user interface. Once the query is received by the search engine, it processes the search query, retrieves the documents that respond to the search query, and returns a list of documents to the user.
알파벳 기록 시스템에 주로 기초하지 않는 언어에 있어서, 종종 일련의 문자들은 쿼리의 언어 컴포넌트들을 생성하기 위하여 키보드로부터 입력된다. 이러한 방식으로 쿼리를 입력하는 것은 시간 소모적일 수 있다.In languages that are not primarily based on alphabetic writing systems, a series of characters is often input from the keyboard to create language components of a query. Entering a query in this manner can be time consuming.
쿼리들은 일반적으로 사용자가 쿼리가 완전하다는 신호를 보낼 때까지 검색 엔진으로 전송되지 않기 때문에, 사용자가 전체 검색 쿼리를 완성하는 동안 시간이 지나간다. 이러한 공정의 속도를 높이는 시스템 및 방법이 필요할 것이다.Queries are typically not sent to the search engines until the user signals that the query is complete, so time passes while the user completes the entire search query. There will be a need for systems and methods to speed up this process.
일 실시예에서, 표의 문자 및 표음 문자를 갖는 언어에 대한 쿼리 완성을 제안하기 위한 방법은 검색 요청자로부터 부분적 쿼리를 수신하는 단계를 포함한다. 부분적 쿼리는 완전한 쿼리의 일부분이다. 예측된 쿼리의 세트는 순위 조건에 따라 정렬된 하나 이상의 표의 문자를 갖는 적어도 하나의 스트링을 포함하는 것으로 예측된다. 상기 정렬된 예측 쿼리들의 세트는 검색 요청자로 운반된다.In one embodiment, a method for suggesting query completion for a language having ideographic and phonetic characters includes receiving a partial query from a search requester. Partial queries are part of a complete query. The set of predicted queries is expected to include at least one string with one or more ideographic characters sorted according to ranking criteria. The set of sorted prediction queries is carried to the search requester.
검색 요청자는 정렬된 예측 쿼리 세트로부터 개별적인 쿼리를 선택할 수 있으며, 그 후, 쿼리의 완성을 지시한다. 검색 엔진은 검색 결과들의 세트를 제작하도록 쿼리를 처리한다. 대안적으로, 검색 요청자는 완성 쿼리가 입력될 때까지, 또는 새로운 예측된 쿼리들의 세트가 검색 엔진으로 전송되어 나타날 때까지 계속해서 쿼리 정보를 입력할 수 있다.The search requester can select an individual query from a sorted set of predictive queries, which then indicates the completion of the query. The search engine processes the query to produce a set of search results. Alternatively, the search requester may continue to enter query information until a complete query is entered, or until a new set of predicted queries is sent to the search engine for display.

본 발명의 일 실시예에서, 사용자 쿼리의 부분들은 완전한 쿼리 입력을 완료하기 전에 검색 엔진으로 전송된다. 검색 엔진은 사용자의 최종 쿼리를 예측하기 위하여 쿼리의 전송된 부분을 사용한다. 이러한 예측은 사용자에게 다시 전송된 다. 예측값들 중 하나가 사용자의 의도된 쿼리라면, 그 후, 사용자는 쿼리의 완전한 엔트리를 갖지 않는 예측된 쿼리를 선택할 수 있다. 몇몇 실시예들에서, 선택된 쿼리는 검색 엔진으로 전송되어, 선택된 쿼리에 대응하는 쿼리 결과값 세트를 리턴한다.In one embodiment of the invention, portions of the user query are sent to the search engine before completing the complete query entry. The search engine uses the transmitted portion of the query to predict the user's final query. This prediction is sent back to the user. If one of the predictions is the user's intended query, then the user can select a predicted query that does not have a complete entry of the query. In some embodiments, the selected query is sent to a search engine to return a query result set corresponding to the selected query.

URL들은 또한 문서들, 폴더들 또는 서비스들과 같은 컴퓨터상에 국부적으로 이용가능한 리소스들을 식별하기 위하여 사용될 수 있다. 본 명세서에서 용어 "URL"은 인터넷 어드레스, RFC 1738 컴플리언트 어드레스(compliant address) 및 다수의 컴퓨터 시스템들 및 지역 정보 통신망에서 사용되는 것들과 같은 파일 경로이름을 포함하나 이에 제한되지 않는, 임의의 형태의 컨텐츠 위치 식별자를 의미하도록 사용된다. 검색 엔진(106)은 처리 단계(112)를 위한 부분적 쿼리를 수신하고, 사용자 고안 완성 쿼리(또는 URL)(114)로서 예측값들을 만들어낸 다. 예측값들은 순위 기준에 따라 정렬된다. 예를 들어, 몇몇 실시예들에서, 높은 서브미션 주파수를 갖는 쿼리들은 낮은 서브미션 주파수를 갖는 쿼리들 이전에 정렬된다. 검색 엔진(106)은 정렬된 예측값들을 만드는 것을 보조하기 위하여 다수의 쿼리 완성 테이블들(아래에서 보다 상세히 개시되는)을 사용한다. 쿼리 완성 테이블들은 검색 엔진(106)에 의해 수신된 d전에 입력된 검색 쿼리들을 사용하여 생성된다. 몇몇 실시예들에서, 이전 쿼리들은 사용자의 커뮤니티로부터의 검색 쿼리들을 포함한다. 예측된 쿼리들은 클라이언트 시스템(106)으로 다시 보내지(116)고, 그 후, 사용자에게 보여진다(118). 예측된 쿼리들 중 하나가 사용자가 원하는 쿼리로서 의도된 것이라면, 사용자는 원하는 쿼리의 입력을 종료할 필요 없이 이러한 완성된 쿼리를 선택하고 처리할 수 있다. 예측된 쿼리들이 사용자가 염두하고 있는 것을 반영하지 않는다면, 그 후, 사용자는 원하는 검색 쿼리를 입력하는 것을 계속할 수 있다.1 illustrates an exemplary embodiment of the present invention that includes a

사용자가 최종 입력 신호를 보내기에 앞서, 부분적 입력이 식별될 수 있다(302-부분적 입력). 부분적 입력은 다수의 방법들로 식별될 수 있다. 검색 쿼리를 위하여, 부분적 입력은 검색 쿼리의 단일 검색어, 다수의 검색어들 또는, 검색어의 다수의 한정된 문자들을 포함한다.Prior to the user sending the final input signal, the partial input may be identified (302-partial input). Partial input can be identified in a number of ways. For a search query, the partial input includes a single search term, multiple search terms, or multiple qualified characters of the search query.
몇몇 실시예들에서, 부분적 입력은 구획 문자 또는 다른 문자의 엔트리를 감지함으로써 식별된다(예를 들어, 제한(limitation), 인용 문자, 마침표(period), 삽입 문자(parenthesis character), 사선 문자(slash character), 화살표 키 감지 또는 탭 엔트리 없이). 구획 문자의 엔트리는 사용자가 원하는 용어 또는 입력의 부분을 입력하는 것을 끝마쳤으며, 다음 검색어 또는 부분으로 이동하였음을 나타낼 수 있다.In some embodiments, partial input is identified by detecting an entry of a delimiter or other character (eg, a limit, quote character, period, parenthesis character, slash) character), without arrow key detection or tab entry). The entry of the delimiter character may indicate that the user has finished entering the desired term or portion of the input and has moved to the next search term or portion.
몇몇 실시예들에서, 부분적 입력은 미리 결정된 개수의 문자들의 엔트리를 감지함으로써 식별된다. 이러한 실시예들에서, 입력은 전체 입력보다 다수의 문자들을 포함하나, 여전히 사용자가 모든 문자들을 입력하기 전에 부분적 입력을 식별하는 것이 바람직할 수 있다. 이러한 기술은 예를 들어, 검색어 또는 URL이 많은 개수의 문자들을 포함할 때, 또는 미리 결정된 개수의 문자들이 유용한 예측값들을 초래하기에 충분히 클 때 바람직하다.In some embodiments, the partial input is identified by detecting an entry of a predetermined number of characters. In such embodiments, the input includes more characters than the full input, but it may still be desirable to identify the partial input before the user enters all the characters. Such a technique is desirable, for example, when a search term or URL contains a large number of characters, or when the predetermined number of characters is large enough to yield useful predictions.

Query completion tables for partial queries of length less than C remain unchanged. For partial queries of length at least C, the partial query is divided into two parts: the prefix part and the suffix part. The length of the suffix (S) is equal to the length of the partial query (L) modulo C:
S = L modulo CS = L modulo C
여기서, L은 부분적 쿼리의 길이이다. 접두사부(P)의 길이는 부분적 쿼리 길이 빼기 접미사 길이이다: P = L - S. 따라서, 예를 들어, 10개 문자의 길이를 갖는 부분적 쿼리는 청크 크기(C)가 4일 때 2의 접미사 길이(S) 및 8의 접두사 길 이(P)를 가질 것이다.Where L is the length of the partial query.
