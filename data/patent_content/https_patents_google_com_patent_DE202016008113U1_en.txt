HINTERGRUNDBACKGROUND
Bei digitalen Kommunikationssystemen wie z. B. jenen, die auf unbemannten Luftfahrzeugen (UAV) basieren, kann das System größere Leistungsfähigkeit haben, wenn weniger Strom von der digitalen Nutzlast benötigt wird. Außerdem kann ein größerer Abdeckungsbereich bereitgestellt werden, wenn weniger Strom benötigt wird. Einer der größten Stromverbraucher im UAV ist eine Empfängerfunktion von digitalen Modems. Ein Vorwärtsfehlerkorrektur-(FEC)-Decodierungsfunktionsblock verbraucht einen großen Strombetrag. Der FEC-Decodierungsfunktionsblock hält jedoch die Gesamtbitfehlerrate (BER) und die Verbindungsleistung aufrecht.In digital communication systems such. For example, based on unmanned aerial vehicles (UAV), the system may have greater performance if less power is needed from the digital payload. In addition, a larger coverage area can be provided when less power is needed. One of the largest consumers of power in the UAV is a receiver function of digital modems. A forward error correction (FEC) decoding function block consumes a large amount of current. However, the FEC decoding function block maintains the total bit error rate (BER) and link performance.
Ein UAV unterstützt Benutzerverbindungen, Gateway-Verbindungen und Zwischen-UAV-Verbindungen. Der FEC-Decodierer ist ein großer Stromverbraucher für alle drahtlosen Verbindungen. Jeder Funkverbindung ist ein FEC-Decodierer zugeordnet. Es gibt eine große Zahl von Benutzerverbindungen, da es für jeden Benutzer eine Benutzerverbindung gibt. Es kann hunderte aktive Benutzerverbindungen geben. Typischerweise gibt es eine kleine Zahl von Gateway- und Zwischen-UAV-Verbindungen in der Größenordnung von 1 bis 10.A UAV supports user connections, gateway connections, and intermediate UAV connections. The FEC decoder is a big power consumer for all wireless connections. Each radio link is assigned an FEC decoder. There are a large number of user connections because there is a user connection for each user. There can be hundreds of active user connections. Typically, there are a small number of gateway and intermediate UAV links on the order of 1 to 10.

DETAILLIERTE BESCHREIBUNGDETAILED DESCRIPTION
In Übereinstimmung mit dieser Offenbarung wird die FEC-Decodiererfunktion auf allen Benutzerverbindungen, Gateway-Verbindungen und Zwischen-UAV-Verbindungen dynamisch von dem UAV weggenommen. Dementsprechend wird der Stromverbrauch im UAV wesentlich vermindert. Das dynamische Wegnehmen der FEC-Decodiererfunktion basiert auf der momentanen Verbindungsgüte, wie z. B. Bitfehlerrate (BER) oder Rahmenfehlerrate (FER), und anderen Parametern, wie z. B. verfügbarem Strom und geschätztem zukünftigen Stromverbrauch. Wenn die drahtlosen Verbindungen in dem UAV-basierten System sehr hohe Güte haben, können FEC-Decodierer mit wenig bis keiner Auswirkung auf die Gesamt-Bitfehlerrate vom UAV weggenommen werden. In manchen Beispielen wird, abhängig von den Verbindungsdetails, die FEC-Decodiererfunktion während hoher Signalgüte-Bedingungen vom UAV zu einem Gateway-Endgerät verlagert. Wenn die Signalgüte niedrig ist, werden, um die höchstmögliche Funkverbindungsgüte und niedrigstmögliche BER zu erreichen, die FEC-Decodierer nicht umgangen, so dass der höchste Schutz bereitgestellt wird. Es wird erwartet, dass diese Ereignisse mit niedriger Signalgüte von kurzer Dauer sein werden, so dass der höhere Strom, der für diese Betriebsart erforderlich ist, nur für eine kurze Zeitspanne fließen wird. Es wird ein Hintergrund-Stromzuweisungsalgorithmus verwendet, um genau zu bestimmen, wann in welcher Betriebsart zu arbeiten ist. Dieser Hintergrundalgorithmus stellt die Kompromisse zwischen akzeptablen BER-Raten, dem Stromverbrauch und der voraussichtlichen Nutzung für die nächsten paar Stunden und somit dem voraussichtlichen Strombedarf her.In accordance with this disclosure, the FEC decoder function is dynamically removed from the UAV on all user connections, gateway connections and inter-UAV connections. Accordingly, the power consumption in the UAV is significantly reduced. The dynamic removal of the FEC decoder function is based on the current connection quality, such. B. bit error rate (BER) or frame error rate (FER), and other parameters such. Available power and estimated future power consumption. If the wireless connections in the UAV based system are very high quality, FEC decoders with little to no impact on the overall bit error rate can be taken away from the UAV. In some examples, depending on the connection details, the FEC decoder function is relocated from the UAV to a gateway terminal during high-signal-quality conditions. When the signal quality is low, in order to achieve the highest possible radio link quality and lowest possible BER, the FEC decoders are not bypassed, thus providing the highest protection. It is expected that these low-signal-quality events will be short-lived, so that the higher current required for this mode of operation will only flow for a short period of time. A background current allocation algorithm is used to determine exactly when to operate in which mode. This background algorithm makes the trade-offs between acceptable BER rates, power consumption, and expected usage for the next few hours, and thus the anticipated power demand.
