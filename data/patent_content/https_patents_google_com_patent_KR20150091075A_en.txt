Abstract
본 개시내용은 연락처를 대응되는 시스템 동작과 연계시키는 것과 관련된다. 이는, 주소록, 사용자 입력, 온라인 계정, 또는 연락처 프로파일에 대응되는 연락처와의 상호작용 중 하나 이상에 기초하여, 시스템 동작과의 구성을 위해 연락처 프로파일을 식별하는 것, 식별된 연락처 프로파일에 대한 연락처 정보를 검색하는 것, 그리고 식별된 연락처 프로파일에 대한 검색된 연락처 정보에 기초하여 적어도 하나의 가용 시스템 동작을 결정하는 것을 포함한다. 본 개시내용은 또한, 검색된 연락처 정보에 기초하여, 운영 체제의 그래픽 사용자 인터페이스 컴포넌트에서 연락처 프로파일을 표시하기 위해 연락처 프로파일을 나타내는 시스템 아이콘을 생성하는 것, 그리고 연락처 프로파일을 나타내는 시스템 아이콘에서 선택되는 경우 적어도 하나의 가용 시스템 동작이 자동으로 기동(launch)되도록 구성함으로써 연락처 프로파일과 적어도 하나의 가용 시스템 동작을 연계시키는 것을 포함한다.The present disclosure relates to associating a contact with a corresponding system operation. This may include identifying a contact profile for configuration with the system action, based on one or more of address book, user input, online account, or interaction with a contact corresponding to the contact profile, contact information for the identified contact profile And determining at least one available system operation based on the retrieved contact information for the identified contact profile. The disclosure also provides for creating a system icon representing a contact profile to display a contact profile in a graphical user interface component of an operating system based on the retrieved contact information and selecting at least And associating the contact profile with at least one available system operation by configuring one of the available system operations to be automatically launched.
Description
본 개시내용은 일반적으로 애플리케이션 관리에 관한 것이며, 보다 구체적으로는 연락처를 대응되는 시스템 동작에 연계시키는 것과 관련된다.This disclosure relates generally to application management and, more particularly, to associating contacts with corresponding system operations.
현재 운영 체제의 대부분의 기능은 애플리케이션 중심 접근방식으로 만들어진다. 즉, 연락처에 이메일을 전송하기 위해, 사용자는 이메일 애플리케이션을 기동(launch)하고, 연락처에 인스턴트 메시지를 전송하기 위해, 사용자는 또 다른 애플리케이션을 기동하는 등이다. 일단 적절한 애플리케이션이 기동되면, 사용자는 통신하거나 상호작용(예컨대, 문서를 공유)하고 싶은 하나 이상의 연락처를 선택한다. 나아가, 사용자가 통신을 수신하는 경우, 사용자는 착신되는 통신의 유형에 대응되는 각각의 애플리케이션을 체크해야 한다. 통지가 제공되는 경우에도, 사용자는 이러한 통신을 어느 연락처에서 전송했는지를 파악하기 어려울 수도 있다. 그러므로, 통지만으로는 중요한 통신을 식별하기 어렵고, 사용자가 올바른 애플리케이션을 열어 체크하지 않는다면 특정 연락처로부터의 중요한 통신을 놓칠 수 있다.Most of the current operating system features are built around an application-centric approach. That is, to send an email to a contact, the user launches an email application, sends an instant message to the contact, the user activates another application, and so on. Once the appropriate application is launched, the user selects one or more contacts that they want to communicate or interact with (e.g., share documents with). Further, when a user receives a communication, the user must check each application corresponding to the type of communication being received. Even if a notification is provided, it may be difficult for the user to know from which contact this communication was sent. Therefore, notification alone is difficult to identify important communications, and important communications from specific contacts may be missed unless the user opens and checks the correct application.
본 개시내용은 연락처를 대응되는 시스템 동작과 연계시키기 위한 기계 구현 방법에 관한 것이다. 이러한 방법은, 주소록, 사용자 입력, 온라인 계정, 또는 연락처 프로파일에 대응되는 연락처와의 상호작용 중 하나 이상에 기초하여, 시스템 동작과의 구성을 위해 연락처 프로파일을 식별하는 단계, 상기 식별된 연락처 프로파일에 대한 연락처 정보를 검색하는 단계, 상기 식별된 연락처 프로파일에 대한 상기 검색된 연락처 정보에 기초하여 적어도 하나의 가용 시스템 동작을 결정하는 단계를 포함하며, 각각의 가용 시스템 동작은 상기 식별된 연락처 프로파일에 대한 상기 검색된 연락처 정보의 적어도 일부분에 대응된다. 방법은 또한, 상기 검색된 연락처 정보에 기초하여, 운영 체제의 그래픽 사용자 인터페이스 컴포넌트에서 상기 연락처 프로파일을 표시하기 위해 상기 연락처 프로파일을 나타내는 시스템 아이콘을 생성하는 단계, 및 상기 연락처 프로파일을 나타내는 상기 시스템 아이콘에서 선택되는 경우 상기 적어도 하나의 가용 시스템 동작이 자동으로 기동(launch)되도록 구성함으로써 상기 연락처 프로파일과 상기 적어도 하나의 가용 시스템 동작을 연계시키는 단계를 포함한다.The present disclosure relates to a machine implementation method for associating a contact with a corresponding system operation. Alternatively, a contact that is physically very close to the user may be considered a priority contact.
