HINTERGRUNDBACKGROUND
Drahtlose Kommunikation ist zu einem führenden Medium für den Zugang zu und das Empfangen von Daten geworden. In vielen Fällen kommuniziert ein Benutzergerät über Mobilfunkdatennetzwerke (z. B. GSM, CDMA, LTE, 3G, 4G, 5G usw.), die innerhalb eines administrativ lizenzierten Bandes betrieben werden, sodass nur lizenzierte Mobilfunknetzbetreiber Datennetzwerke (jedes ein „primäres Mobilfunkdatennetz“) innerhalb des lizenzierten Bandes einrichten dürfen. Den lizenzierten Betreibern werden entsprechende Spektren innerhalb des lizenzierten Bandes zugewiesen, was verhindert, dass nicht lizenzierte Betreiber das lizenzierte Band verwenden.Wireless communication has become a leading medium for accessing and receiving data. In many cases, a user device communicates over cellular data networks (e.g., GSM, CDMA, LTE, 3G, 4G, 5G, etc.) that operate within an administratively licensed band so that only licensed cellular network operators are data networks (each a "primary cellular data network") within the licensed band. The licensed operators are assigned corresponding spectra within the licensed band, which prevents unlicensed operators from using the licensed band.
Um sich mit einem primären Mobilfunkdatennetzwerk zu verbinden, verwendet das Benutzergerät ein Abonnementprofil, das von einem assoziierten Netzwerkbetreiber verwaltet wird. Das Abonnementprofil beinhaltet im Allgemeinen eine internationale Mobilfunkteilnehmerkennung (IMSI) und den zugehörigen Schlüssel. Das Abonnementprofil wird auf dem Benutzergerät gespeichert und stellt dem Benutzergerät Netzwerkzugangsinformationen für den Zugang zu dem assoziierten primären Mobilfunkdatennetzwerk bereit. Ein einzelnes Abonnementprofil kann es dem Benutzergerät ermöglichen, sich mit mehreren assoziierten primären Mobilfunkdatennetzwerkknoten zu verbinden, sodass das Benutzergerät mit dem assoziierten primären Mobilfunkdatennetzwerk innerhalb eines weiten geografischen Bereichs kommunizieren kann.To connect to a primary cellular data network, the user device uses a subscription profile managed by an associated network operator. The subscription profile generally includes an international mobile subscriber identity (IMSI) and the associated key. The subscription profile is stored on the user device and provides the user device with network access information for access to the associated primary cellular data network. A single subscription profile can enable the user device to connect to multiple associated primary cellular data network nodes so that the user device can communicate with the associated primary cellular data network within a wide geographic area.
Citizens Broadband Radio Service (CBRS) ist ein nicht lizenziertes Funkfrequenzband im 3,5-GHz-Spektrum, das für die gemeinsame zivile Nutzung der drahtlosen Datenkommunikation zugelassen wurde. CBRS-Bandnetzwerke ähneln den primären Mobilfunkdatennetzwerken, jedoch ist für die Nutzung des CBRS-Bandes keine kostspielige, exklusiv lizenzierte Frequenzlizenz von einer Verwaltungsbehörde erforderlich. Daher können viele verschiedene Einrichtungen (z. B. Krankenhäuser, Bildungseinrichtungen, Cafes, Stadien, Einkaufszentren) in der Lage sein, Datenzugriff für Benutzergeräte über ein CBRS-Netzwerk bereitzustellen. CBRS bietet auch viele Vorteile gegenüber dem Datenzugriff über ein drahtloses lokales Netzwerk (WLAN) (z. B. Wi-Fi), da die WLAN-Frequenzbänder in der Regel sehr überfüllt sind, was zu schlechten Verbindungen und langsamen Datenübertragungsgeschwindigkeiten führen kann.Citizens Broadband Radio Service (CBRS) is an unlicensed radio frequency band in the 3.5 GHz spectrum that has been approved for the joint civil use of wireless data communication. CBRS band networks are similar to primary cellular data networks, except that the use of the CBRS band does not require an expensive, exclusively licensed frequency license from a government agency. Therefore, a wide variety of facilities (e.g., hospitals, educational institutions, cafes, stadiums, shopping malls) may be able to provide data access for user devices over a CBRS network. CBRS also offers many advantages over accessing data over a wireless local area network (WLAN) (e.g. Wi-Fi), as the WLAN frequency bands are typically very crowded, which can result in poor connections and slow data transfer speeds.
Ein Problem bei der Nutzung des CBRS-Bandes besteht darin, dass für den Zugang zu einem bestimmten CBRS-Netzwerk ein separates CBRS-Profil oder -Zertifikat (zusammen „CBRS-Profil“) erforderlich ist, (im Gegensatz zu primären Mobilfunkdatennetzwerken, in denen ein einzelnes Profil verwendet werden kann). Infolgedessen müssen für den Zugang zu den verschiedenen CBRS-Netzwerken, für die das Benutzergerät in den Verbindungsbereich eintreten kann, mehrere CBRS-Profile auf dem Benutzergerät oder einem Remote-Dienst gespeichert werden. Um beispielsweise auf fünf verschiedene CBRS-Netzwerke zuzugreifen, können fünf verschiedene CBRS-Profile erforderlich sein.One problem with using the CBRS band is that a separate CBRS profile or certificate (collectively, "CBRS Profile") is required to access a particular CBRS network (as opposed to primary cellular data networks in which a single profile can be used). As a result, multiple CBRS profiles must be stored on the user device or on a remote service in order to access the various CBRS networks that the user equipment can enter into the connection area. For example, to access five different CBRS networks, five different CBRS profiles may be required.
Dies führt zu einem Problem bei der Auswahl des richtigen CBRS-Profils in einem Verbindungsbereich. Der zur Zeit verfügbare Ansatz besteht darin, dass das Benutzergerät das verfügbare CBRS-Frequenzband unter Nutzung eines Modems des Benutzergeräts scannt, ermittelt, ob ein CBRS-Netzwerk verfügbar ist, ermittelt, ob das Benutzergerät Zugang zu dem CBRS-Netzwerk hat und ob mehrere CBRS-Netzwerke verfügbar sind, zu denen das Benutzergerät Zugang hat -- welches CBRS-Profil das richtige zu verwendende Profil ist. Dieser Vorgang kann in der Größenordnung von Sekunden und manchmal Minuten dauern. Zur Optimierung der CBRS-Erfassung bei gleichzeitiger Minimierung negativer Auswirkungen auf ein primäres Mobilfunkdatennetzwerk und Minimierung des Gesamtenergieverbrauchs der Benutzergeräte ist es erforderlich, sich an ein CBRS-Netzwerk anschließen zu können, sobald das Benutzergerät in den Verbindungsbereich eintritt.This creates a problem in choosing the correct CBRS profile in a connection area. The currently available approach is for the user equipment to scan the available CBRS frequency band using a modem of the user equipment, determine whether a CBRS network is available, determine whether the user equipment has access to the CBRS network and whether multiple CBRS - Networks are available to which the user device has access - which CBRS profile is the correct profile to use. This process can take on the order of seconds and sometimes minutes. To optimize CBRS coverage while minimizing the negative impact on a primary cellular data network and minimizing the overall power consumption of user devices, it is necessary to be able to connect to a CBRS network as soon as the user device enters the link area.
Eine weitere Schwierigkeit bei der Nutzung des CBRS-Bandes besteht darin, dass es dasselbe Protokoll-Modem wie das primäre Mobilfunkdatennetzwerk verwendet. Somit kann das Protokoll-Modem eine Verbindung zu einem CBRS-Netzwerk oder einem primären Mobilfunkdatennetzwerk herstellen, im Allgemeinen jedoch nicht gleichzeitig mit mehreren Netzwerken. Dementsprechend kann die Verbesserung des Vertrauens, dass eine CBRS-Datenverbindung verfügbar und zuverlässig ist, bevor von einer primären Mobilfunkdatennetzwerkverbindung oder von einer anderen CBRS-Netzwerkverbindung gewechselt wird, schlechte oder verlorene Datenverbindungen verhindern.Another difficulty with using the CBRS band is that it uses the same protocol modem as the primary cellular data network. Thus, the protocol modem can establish a connection to a CBRS network or a primary cellular data network, but in general not to several networks at the same time. Accordingly, improving confidence that a CBRS data connection is available and reliable before switching from a primary cellular data network connection or from another CBRS network connection can prevent poor or lost data connections.
Darüber hinaus beinhaltet die herkömmliche Art und Weise der Auswahl eines Datennetzwerks (einschließlich primärer Mobilfunkdatennetzwerke, CBRS-Netzwerke und WLAN-Netzwerke) die Verwendung des Modems des Benutzergeräts zum Scannen nach verfügbaren Netzwerken und die anschließende Verbindung mit einem der Netzwerke. Obwohl diese Technik funktioniert, ist sie oft sehr langsam und kann aufgrund schlechter Übergabesituationen zwischen Zellen oder Netzwerken zu Verbindungsabbrüchen führen. Dementsprechend besteht die Notwendigkeit, ein geeignetes CBRS-Profil für einen bestimmten Gerätekontext (z. B. Standort, Verkehr, Tageszeit) auszuwählen, bevor die Ergebnisse einer Modemabfrage empfangen werden oder ohne die Ergebnisse einer Modemabfrage zu verwenden, sodass das Benutzergerät schnell und zuverlässig zwischen einem primären Mobilfunkdatennetzwerk und einem assoziierten CBRS-Netzwerk wechseln kann.Additionally, the traditional way of selecting a data network (including primary cellular data networks, CBRS networks, and WiFi networks) involves using the user device's modem to scan for available networks and then connecting to one of the networks. Although this technology works, it is often very slow and can lead to disconnections due to poor handover situations between cells or networks. Accordingly, there is a need to select a suitable CBRS profile for a specific device context (e.g. location, traffic, time of day) before the results of a modem query are received or without using the results of a modem poll, so that the user equipment can quickly and reliably switch between a primary cellular data network and an associated CBRS network.

Die vorliegende Offenbarung beschreibt einen oder mehrere Aspekte für die Auswahl eines Kommunikationsprofils des Citizens Broadband Radio Service (nachfolgend „CBRS-Profil“) für eine Datenverbindung basierend auf Kontextdaten, die mit einem Benutzergerät assoziiert sind. Zu den für die Auswahl verwendeten Kontextdaten können zumindest eines der folgenden Profile gehören: das zur Zeit auf dem Benutzergerät ausgewählte Teilnehmeridentitätsmodul-Profil (SIM-Profil), Geolokalisierungsdaten, Mobilitätszoneninformationen des Benutzergeräts, eine von dem Benutzergerät gemessene CBRS-Netzwerkzelle, eine von dem Benutzergerät erfasste CBRS-Netzwerkzelle, eine von dem Benutzergerät gemessene Funkzugangsnetzverbindung (RAN-Verbindung), eine von dem Benutzergerät erfasste RAN-Verbindung, ein von dem Benutzergerät gemessenes Funkfrequenzband, ein von dem Benutzergerät erfasstes Funkfrequenzband, ein von dem Benutzergerät erfasster durchschnittlicher Datendurchsatz oder eine Zeit. Auf diese Weise kann ein Benutzergerät oder ein Remote-Dienst ein geeignetes CBRS-Profil für einen bestimmten Gerätekontext auswählen und gleichzeitig Zeitverzögerungen bei der Verbindung mit CBRS-Datennetzwerken und Übergaben zwischen einem Quellnetzwerk und dem ausgewählten CBRS-Netzwerk minimieren.The present disclosure describes one or more aspects for the selection of a communication profile of the Citizens Broadband Radio Service (hereinafter “CBRS profile”) for a data connection based on context data associated with a user device. The context data used for the selection can include at least one of the following profiles: the subscriber identity module profile (SIM profile) currently selected on the user device, geolocation data, mobility zone information of the user device, a CBRS network cell measured by the user device, one from the user device detected CBRS network cell, a radio access network connection (RAN connection) measured by the user device, a RAN connection detected by the user device, a radio frequency band measured by the user device, a radio frequency band detected by the user device, an average data throughput detected by the user device or a time . This allows a user device or remote service to select an appropriate CBRS profile for a particular device context while minimizing time delays in connecting to CBRS data networks and handovers between a source network and the selected CBRS network.
In einigen Aspekten wird ein Verfahren beschrieben, mit dem ein Benutzergerät ein CBRS-Profil auswählen kann. Im Rahmen des Verfahrens empfängt und speichert das Benutzergerät eine Vielzahl von CBRS-Profilen, die von dem Benutzergerät verwendet werden können. Basierend auf den empfangenen Kontextdaten wählt das Benutzergerät dann ein geeignetes CBRS-Profil aus und konfiguriert ein Modem für die Verbindung mit der assoziierten CBRS-Netzwerkzelle. Das Verfahren wird fortgesetzt, indem das Benutzergerät unter Nutzung des ausgewählten CBRS-Profils mit der assoziierten CBRS-Netzwerkzelle kommuniziert.In some aspects, a method is described by which a user device can select a CBRS profile. As part of the method, the user device receives and stores a large number of CBRS profiles that can be used by the user device. Based on the received context data, the user device then selects a suitable CBRS profile and configures a modem for connection to the associated CBRS network cell. The method continues with the user equipment communicating with the associated CBRS network cell using the selected CBRS profile.
In anderen Aspekten wird ein Verfahren zum Auswählen eines CBRS-Profils durch einen Remote-Dienst beschrieben. Im Rahmen des Verfahrens empfängt und speichert der Remote-Dienst eine Vielzahl von CBRS-Profilen, die von einem Benutzergerät verwendet werden können. Basierend auf den empfangenen Kontextdaten wählt der Remote-Dienst dann ein geeignetes CBRS-Profil aus und sendet das ausgewählte CBRS-Profil an das Benutzergerät, sodass das Benutzergerät ein Modem für die Verbindung mit der assoziierten CBRS-Netzwerkzelle konfigurieren kann. Das Verfahren wird fortgesetzt, indem das Benutzergerät unter Nutzung des ausgewählten CBRS-Profils mit der assoziierten CBRS-Netzwerkzelle kommuniziert.In other aspects, a method for selecting a CBRS profile by a remote service is described. As part of the process, the remote service receives and stores a variety of CBRS profiles that can be used by a user device. Based on the received context data, the remote service then selects an appropriate CBRS profile and sends the selected CBRS profile to the user device so that the user device can configure a modem to connect to the associated CBRS network cell. The method continues with the user equipment communicating with the associated CBRS network cell using the selected CBRS profile.

AUSFÜHRLICHE BESCHREIBUNGDETAILED DESCRIPTION
Mit der Möglichkeit, sowohl auf das primäre Mobilfunkdatennetzwerk als auch auf ein CBRS-Netzwerk zuzugreifen, kann ein Benutzergerät mit der Fähigkeit zur Aktivierung eines dynamischen dualen Standby-Modus bereitgestellt werden, wobei das Benutzergerät Daten in dem primären Mobilfunkdatennetzwerk oder in einem ausgewählten CBRS-Netzwerk aktivieren kann, wenn eine entsprechende Abdeckung verfügbar ist. Das Benutzergerät kann auch zwischen einem ersten CBRS-Netzwerk und einem zweiten CBRS-Netzwerk wechseln. Dies ähnelt einem Dual-SIM-Dual-Standby-Betrieb, wobei die Mobilfunkmodemarchitektur drahtlose Protokollstapel enthält, die sich in dem gleichen Benutzergerät befinden (die sich auf demselben Modemchip befinden können oder auch nicht) und einige oder alle der assoziierten Funkfrequenz-(RF)-Ressourcen teilen.With the ability to access both the primary cellular data network and a CBRS network, a user device can be provided with the ability to activate a dynamic dual standby mode, with the user device storing data on the primary cellular data network or on a selected CBRS network can activate if an appropriate cover is available. The user equipment can also switch between a first CBRS network and a second CBRS network. This is similar to dual SIM dual standby, with the cellular modem architecture including wireless protocol stacks residing in the same user device (which may or may not reside on the same modem chip) and some or all of the associated radio frequency (RF) -Share resources.
Vor einem Wechsel zu einem zweiten CBRS-Netzwerk aus einem primären Mobilfunkdatennetzwerk oder aus einem ersten CBRS-Netzwerk wird zunächst ein dem zweiten CBRS-Netzwerk entsprechendes CBRS-Profil aus einer Bibliothek von CBRS-Profilen ausgewählt. Da eine beliebige Anzahl an CBRS-Netzwerken zusammen mit einer beliebigen Anzahl an assoziierten CBRS-Profilen existieren kann, sollte für eine bestimmte Situation ein geeignetes CBRS-Profil ausgewählt werden, insbesondere angesichts der Tatsache, dass Netzwerkdaten aus dem primären Mobilfunkdatennetzwerk nicht verfügbar sind, wenn eine Verbindung zu dem CBRS-Netzwerk besteht. Da Modem-Scans viel Zeit in Anspruch nehmen, ist ein schnellerer Weg zur Auswahl eines CBRS-Profils erforderlich. Dementsprechend werden Verfahren und Systeme beschrieben, die die Auswahl eines geeigneten CBRS-Profils basierend auf Kontextinformationen über das Benutzergerät ermöglichen.Before changing to a second CBRS network from a primary cellular data network or from a first CBRS network, a CBRS profile corresponding to the second CBRS network is first selected from a library of CBRS profiles. Since any number of CBRS networks can coexist with any number of associated CBRS profiles, an appropriate CBRS profile should be selected for a particular situation, especially in view of the fact that network data from the primary cellular data network is not available when there is a connection to the CBRS network. Because modem scans are time consuming, a faster way to select a CBRS profile is needed. Accordingly, methods and systems are described that enable the selection of a suitable CBRS profile based on context information about the user device.
