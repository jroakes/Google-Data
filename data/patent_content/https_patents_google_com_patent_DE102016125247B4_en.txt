Abstract
Computerimplementiertes Verfahren, umfassend:das Einholen von Klassifizierungsdaten, die mit mehreren in einem Fahrzeug ausführbaren Anwendungen in Verbindung stehen, wobei die Klassifizierungsdaten, für jede von einer oder mehreren Anwendungen in den mehreren Anwendungen, einen Hinweis eines ersten Sicherheitsniveaus der Anwendung, einen Hinweis eines zweiten Sicherheitsniveaus der Anwendung, einen oder mehrere Fahrzeugbetriebsmodi, die mit dem ersten Sicherheitsniveau in Verbindung gebracht werden und während dem die Anwendung konfiguriert ist, im Fahrzeug ausgeführt zu werden, und einen oder mehrere Fahrzeugbetriebsmodi, die mit dem zweiten Sicherheitsniveau in Verbindung gebracht werden und während dem die Anwendung konfiguriert ist, nicht im Fahrzeug ausgeführt zu werden;das Erhalten eines Hinweises, eine erste Anwendung von den mehreren Anwendungen auszuführen;das Ermitteln eines Betriebszustands des Fahrzeugs;das Ermitteln dass das erste Sicherheitsniveau der ersten Anwendung nicht erfüllt wird auf der Grundlage, dass der Betriebszustand des Fahrzeugs dem einen oder mehreren Fahrzeugbetriebsmodi entspricht, die mit dem zweiten Sicherheitsniveau der ersten Anwendung in Verbindung gebracht werden;das Erzeugen, als Reaktion auf das Ermitteln, dass das erste Sicherheitsniveau der ersten Anwendung nicht erfüllt ist, einer Empfehlung zum Ausführen einer zweiten Anwendung, die Fahrzeugbetriebsmodi in einem ersten Sicherheitsniveau aufweist, die dem Betriebszustand des Fahrzeugs entsprechen;das Ausgeben der Empfehlung zum Ausführen der zweiten Anwendung;das Erhalten von Daten, die eine Benutzerauswahl angeben, um die zweite Anwendung auszuführen; unddas Ausführen der zweiten Anwendung im Fahrzeug, als Reaktion auf das Erhalten der Daten, die die Benutzerauswahl angeben, um die zweite Anwendung auszuführen.A computer-implemented method comprising:obtaining classification data associated with a plurality of applications executable in a vehicle, the classification data, for each of one or more applications in the plurality of applications, an indication of a first level of security of the application, an indication of a second Security levels of the application, one or more vehicle operating modes associated with the first level of security and during which the application is configured to run in the vehicle, and one or more vehicle operating modes associated with the second level of security and during which the application is configured not to run in the vehicle; receiving an indication to run a first application from the plurality of applications; determining an operating state of the vehicle; determining that the first security level of the first application is not met based on that the operating state of the vehicle corresponds to the one or more vehicle operating modes associated with the second level of safety of the first application; generating, in response to determining that the first level of safety of the first application is not met, a recommendation to execute a second Application having vehicle operating modes in a first safety level corresponding to the operating state of the vehicle; issuing the recommendation to execute the second application; obtaining data indicating a user selection to execute the second application; andexecuting the second application in the vehicle in response to receiving the data indicating the user selection to execute the second application.

TECHNISCHES GEBIETTECHNICAL FIELD
Diese Offenbarung betrifft im Allgemeinen Automobile.This disclosure generally relates to automobiles.
HINTERGRUNDBACKGROUND
Die Interaktionen zwischen Fahrzeugen und Fahrern nehmen immer mehr zu. Fahrzeugsysteme und Benutzerschnittstellen versetzen Fahrer in die Lage, mehrere wünschenswerte Anwendungen, wie beispielsweise etwa eine Navigationsanwendung für das Ausführen von navigationsbezogenen Operationen auszuwählen. Im Allgemeinen können verschiedene geeignete Anwendungen auf ein Fahrzeug heruntergeladen werden. Da die Anzahl der Anwendungen, die auch auf einem Fahrzeug installiert und ausgeführt werden können, immer weiter ansteigt, ist es wichtig, sich mit Themen hinsichtlich der Sicherheit, wie zum Beispiel der Ablenkung des Fahrers, auseinanderzusetzen.Interactions between vehicles and drivers are increasing. Vehicle systems and user interfaces enable drivers to select multiple desirable applications, such as a navigation application, for performing navigation-related operations. In general, various suitable applications can be downloaded onto a vehicle. As the number of applications that can be installed and run on a vehicle continues to increase, it is important to address safety issues such as driver distraction.

Wenn eine Anfrage gemäß Implementierungen erhalten wird, um eine Anwendung (app)für ein Fahrzeugverarbeitungssystem auszuführen, oder eine Änderung des Betriebszustands eines Fahrzeugs erfasst wird, kann das Fahrzeugverarbeitungssystem mehrere Aktionen durchführen, um die Ausführung der Anwendung im Fahrzeug zu aktivieren oder zu deaktivieren.According to implementations, when a request is received to execute an application (app) for a vehicle processing system or a change in the operating state of a vehicle is detected, the vehicle processing system may perform multiple actions to enable or disable execution of the application in the vehicle.
Wenn in einigen Implementierungen eine Anfrage erhalten wird, um eine Anwendung auszuführen, kann das Fahrzeugverarbeitungssystem Klassifizierungsdaten erhalten, welche die Anwendung und den gegenwärtigen Betriebszustand des Fahrzeugs einstufen. Die Klassifizierungsdaten können einen Anwendungstyp der Anwendung und die Arten der Betriebszustände des Fahrzeugs, unter denen die Anwendung ausgeführt werden kann, angeben. Zum Beispiel können die Klassifizierungsdaten einer Textnachrichtenanwendung angeben, dass es sich bei der Textnachrichtenanwendung um eine Front-End-Anwendung handelt, die eine grafische Benutzerschnittstelle verwendet, aber die nur ausgeführt werden sollte, wenn sich das Fahrzeug in einem Parkmodus befindet oder sich nicht bewegt. Anwendungen, die beim Fahren verwendet werden können, können einen spezifischen grafischen Anzeiger anzeigen, bei dem es sich um jede dafür geeignete grafische Darstellung handeln kann, wie etwa ein Symbol, ein Bild, ein Design, ein Text, oder ein Muster. Auf Basis der Klassifizierungsdaten, des Betriebszustands des Fahrzeugs und der einen oder der mehreren Regeln, ermittelt das Fahrzeugverarbeitungssystem, ob die Ausführung der Anwendung aktiviert werden soll oder ob die Ausführung der Anwendung verweigert werden soll.In some implementations, when a request is received to execute an application, the vehicle processing system may obtain classification data that classifies the application and the current operating state of the vehicle. The classification data may indicate an application type of the application and the types of vehicle operating conditions under which the application can be executed. For example, the classification data of a text messaging application may indicate that the text messaging application is a front-end application that uses a graphical user interface, but that should only run when the vehicle is in a parking mode or not moving. Applications that can be used while driving can display a specific graphical indicator, which can be any appropriate graphical representation, such as a symbol, an image, a design, a text, or a pattern. Based on the classification data, the operating state of the vehicle and the one or more rules, the vehicle processing system determines whether execution of the application should be enabled or whether execution of the application should be denied.
Wenn eine Änderung im Betriebszustand eines Fahrzeugs in einigen Implementierungen erfasst wird, ermittelt das Fahrzeugverarbeitungssystem, welche Anwendungen im Fahrzeug ausgeführt werden. Für eine oder mehrere der Fahrzeuganwendungen, die zur Zeit der Änderung des Betriebszustands des Fahrzeugs ausgeführt werden, kann das Fahrzeugverarbeitungssystem Klassifizierungsdaten einholen, welche die Anwendung einstufen. Auf Basis der Klassifizierungsdaten, des Betriebszustands des Fahrzeugs und der einen oder der mehreren Regeln, ermittelt das Fahrzeugverarbeitungssystem, ob die Ausführung der Anwendung fortgeführt werden soll, oder ob die Ausführung der Anwendung nicht fortgeführt werden soll. Zum Beispiel kann eine elektronische Spielanwendung in manchen Fällen als eine Anwendung klassifiziert werden, die ausgeführt werden sollte, wenn das Fahrzeug sich in einem Parkmodus befindet oder sich nicht bewegt. Falls der Modus des Fahrzeugs demzufolge zum Beispiel vom Parkmodus in den Fahrmodus geändert wird und das Fahrzeug beginnt sich zu bewegen, deaktiviert das Fahrzeugverarbeitungssystem des Fahrzeugs die elektronische Spielanwendung, damit der Fahrer nicht abgelenkt wird, während sich das Fahrzeug in Bewegung befindet. Das Fahrzeugverarbeitungssystem kann eine Fahrsicherheitsnachricht ausgeben, die in einigen Implementierungen nach einer zuvor festgelegten Zeitdauer eingestellt werden kann. Der standardmäßige Hauptbildschirm kann vom Fahrzeugverarbeitungssystem angezeigt werden, nachdem die Fahrsicherheitsnachricht nicht mehr länger ausgegeben wird.In some implementations, when a change in the operating state of a vehicle is detected, the vehicle processing system determines which applications are running in the vehicle. For one or more of the vehicle applications that are executing at the time of the vehicle operating state change, the vehicle processing system may obtain classification data that classifies the application. Based on the classification data, the operating state of the vehicle and the one or more rules, the vehicle processing system determines whether execution of the application should continue or whether execution of the application should not continue. For example, in some cases, an electronic gaming application may be classified as an application that should be executed when the vehicle is in a parking mode or not moving. Accordingly, if the mode of the vehicle is changed from parking mode to driving mode, for example, and the vehicle begins to move, the vehicle's vehicle processing system disables the electronic gaming application so that the driver is not distracted while the vehicle is in motion. The vehicle processing system may issue a driving safety message, which in some implementations may be set after a predetermined period of time. The default main screen may be displayed by the vehicle processing system after the driving safety message is no longer issued.
Falls das Fahrzeug sich in Bewegung setzt oder ein Wechsel aus dem Parkmodus in einen Fahrmodus oder Rückfahrmodus erfasst wird, ermittelt ein Prozessor im Fahrzeug alle Anwendungen, die gegenwärtig auf dem Fahrzeug laufen und ob diese Anwendungen fortgesetzt werden dürfen, wenn sich das Fahrzeug in Bewegung befindet. Wie nachstehend in weiteren Einzelheiten erläutert wird, werden eine oder mehrere Regeln und Bedingungen und Daten verwendet, welche die entsprechenden Anwendungen einstufen, um zu ermitteln, welche Anwendungen weiterhin ausgeführt werden dürfen, wenn sich der Betriebszustand des Fahrzeugs ändert.If the vehicle starts moving or a transition from park mode to drive mode or reverse mode is detected, a processor in the vehicle determines all applications currently running on the vehicle and whether those applications are allowed to continue while the vehicle is in motion . As explained in more detail below, one or more rules and conditions and data that rank the appropriate applications are used to determine which applications are allowed to continue running when the operating state of the vehicle changes.

Eine Front-End-Anwendung bezieht sich auf eine Anwendung, mit der ein Fahrer auf direkte Weise interagieren kann. Beispiele von Front-End-Anwendungen beinhalten unter anderem auch eine Telefonanrufanwendung, eine Webbrowser-Anwendung, eine Textnachrichtenanwendung und eine Navigationsanwendung.A front-end application refers to an application that a driver can interact with in a direct manner. Examples of front-end applications also include, but are not limited to, a telephone calling application, a web browser application, a text messaging application, and a navigation application.
Eine Back-End-Anwendung bezieht sich auf eine Service Anwendung, die keine direkte Interaktion mit einem Fahrer erfordert. Back-End-Anwendungen können mit Front-End-Anwendungen verbunden werden. Beispiele von Back-End-Anwendungen beinhalten unter anderem auch Anwendungen, welche die Leistung des Fahrzeugs oder Teile des Fahrzeugs überwachen, Anwendungen, welche bestimmte Funktionen, wie etwa die Beleuchtung oder die Scheibenwischeraktivierung steuern, oder Anwendungen, die Services für andere Anwendungen bereitstellen, wie etwa Anbieter von Inhalten und Rundfunkempfänger.A back-end application refers to a service application that does not require direct interaction with a driver. Back-end applications can be connected to front-end applications. Examples of back-end applications also include, but are not limited to, applications that monitor the performance of the vehicle or parts of the vehicle, applications that control certain functions such as lighting or windshield wiper activation, or applications that provide services to other applications, such as such as content providers and broadcast receivers.
