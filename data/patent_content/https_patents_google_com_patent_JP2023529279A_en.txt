Abstract
本明細書に記載の実装形態は、ユーザが特定の選択可能なオートフィル提案を選択する前に、サーバコマンドデータを使用して少なくとも部分的に履行されるアプリケーションアクションに対応する、選択可能なオートフィル提案を提供することに関する。このようにコマンドデータを積極的に履行することで、ユーザが提案を選択してから特定のアクションを履行するまでの待ち時間が短縮される。最初に、部分入力を処理してオートフィル提案を生成できる。この提案は、サーバデバイスに通信してさらに処理することができる。オートフィル提案は、タッチディスプレイインターフェースでの選択のためにレンダリングすることもでき、それにより、ユーザはオートフィル提案の1つを選択できる。コマンド履行データがサーバによって提供されると、コマンド履行データは、対応するアクションがユーザ選択の前に少なくとも部分的に履行され得るように、対応するアプリケーションで利用可能になる。Implementations described herein relate to providing selectable autofill suggestions that correspond to application actions that are at least partially fulfilled using server command data before a user selects a particular selectable autofill suggestion. Aggressive fulfillment of command data in this manner reduces the latency between a user selecting a suggestion and fulfilling a particular action. First, the partial input can be processed to generate autofill suggestions. The proposal can be communicated to a server device for further processing. Autofill suggestions can also be rendered for selection on a touch display interface, allowing a user to select one of the autofill suggestions. Once the command fulfillment data is provided by the server, the command fulfillment data is made available to the corresponding application so that the corresponding action can be at least partially fulfilled prior to user selection.
