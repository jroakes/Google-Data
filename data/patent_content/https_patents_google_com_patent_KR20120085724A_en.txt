Abstract
배터리 동작형 전자 디바이스에서 전형 소비에 따른 피드백을 사용자에게 제공하기 위한, 컴퓨터 저장 매체에 인코딩된 컴퓨터 프로그램을 포함하는 방법, 시스템 및 장치에 관한 것이다. 일 면에서, 하나 이상의 데이터 처리 장치들에 의해 실행되는 방법이다. 방법은 데이터 처리 장치에서, 서로 다른 사용자에 의해 구동되는 배터리 동작형 전자 디바이스들의 컬렉션에 의한 전력 소비의 이력 기록을 전송받는 단계, 상기 배터리 동작형 전자 디바이스의 서로 다른 클래스들에서 전력 소비의 대표적인 특징들을 생성하기 위해, 상기 데이터 처리 장치에 의해 상기 이력 기록을 집계하고 분석하는 단계, 및 상기 데이터 처리 장치로부터, 상기 서로 다른 클래스들에서 구동되는 서로 다른 애플리케이션에 의한 전력 소비의 대표적인 특징들을 출력하는 단계를 포함하며, 각 배터리 동작형 전자 디바이스들은 클래스에 속한다.A method, system and apparatus comprising a computer program encoded on a computer storage medium for providing a user with feedback on typical consumption in a battery operated electronic device. In one aspect, the method is executed by one or more data processing devices. The method comprises receiving, at a data processing apparatus, a history record of power consumption by a collection of battery operated electronic devices driven by different users, representative of power consumption in different classes of the battery operated electronic device. Aggregating and analyzing the history record by the data processing device to produce the data, and outputting representative features of power consumption by different applications running in the different classes from the data processing device. Wherein each battery operated electronic device belongs to a class.

Using Wake Lock: Wake lock is a lock held by an application (or a version of an application) to prevent the data processing unit from entering sleep or other power saving modes. As a result, the data processing unit consumes power that will not be consumed if it enters the power saving mode. Multiple applications can maintain wake locks at the same time. In converting the maintenance of the wake lock to power consumption that can be allocated to the application, the request or release for each wake lock by the application (or version of the application) is obtained as a trigger that distinguishes the end of the period. Can be. The total power consumption for that period can be determined by multiplying the time of the period with power not saved by the data processing unit not entering the power saving mode. In column 430 (FIG. 4) of table 410, this total power consumption can be divided by the number of applications that are keeping a wake lock during that period, and the resulting share can be attributed to such applications.

Hardware sensors such as accelerometers, compasses, digital cameras, Global Positioning System (GPS) : Applications using such sensors may be required to request that data from the sensor be passed to the application. 애플리케이션들에 의한 전력 소비의 기록들은 애플리케이션들이 다수의 개발자들에 의해 개발된 경우 특히 적절하다. 특히, 다른 개발자들은 다른 레벨의 전문 지식을 가지고, 다른 전력량을 소비하는 유사 애플리케이션들을 개발한다. 다른 애플리케이션들에 의해 소비된 전력량이 정격일 때, 사용자는 애플리케이션을 설치하거나 또는 실행할지를 결정하는데 인자(factor)로서 전력 소비를 고려한다. 이는 유사한 기능을 갖는 다른 애플리케이션들과 하나의 애플리케이션이 서로 대체물로서 간주되는 경우 특히 적합하다. Records of power consumption by applications are particularly relevant when applications are developed by multiple developers. In particular, different developers have similar levels of expertise and develop similar applications that consume different amounts of power. When the amount of power consumed by other applications is rated, the user considers power consumption as a factor in determining whether to install or run the application. This is particularly suitable when other applications with similar functionality and one application are regarded as a substitute for each other.
