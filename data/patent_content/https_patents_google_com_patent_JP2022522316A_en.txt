Abstract
コンテンツプラットフォーム上で表示される特定のコンテンツとの対話が実際の人のユーザによる対話であるかどうかを検証するための、コンピュータ記憶媒体に符号化されているコンピュータプログラムを含む方法、システム、および装置。クリック検証トークンを生成するようにとの要求が、クライアントデバイス上で実行されている第1のアプリケーションから受信され得る。要求は、第1の対話のためのパラメータの第1のセットに対するデータおよび第1のメッセージ認証コード(MAC)セットを含み得る。第1のMACセットが、要求に含まれるデータの一部を使用して生成される第2のMACセットと一致する場合、クリック検証トークンが生成され得る。クリック検証トークンは第1のアプリケーションに提供されてもよく、第1のアプリケーションは次いでコンテンツ検証システムにこのトークンを提供することができる。コンテンツ検証システムは、このトークンを使用して、第1の対話が実際の人のユーザ対話であるかどうかを検証することができる。Methods, systems, and devices that include computer programs encoded in computer storage media to verify whether a dialogue with a particular content displayed on a content platform is a dialogue by a real person user. .. A request to generate a click validation token may be received from a first application running on the client device. The request may include data for a first set of parameters for the first dialogue and a first set of message authentication codes (MAC). A click validation token can be generated if the first MAC set matches the second MAC set generated using some of the data contained in the request. The click verification token may be provided to the first application, which can then provide this token to the content validation system. The content verification system can use this token to verify whether the first dialogue is a real human user dialogue.
