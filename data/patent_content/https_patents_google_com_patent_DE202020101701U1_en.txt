Abstract
System, das einen oder mehrere Computer umfasst, und eine oder mehrere Speichervorrichtungen, die Anweisungen speichern, die dann, wenn sie durch den einen oder die mehreren Computer ausgeführt werden, betreibbar sind, um zu veranlassen, dass der eine oder die mehreren Computer Operationen durchführt oder durchführen, die folgendes umfassen:
Erhalten eines Trainingsdatensatzes zum Trainieren eines Maschinenlernmodells mit einer Vielzahl von Netzparametern, um eine bestimmte Maschinenlernaufgabe durchzuführen, wobei der Trainingsdatensatz eine Vielzahl von Trainingseingaben umfasst;
Bestimmen einer Vielzahl von Datenvermehrungsstrategien, wobei jede Datenvermehrungsstrategie eine Prozedur zum Verarbeiten einer Trainingseingabe definiert, um eine transformierte Trainingseingabe zu erzeugen, wobei die Prozedur ein Auswählen einer Sequenz von einer oder mehreren Transformationsoperationen umfasst, um auf die Trainingseingabe angewendet zu werden, und wobei das Bestimmen für jede Datenvermehrungsstrategie folgendes umfasst:
Bestimmen eines Werts für einen ersten Hyperparameter, der eine selbe Anzahl von Transformationsoperationen in jeder Sequenz von Transformationsoperationen spezifiziert, die zum Transformieren einer jeweiligen Trainingseingabe ausgewählt sind; und
Bestimmen eines jeweiligen Werts für jeden von einem oder mehreren zweiten Hyperparametern, wobei die zweiten Hyperparameter einen selben Größenplan zum Bestimmen einer Größe für jede Transformationsoperation in jeder Sequenz von Transformationsoperationen spezifizieren, die zum Transformieren einer jeweiligen Trainingseingabe ausgewählt sind;
für jede Datenvermehrungsstrategie Trainieren des Maschinenlernmodells unter Verwendung der Datenvermehrungsstrategie, umfassend:
Bestimmen eines vermehrten Trainingsdatensatzes durch Verarbeiten der Trainingseingaben im Trainingsdatensatz, um transformierte Trainingseingaben gemäß der Datenvermehrungsstrategie zu erzeugen,
Verarbeiten einer bestimmten transformierten Trainingseingabe im vermehrten Trainingsdatensatz unter Verwendung aktueller Werte für die Vielzahl von Netzparametern des Maschinenlernmodells, um eine Trainingsausgabe zu erzeugen, und
Erzeugen eines Parameter-Updates für die Vielzahl von Parametern des Maschinenlernmodells basierend auf einem Fehler in der Trainingsausgabe;
Bestimmen, für jede Datenvermehrungsstrategie, eines Qualitätsma-ßes des Maschinenlernmodells, das unter Verwendung der Datenvermehrungsstrategie trainiert worden ist, wobei das Qualitätsmaß des Maschinenlernmodells eine Leistungsfähigkeit des Maschinenlernmodells an der bestimmten Maschinenlernaufgabe als Ergebnis eines Trainierens des Maschinenlernmodells unter Verwendung der Datenvermehrungsstrategie darstellt; und
Auswählen einer endgültigen Datenvermehrungsstrategie basierend auf einem Verwenden der Qualitätsmaße der Maschinenlernmodelle.
A system comprising one or more computers and one or more storage devices that store instructions that, when executed by the one or more computers, are operable to cause the one or more computers to perform operations or perform that include:
Obtaining a training data set for training a machine learning model with a plurality of network parameters to perform a specific machine learning task, the training data set comprising a plurality of training inputs;
Determining a plurality of data augmentation strategies, each data augmentation strategy defining a procedure for processing a training input to generate a transformed training input, the procedure comprising selecting a sequence of one or more transformation operations to be applied to the training input, and wherein the determining for any data replication strategy includes:
Determining a value for a first hyperparameter that specifies an equal number of transform operations in each sequence of transform operations selected to transform a respective training input; and
Determining a respective value for each of one or more second hyperparameters, the second hyperparameters specifying a same size map for determining a size for each transform operation in each sequence of transform operations selected to transform a respective training input;
for each data replication strategy, training the machine learning model using the data replication strategy, comprising:
Determining an augmented training data set by processing the training inputs in the training data set in order to generate transformed training inputs according to the data augmentation strategy,
Processing a particular transformed training input in the augmented training data set using current values for the plurality of network parameters of the machine learning model in order to generate a training output, and
Generating a parameter update for the plurality of parameters of the machine learning model based on an error in the training output;
Determining, for each data replication strategy, a quality measure of the machine learning model that has been trained using the data replication strategy, the quality measure of the machine learning model representing a performance of the machine learning model on the particular machine learning task as a result of training the machine learning model using the data replication strategy; and
Selecting a final data replication strategy based on using the quality measures of the machine learning models.

HINTERGRUNDBACKGROUND
Diese Beschreibung betrifft ein Verarbeiten von Maschinenlernmodellen.This description relates to processing machine learning models.
Maschinenlernmodelle empfangen eine Eingabe und erzeugen eine Ausgabe, wie z.B. eine vorhergesagte Ausgabe, basierend auf der empfangenen Eingabe. Einige Maschinenlernmodelle sind parametrische Modelle und erzeugen die Ausgabe basierend auf der empfangenen Eingabe und auf Werten der Parameter des Modells.Machine learning models receive input and generate output, such as a predicted output based on the input received. Some machine learning models are parametric models and generate the output based on the input received and on values of the parameters of the model.
Einige Maschinenlernmodelle sind tiefe Modelle, die mehrere Schichten von Modellen verwenden, um eine Ausgabe für eine empfangene Eingabe zu erzeugen. Zum Beispiel ist ein tiefes neuronales Netz ein tiefes Maschinenlernmodell, das eine Ausgabeschicht und eine oder mehrere versteckte Schichten enthält, die jeweils eine nichtlineare Transformation auf eine empfangene Eingabe anwenden, um eine Ausgabe zu erzeugen.Some machine learning models are deep models that use multiple layers of models to generate output for a received input. For example, a deep neural network is a deep machine learning model that includes an output layer and one or more hidden layers, each of which applies a nonlinear transform to a received input to produce an output.
ZUSAMMENFASSUNGSUMMARY
Diese Beschreibung beschreibt ein als Computerprogramme auf einem oder mehreren Computern bei einem oder mehreren Standorten implementiertes System, das eine Datenvermehrungsstrategie zum Vermehren eines Trainingsdatensatzes auswählt. Der Trainingsdatensatz wird zum Trainieren eines Maschinenlernmodells verwendet, um eine bestimmte Maschinenlernaufgabe durchzuführen; zum Beispiel kann der Trainingsdatensatz eine Gruppe von Bildern zum Trainieren eines Computervision-Maschinenlernmodells sein, wie z.B. ein Bildklassifikations- oder ein Bildregressionsmodell. Eine Datenvermehrungsstrategie kann verwendet werden, um die Quantität und Diversität der beim Trainieren des Maschinenlernmodells verwendeten Trainingseingaben zu erhöhen, um dadurch in dem trainierten Maschinenlernmodell zu resultieren, das die Maschinenlernaufgabe effektiver (mit größerer Vorhersagegenauigkeit und besserer Verallgemeinerung) durchführt.This specification describes a system implemented as computer programs on one or more computers at one or more locations that selects a data replication strategy for replicating a training data set. The training data set is used to train a machine learning model to perform a specific machine learning task; for example, the training data set can be a group of images for training a computer vision machine learning model, such as an image classification or an image regression model. A data augmentation strategy can be used to increase the quantity and diversity of training inputs used in training the machine learning model, thereby resulting in the trained machine learning model that performs the machine learning task more effectively (with greater predictive accuracy and better generalization).
Die Datenvermehrungsstrategie kann eine Prozedur zum Transformieren einer Trainingseingabe im Trainingsdatensatz unter Verwendung einer Sequenz von einer oder mehreren Transformationsoperationen definieren. Unter Verwendung von in dieser Beschreibung beschriebenen Techniken kann ein Datenvermehrungssystem schnell und effizient optimale Werte für einen oder mehrere Hyperparameter der Datenvermehrungsstrategie bestimmen, die für jede Trainingseingabe im Trainingsdatensatz spezifizieren, wie die Transformationsoperationen in der Sequenz von Transformationsoperationen auszuwählen sind.The data augmentation strategy may define a procedure for transforming a training input in the training data set using a sequence of one or more transform operations. Using techniques described in this specification, a data augmentation system can quickly and efficiently determine optimal values for one or more hyperparameters of the data augmentation strategy that specify for each training input in the training dataset how to select the transform operations in the sequence of transform operations.
Der in dieser Beschreibung beschriebene Gegenstand kann in bestimmten Ausführungsformen implementiert werden, um einen oder mehrere der folgenden Vorteile zu realisieren.The subject matter described in this specification can be implemented in certain embodiments to realize one or more of the following advantages.
Einige existierende Systeme versuchen, Datenvermehrungsstrategien durch Durchführen eine „Suchphase“ vor einem Trainieren des Maschinenlernmodells zu lernen. Während einer solchen Phase durchsuchen die existierenden Systeme normalerweise einen großen Suchraum von Kandidatenvermehrungsstrategien, um eine bestimmte Kandidatenvermehrungsstrategie zu finden. Diese Suchphase kann oft zeitaufwendig und rechenaufwendig sein. Unter Verwendung von in dieser Beschreibung beschriebenen Techniken kann ein Datenvermehrungssystem eine optimale Datenvermehrungsstrategie zum Trainieren eines Maschinenlernmodells ohne Ausführen einer Suchphase vor dem Trainieren bestimmen. Vielmehr kann das Datenvermehrungssystem optimale Werte für die Hyperparameter der Datenvermehrungsstrategie parallel zu einem Bestimmen anderer Hyperparameter des Maschinenlernmodells selbst bestimmen. Weiterhin kann das Datenvermehrungssystem nach optimalen Werten für die Hyperparameter in einem viel kleineren Suchraum als einige existierende Systeme suchen. Zum Beispiel kann es bei einigen Implementierungen des in dieser Beschreibung beschriebenen Datenvermehrungssystems nur z.B. 2, 4 oder 6 Hyperparameter geben, die bestimmt werden müssen, und kann der Raum von möglichen Werten für jeden Hyperparameter auf einfache Weise diskretisiert werden. Somit kann die Suche signifikant schneller und weniger rechenaufwendig als existierende Systeme sein, die ungeheuer große Suchräume durchsuchen müssen.Some existing systems attempt to learn data augmentation strategies by performing a "search phase" prior to training the machine learning model. During such a phase, the existing systems normally search a large search space of candidate growth strategies in order to find a particular candidate growth strategy. This search phase can often be time consuming and computationally intensive. Using techniques described in this specification, a data replication system can determine an optimal data replication strategy for training a machine learning model without performing a search phase prior to training. Rather, the data replication system can itself determine optimal values for the hyperparameters of the data replication strategy in parallel with determining other hyperparameters of the machine learning model. Furthermore, the data augmentation system can search for optimal values for the hyperparameters in a much smaller search space than some existing systems. For example, in some implementations of the data replication system described in this specification it may only e.g. There are 2, 4, or 6 hyperparameters that need to be determined, and the space of possible values for each hyperparameter can be discretized in a simple manner. Thus, the search can be significantly faster and less computationally expensive than existing systems that have to search vastly large search spaces.
Einige existierende Techniken versuchen, eine optimale Datenvermehrungsstrategie durch Trainieren eines „Spielzeug“- bzw. „kleinen“ oder „Proxy“-Maschinenlernmodells unter Verwendung von mehreren Kandidaten-Datenvermehrungsstrategien und Auswerten der Leistungsfähigkeit der trainierten kleinen Maschinenlernmodelle bzw. Spielzeug-Maschinenlernmodelle zu lernen. Die Spielzeug-Maschinenlernmodelle sind normalerweise signifikant kleiner als die Maschinenlernmodelle, die letztlich unter Verwendung der ausgewählten Datenvermehrungsstrategie trainiert werden; z.B. können die kleinen Maschinenlernmodelle bzw. Spielzeug-Maschinenlernmodelle viel weniger Parameter als das richtige Maschinenlernmodell haben. Weiterhin trainieren die existierenden Systeme die Spielzeug-Maschinenlernmodelle oft durch Vermehren eines kleinen Trainingsdatensatzes bzw. Spielzeug-Trainingsdatensatzes, der viel kleiner als der Trainingsdatensatz ist, der letztlich unter Verwendung der ausgewählten Datenvermehrungsstrategie vermehrt werden wird; d.h. der kleine Trainingsdatensatz hat weniger Trainingseingaben als der richtige Trainingsdatensatz. Ein Auswählen einer Datenvermehrungsstrategie basierend auf der Leistungsfähigkeit von kleinen Maschinenlernmodellen, die an einem kleinen Trainingsdatensatz trainiert sind, und darauffolgendes Verwenden der Datenvermehrungsstrategie, um ein großes Maschinenlernmodell unter Verwendung eines großen Trainingsdatensatzes zu trainieren, kann ineffektive Ergebnisse ergeben, weil die optimalen Parameter für eine Datenvermehrungsstrategie zum Trainieren eines kleinen Maschinenlernmodells unter Verwendung eines kleinen Trainingsdatensatzes oft nicht die optimalen Parameter für eine Datenvermehrungsstrategie zum Trainieren eines großen Maschinenlernmodells unter Verwendung eines großen Trainingsdatensatzes sind. Unter Verwendung von in dieser Beschreibung beschriebenen Techniken kann ein Datenvermehrungssystem eine optimale Datenvermehrungsstrategie zum Trainieren eines Maschinenlernmodells an einem Trainingsdatensatz durch Auswerten der Leistungsfähigkeit von Kandidaten-Datenvermehrungsstrategien beim Trainieren des Maschinenlernmodells selbst unter Verwendung des Trainingsdatensatzes selbst bestimmen. Das bedeutet, dass das Datenvermehrungssystem Kandidaten-Datenvermehrungsstrategien selbst dann effizient auswerten kann, wenn das vollständige Maschinenlernmodell an dem vollständigen Trainingsdatensatz trainiert wird, um dadurch die Notwendigkeit zum Trainieren von Spielzeug-Maschinenlernmodellen bzw. kleinen Maschinenlernmodellen an Spielzeug-Trainingsdatensätzen bzw. kleinen Trainingsdatensätzen zu eliminieren.Some existing techniques attempt to learn an optimal data replication strategy by training a "toy" or "small" or "proxy" machine learning model using multiple candidate data replication strategies and evaluating the performance of the trained small machine learning models or toy machine learning models. The toy machine learning models are typically significantly smaller than the machine learning models that are ultimately trained using the selected data replication strategy; For example, the small machine learning models or toy machine learning models can have far fewer parameters than the correct machine learning model. Furthermore, the existing systems often train the toy machine learning models by multiplying a small training data set or toy training data set that is much smaller than the training data set ultimately produced using the selected data multiplication strategy will be increased; ie the small training data set has fewer training inputs than the correct training data set. Selecting a data replication strategy based on the performance of small machine learning models trained on a small training data set, and then using the data replication strategy to train a large machine learning model using a large training data set, can give ineffective results because the optimal parameters for a data replication strategy for training a small machine learning model using a small training data set are often not the optimal parameters for a data replication strategy for training a large machine learning model using a large training data set. Using techniques described in this specification, a data replication system can determine an optimal data replication strategy for training a machine learning model on a training data set by evaluating the performance of candidate data replication strategies in training the machine learning model itself using the training data set itself. This means that the data replication system can efficiently evaluate candidate data replication strategies even if the complete machine learning model is trained on the complete training data set, thereby eliminating the need for training toy machine learning models or small machine learning models on toy training data sets or small training data sets .
Als ein bestimmtes Beispiel kann die optimale Größe von Transformationsoperationen von Datenvermehrungsstrategien mit sowohl der Größe des Maschinenlernmodells als auch der Größe des Trainingsdatensatzes größer werden. Beim Verwenden einiger existierender Techniken könnte die ausgewählte Größe der Transformationsoperationen für alle Maschinenlernmodelle und alle Trainingsdatensätze konstant sein. Das in dieser Beschreibung beschriebene System kann die Grö-ßen der Transformationsoperationen auf das spezifische Maschinenlernmodell und den spezifischen Trainingsdatensatz abstimmen. Das System kann auch die Größen gemäß einem Größenplan variieren, wenn das Trainieren fortschreitet, was für grö-ßere Maschinenlernmodelle vorteilhaft ist.As a particular example, the optimal size of data replication strategy transformation operations may increase with both the size of the machine learning model and the size of the training data set. Using some existing techniques, the selected size of the transform operations could be constant for all machine learning models and all training data sets. The system described in this description can adapt the sizes of the transformation operations to the specific machine learning model and the specific training data set. The system can also vary sizes according to a size plan as training progresses, which is beneficial for larger machine learning models.
Bei einigen Implementierungen kann ein Datenvermehrungssystem unter Verwendung von in dieser Beschreibung beschriebenen Techniken eine Datenvermehrungsstrategie lernen, die zwischen unterschiedlichen Trainingsdatengruppen transferierbar ist. Das bedeutet, dass eine unter Bezugnahme auf einen ersten Trainingsdatensatz gelernte Datenvermehrungsstrategie verwendet werden kann, um ein Maschinenlernmodell an einem zweiten Trainingsdatensatz effektiv zu trainieren (d.h. selbst wenn die Datenvermehrungsstrategie nicht durch Bezugnahme auf den zweiten Trainingsdatensatz gelernt wurde). Die Transferierbarkeit der durch das Datenvermehrungssystem gelernten Datenvermehrungsstrategien kann signifikante Effizienzgewinne ergeben, da gelernte Datenvermehrungsstrategien an neuen Trainingsdatensätzen erneut verwendet werden können, ohne dass es nötig ist, zusätzliche rechenaufwendige Suchprozesse zu verwenden, um eine neue Datenvermehrungsstrategie zu lernen.In some implementations, using techniques described in this specification, a data replication system can learn a data replication strategy that is transferable between different training data groups. This means that a data replication strategy learned with reference to a first training data set can be used to effectively train a machine learning model on a second training data set (i.e. even if the data replication strategy was not learned by referring to the second training data set). The transferability of the data replication strategies learned by the data replication system can result in significant efficiency gains, since learned data replication strategies can be used again on new training data sets without it being necessary to use additional computationally expensive search processes in order to learn a new data replication strategy.

DETAILLIERTE BESCHREIBUNGDETAILED DESCRIPTION
Diese Beschreibung beschreibt ein System, das eine Datenvermehrungsstrategie zum Trainieren eines Maschinenlernmodells an einem Trainingsdatensatz erzeugt.This description describes a system that generates a data replication strategy for training a machine learning model on a training data set.
Zum Beispiel kann die Maschinenlernaufgabe eine Spracherkennungsaufgabe sein, wo das Maschinenlernmodell konfiguriert ist, um eine Darstellung einer Audio-Wellenform zu verarbeiten, um eine Ausgabe zu erzeugen, die eine Sequenz von Phonemen, Zeichen oder Wörtern entsprechend der Audio-Wellenform charakterisiert.For example, the machine learning task may be a speech recognition task where the machine learning model is configured to process a representation of an audio waveform to produce output characterizing a sequence of phonemes, characters, or words corresponding to the audio waveform.
Als ein weiteres Beispiel kann die Maschinenlernaufgabe eine Videoanalyseaufgabe sein, wo das Maschinenlernmodell konfiguriert ist, um eine Sequenz von Videoframes bzw. Video-Einzelbildern zu verarbeiten, um eine Ausgabe zu erzeugen, die die Videoframes charakterisiert, wie z.B. durch Charakterisieren, ob die Videoframes eine Person darstellen, die eine bestimmte Aktion durchführt. As another example, the machine learning task may be a video analysis task where the machine learning model is configured to process a sequence of video frames to generate output that characterizes the video frames, such as by characterizing whether the video frames have a Represent a person who takes a certain action.
Als ein weiteres Beispiel kann die Maschinenlernaufgabe eine Aufgabe einer Verarbeitung natürlicher Sprache sein, wo das Maschinenlernmodell konfiguriert ist, um einen Teilbereich von Text zu verarbeiten, um eine Ausgabe zu erzeugen, die den Teilbereich von Text charakterisiert, wie z.B. durch Charakterisieren einer Übersetzung des Teilbereichs von Text in eine andere natürliche Sprache.As another example, the machine learning task may be a natural language processing task where the machine learning model is configured to process a portion of text to produce output characterizing the portion of text, such as e.g. by characterizing a translation of the portion of text into another natural language.
Als ein weiteres Beispiel kann die Maschinenlernaufgabe eine Bildverarbeitungsaufgabe sein, wo das Maschinenlernmodell konfiguriert ist, um eine Eingabe zu verarbeiten, die ein Bild enthält, um eine entsprechende Ausgabe zu erzeugen, wie z.B. eine Klassifikationsausgabe, eine Regressionsausgabe oder eine Kombination davon.As another example, the machine learning task may be an image processing task where the machine learning model is configured to process an input containing an image to produce a corresponding output, such as a classification output, a regression output, or a combination thereof.
Als ein bestimmtes Beispiel kann das Maschinenlernmodell konfiguriert sein, um ein Bild zu verarbeiten, um eine Klassifikationsausgabe zu erzeugen, die eine jeweilige Kennzahl entsprechend jeder von mehreren Kategorien enthält. Die Kennzahl für eine Kategorie zeigt eine Wahrscheinlichkeit an, dass das Bild zu der Kategorie gehört. In einigen Fällen können die Kategorien Klassen von Objekten (z.B. Hund, Katze, Person und ähnliches) sein und kann das Bild zu einer Kategorie gehören, wenn es ein Objekt darstellt, das in der Objektklasse entsprechend der Kategorie enthalten ist. In einigen Fällen können die Kategorien globale Bildeigenschaften (wie z.B., ob das Bild eine Szene am Tag oder bei Nacht darstellt oder ob das Bild eine Szene im Sommer oder im Winter darstellt) enthalten und kann das Bild zu der Kategorie gehören, wenn es die globale Eigenschaft entsprechend der Kategorie hat.As a particular example, the machine learning model can be configured to process an image to produce a classification output that includes a respective metric corresponding to each of several categories. The measure for a category indicates a probability that the picture belongs to the category. In some cases, the categories can be classes of objects (e.g., dog, cat, person and the like) and the image can belong to a category if it represents an object included in the object class corresponding to the category. In some cases, the categories can include global image properties (such as whether the image represents a daytime or nighttime scene, or whether the image represents a summer or winter scene) and the image can belong to the category if it is global Has property according to the category.
Als ein weiteres bestimmtes Beispiel kann das Maschinenlernmodell konfiguriert sein, um ein Bild zu verarbeiten, um eine Klassifikationsausgabe auf Pixelebene zu erzeugen, die für jedes Pixel eine jeweilige Kennzahl entsprechend jeder von mehreren Kategorien enthält. Für ein gegebenes Pixel zeigt die Kennzahl für eine Kategorie eine Wahrscheinlichkeit an, dass ein Pixel zu der Kategorie gehört. In einigen Fällen können die Kategorien Klassen von Objekten sein und kann ein Pixel zu einer Kategorie gehören, wenn es Teil auf einem Objekt ist, das in der Objektklasse entsprechend der Kategorie enthalten ist. Das bedeutet, dass die Klassifikationsausgabe auf Pixelebene eine Ausgabe einer semantischen Segmentierung sein kann.As another specific example, the machine learning model can be configured to process an image to produce a pixel-level classification output that includes, for each pixel, a respective index corresponding to each of a plurality of categories. For a given pixel, the index for a category indicates a probability that a pixel belongs to the category.
