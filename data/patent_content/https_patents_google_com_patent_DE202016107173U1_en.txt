HINTERGRUND BACKGROUND
Die Kosten des Datenzuganges auf Mobiltelefonen können für die meisten Benutzer in der Welt unerschwinglich sein. Benutzer haben keinen Zugang zu bequemen Kontrollen, um die Nutzung von Daten durch auf ihrem Telefon installierte Anwendungen zu begrenzen. Häufig gehen Benutzer dazu über, ihren Mobil- und Wifi-Funk von Hand abzuschalten oder den Flugzeugmodus zu wählen. Jedoch beschränken diese Optionen den Datenzugang für alle Anwendungen ein, die auf ihrem Mobiltelefon installiert sind, und deaktivieren Telefoniefunktionen. The cost of data access on mobile phones can be prohibitive for most users in the world. Users have no access to convenient controls to limit the use of data by applications installed on their phone. Frequently, users are turning off their mobile and wifi radio by hand or choosing airplane mode. However, these options restrict data access for all applications installed on their mobile phones and disable telephony features.

Implementierungen der Offenbarung können eine Datenmanagementanwendung bereitstellen, die den Zugang zu einem Datennetz durch bestimmte Anwendungen auf einem mobilen Gerät einschränken kann, und zwar auf der Grundlage bestimmter Bedingungen. So kann beispielsweise die Datenmanagementanwendung den Zugang durch andere Anwendungen auf einem mobilen Gerät zu einem Netzgerätetreiber kontrollieren. Der Netzgerätetreiber kann mit einer Netzkommunikationskomponente, wie Wifi oder Mobilfunk, zusammenarbeiten, die Zugang zu dem Datennetz bieten. Ein Benutzer kann die Datenmanagementanwendung so konfigurieren, dass sie den Zugang einer bestimmten Anwendung zu dem Netz auf der Grundlage bestimmter Bedingungen einschränkt. So kann beispielsweise ein Benutzer eine Konfiguration wählen, die den Zugang einer Anwendung (z. B. einer zweiten Anwendung) zum Netz verhindert, wenn sie im Hintergrund durchgeführt wird und das mobile Gerät mehr als 90% der Gesamtdaten verwendet hat, die ihm zur Verfügung stehen. Wenn die Anwendung eine Anfrage stellt, Daten zu übertragen, kann die Datenmanagementanwendung die Anfrage empfangen und feststellen, dass die anfragende Anwendung im Hintergrund durchgeführt und der Datenverbrauch sich über der 90%-Schwelle befindet. Infolgedessen kann die Datenmanagementanwendung verhindern, dass die Anfrage der anfragenden Anwendung die Netzkommunikationskomponente erreicht. Implementations of the disclosure may provide a data management application that may restrict access to a data network by certain applications on a mobile device, based on certain conditions. For example, the data management application can control access by other applications on a mobile device to a network device driver. The network device driver may cooperate with a network communication component, such as Wifi or cellular, that provide access to the data network. A user may configure the data management application to restrict the access of a particular application to the network based on certain conditions. For example, a user may choose a configuration that prevents an application (eg, a second application) from accessing the network when it is running in the background and the mobile device has used more than 90% of the total data available to it stand. When the application makes a request to transfer data, the data management application can receive the request and determine that the requesting application is in the background performed and the data consumption is above the 90% threshold. As a result, the data management application can prevent the requesting application from reaching the network communication component.
