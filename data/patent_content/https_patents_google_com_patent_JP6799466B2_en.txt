Description
自動車、二輪車、航空機および船舶などの乗物は、乗物の乗員を支援するまたは通知するまたは楽しませる、さもなけらば乗物の乗員と情報交換する１つ以上のコンピューティング装置を含むことができる。例えば、自動車は、ハードウェアおよびソフトウェアの組み合わせ、例えばタッチスクリーンディスプレイおよび１つ以上のアプリケーションを含む車両ヘッドユニットを含むことができる。 Vehicles such as automobiles, motorcycles, aircraft and ships can include one or more computing devices that assist, notify or entertain the occupants of the vehicle, or otherwise exchange information with the occupants of the vehicle. For example, an automobile can include a combination of hardware and software, such as a vehicle head unit that includes a touch screen display and one or more applications.
車両ヘッドユニットは、ユーザと車両ヘッドユニットとの情報交換を可能にする１つ以上のユーザインターフェイスを提供することができる。一例として、車両ヘッドユニットは、ナビゲーションアプリケーション用のユーザインターフェイスを作り出すことができる。ユーザは、このユーザインターフェイスを介して、目的地を指定し、目的地までのナビゲーション命令を発することができる。一部の車両ヘッドユニットは、個人化設定を改善するために、特定のユーザが車両ヘッドユニットにログインすることを可能にすることができる。 The vehicle head unit can provide one or more user interfaces that allow information exchange between the user and the vehicle head unit. As an example, the vehicle head unit can create a user interface for navigation applications. The user can specify a destination and issue a navigation command to the destination through this user interface. Some vehicle head units can allow certain users to log in to the vehicle head unit in order to improve their personalization settings.
しかしながら、一部の車両は、複数の乗員を載せることがある。この場合、複数の乗員が車両ヘッドユニットへのアクセスを共有することができるが、特定のユーザは、車両ヘッドユニットにログインしなければ、個人化設定の恩恵を受けることができない。 However, some vehicles may carry multiple occupants. In this case, multiple occupants may share access to the vehicle head unit, but certain users will not be able to benefit from the personalization settings without logging in to the vehicle head unit.
概要
本開示は、車両ヘッドユニットで複数のユーザを認証し、共有セッション中に車両ヘッドユニットによって生成されたデータを複数のユーザアカウントに記憶するための技術に関する。例えば、車両ヘッドユニットは、複数の乗員、例えば運転者および乗客を載せる車両の共有スクリーン環境に設けられてもよい。運転者および乗客の両方は、車両ヘッドユニットに直接アクセスして、共有スクリーン環境にユーザ入力を与えることができる。本開示の技術に従って、まず、車両ヘッドユニットで運転者および乗客をそれぞれ認証することができる。車両ヘッドユニットは、運転者および乗客の両方のユーザ識別子情報を含む共有セッションを形成することができる。共有セッション中に、車両ヘッドユニットは、一部の例として、位置履歴データ、再生リストデータおよび検索履歴データなどのデータを生成することができる。車両ヘッドユニットは、生成されたデータを単一ユーザのアカウントに書き込むのではなく、運転者のユーザアカウントおよび乗客のユーザアカウントの両方にデータを書き込むこともできる。
Summary The present disclosure relates to a technique for authenticating multiple users with a vehicle head unit and storing data generated by the vehicle head unit in multiple user accounts during a shared session. For example, the vehicle head unit may be provided in a shared screen environment of a vehicle carrying multiple occupants, such as a driver and passengers. Both the driver and the passenger can directly access the vehicle head unit to provide user input to the shared screen environment. According to the technique of the present disclosure, the vehicle head unit can first authenticate the driver and the passenger, respectively. The vehicle head unit can form a shared session that includes user identifier information for both the driver and passengers. During the shared session, the vehicle head unit can generate data such as position history data, playlist data and search history data, as in some cases. The vehicle head unit can also write data to both the driver's user account and the passenger's user account instead of writing the generated data to a single user account.

Thus, data generated during a shared session that may be useful for contextually relevant information, services and / or personalization settings is not discarded or stored in a single account and is not stored in the vehicle head. The unit stores it in the account corresponding to each authenticated user.
