Abstract
사용자 데이터에 대한 액세스 권한 부여를 위한 시스템 및 방법에는 하나 이상의 블록체인을 사용하여 사용자 데이터 및/또는 권한 부여 목록을 저장하는 것이 포함될 수 있다. 시스템 및 방법은 액세스 요청을 수신 및/또는 획득할 수 있다. 요청자에게 권한이 있는지 확인하기 위해 액세스 요청을 프로세싱할 수 있다. 프로세싱은 사용자 컴퓨팅 시스템 및/또는 인증 목록과의 상호 작용을 포함할 수 있다. 요청자가 제한된 복사 및/또는 덮어쓰기 액세스로 블록체인에 저장된 사용자 데이터를 읽을 수 있도록 허가된 액세스가 제공될 수 있다.Systems and methods for granting access to user data may include using one or more blockchains to store user data and/or authorization lists. Systems and methods may receive and/or obtain access requests. Access requests may be processed to determine if the requester is authorized. Processing may include interaction with a user's computing system and/or authentication list. Privileged access may be provided to allow requesters to read user data stored on the blockchain with limited copy and/or overwrite access.
Description
본 발명은 일반적으로 사용자 데이터 및/또는 사용자와 연관된 개인화 모델에 대한 액세스 권한 부여에 관한 것이다. 특히, 본 발명은 사용자가 웹 상에서 또는 로컬에서 상호작용하는 상이한 웹 플랫폼에 의해 액세스될 수 있는 사용자 정보에 대한 보안 데이터베이스를 제공하기 위해 블록체인의 데이터 보안을 활용하는 것에 관한 것이다.The present invention generally relates to granting access to user data and/or personalization models associated with a user. In particular, the present invention relates to leveraging the data security of blockchain to provide a secure database for user information that can be accessed by different web platforms with which users interact on the web or locally.
복수의 상이한 제3자와 연관된 복수의 상이한 웹사이트는 사용자로부터 사용자 데이터를 요청할 수 있다. 웹사이트에는 소셜 미디어 웹사이트, 쇼핑 웹사이트 및 스트리밍 웹사이트가 포함될 수 있다. 또한, 복수의 서로 다른 웹사이트는 복수의 사용자 각각에 대한 자신의 사용자 특정 데이터를 수집하고 집계할 수 있다. 웹사이트 특정 수집 및 집계는 사용자가 제어할 수 없는 복수의 서로 다른 데이터베이스에 개인 정보를 저장하도록 사용자를 유도할 수 있다.Multiple different websites associated with multiple different third parties may request user data from the user. Websites may include social media websites, shopping websites, and streaming websites. Additionally, multiple different websites may collect and aggregate their own user-specific data for each of multiple users. Website-specific collection and aggregation may lead users to store personal information in multiple different databases over which they have no control.
따라서 사용자는 동일하거나 유사한 데이터를 복수의 제3자에게 반복적으로 제공하고 있을 수 있다. 그런 다음 제3자는 사용자에게 알리지 않고 다양한 방식으로 데이터를 복제 및/또는 변경할 수 있다. 또한 사용자는 데이터 삭제에 대해 제한된 제어 권한을 가질 수 있다. 액세스 가능한 삭제 기능을 제공하는 제3자의 경우에도 사용자 데이터는 복수의 제3자에 의해 저장될 수 있으며, 이로 인해 삭제 태스크가 지루하고 경우에 따라 비실용적일 수 있다.Therefore, users may be repeatedly providing the same or similar data to multiple third parties. Third parties can then copy and/or alter your data in various ways without your knowledge. Additionally, users may have limited control over data deletion. Even with third parties providing accessible deletion functionality, user data may be stored by multiple third parties, making the deletion task tedious and in some cases impractical.

One example aspect of the disclosure relates to a computing system. The system may include one or more processors and one or more non-transitory computer-readable media that collectively store instructions that, when executed by the one or more processors, cause the computing system to perform operations. Operations may include receiving an access request from a third party computing system. An access request may describe a request to access user data for a specific user. In some implementations, user data may be stored on the blockchain. Operations may include determining a specific entity associated with a third-party computing system. Certain entities may be associated with the web platform. Operations may include sending a notification to a particular user computing system. A specific user computing system may be associated with a specific user. In some implementations, a notification may describe an access request and a specific entity. Operations may include receiving user input data from a particular user computing system. User input data may describe (describe) the user's response to the access request. Operations may include determining an access action based on user input data. An access action may describe the level of access that a third-party computing system will provide to user data on the blockchain.
