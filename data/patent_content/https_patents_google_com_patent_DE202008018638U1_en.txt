Abstract
System, das Folgendes umfasst: ein oder mehrere Computer, die zur Interaktion mit einem Client-Gerät konfiguriert werden und in der Lage sind: eine Zeichenfolge vom Client-Gerät zu empfangen; die Folge nach einem globalen Parameter abzusuchen; eine erste Projektkennung nach dem globalen Parameter in der Folge zu lesen; eine erste Gruppe von zwei oder mehreren getrennten Datenelementen nach der ersten Projektkennung in der Folge zu lesen, wobei die Datenelemente die entsprechenden Indexnummern besitzen; die Datenelemente gemeinsam mit der ersten Projektkennung und dem entsprechenden Indexwert zu speichern; eine zweite Projektkennung nach der ersten Projektkennung in der Folge zu lesen; eine zweite Gruppe von zwei oder mehreren getrennten Datenelementen nach der zweiten Projektkennung in der Folge zu lesen, wobei die Datenelemente die entsprechenden Indexnummern besitzen; und die Datenelemente gemeinsam mit der zweiten Projektkennung und dem entsprechenden Indexwert zu speichern.A system, comprising: one or more computers configured to interact with a client device and capable of: receiving a string from the client device; search the sequence for a global parameter; read a first project identifier for the global parameter in the sequence; read a first group of two or more separate data items after the first project identifier in the sequence, the data items having the corresponding index numbers; store the data elements together with the first project identifier and the corresponding index value; to read a second project identifier after the first project identifier in the sequence; read a second group of two or more separate data items after the second project identifier in the sequence, the data items having the corresponding index numbers; and store the data elements together with the second project identifier and the corresponding index value.

Diese Spezifikation bezieht sich auf Datenübertragungsformate.This specification relates to data transmission formats.
In einem Netzwerk verbundene Computer sind in der Lage, über eine Vielzahl von Protokollen miteinander zu kommunizieren. Jedes Protokoll unterstützt eine beliebige Anzahl von Kodierungsformaten. Das Hypertext Transfer Protocol (HTTP) beispielsweise dient dazu, Daten von einem Computer zum anderen zu übertragen. Um Daten erfolgreich von einem Senderechner zu einem Empfangsrechner zu übertragen, wird ein Kodierungsformat festgelegt, damit die vom Sender verschlüsselten Daten vom Empfänger entschlüsselt werden können.Network-connected computers are capable of communicating through a variety of protocols. Each protocol supports any number of encoding formats. For example, Hypertext Transfer Protocol (HTTP) is used to transfer data from one computer to another. To successfully transfer data from a sending computer to a receiving computer, a coding format is set so that the data encrypted by the sender can be decrypted by the recipient.
Um mehrere Datenelemente über ein Netzwerk zum Empfang an einen Rechner zu übertragen, müssen Empfangs- und Senderechner dasselbe Kodierungsformat verwenden, sodass übertragene Daten erfolgreich vom Empfänger analysiert und interpretiert werden können. Es können zwar beliebig viele Formate festgelegt werden, aber um die für die Übertragung einer bestimmten Datenmenge benötigte Bandbreite zu begrenzen, sollte der für ein Format erforderliche Overhead möglichst begrenzt werden. Zur Übertragung von Zeichendaten, die aus mehreren Elementen bestehen (z. B. mehrere ASCII-Variablen), werden in Übertragungsformaten normalerweise Trennzeichen zur Trennung variabler Elemente eingesetzt. Ein Analyseprogramm auf dem Empfangscomputer kann dann die Daten abhängig vom gewählten Kodierungsformat extrahieren und den entsprechenden Variablen zuordnen.In order to transmit several data elements over a network for reception to a computer, the receiving and transmitting computers must use the same coding format so that transmitted data can be successfully analyzed and interpreted by the receiver. While any number of formats can be specified, limiting the amount of bandwidth required to transmit a given amount of data should minimize the overhead required for a given format. To transfer character data that consists of multiple elements (for example, multiple ASCII variables), transfer separators usually use separators to separate variable elements. An analysis program on the receiving computer can then extract the data depending on the selected coding format and assign it to the corresponding variables.
Idealerweise ist ein Kodierungsformat flexibel genug, damit Änderungen wie das Hinzufügen neuer Variablen von der Kodierungsarchitektur ohne wesentliche (oder jegliche) Änderung des zugrundeliegenden Codes bewältigt werden kann. Die sorgfältige Auswahl der Trennzeichen und des Grundformats eines bestimmten Kodierungsschemas kann deshalb die Wirksamkeit und den Nutzen des Datenkodierungsformates verbessern.Ideally, a coding format is flexible enough to allow for changes such as adding new variables to the coding architecture without substantial (or any) modification of the underlying code. Careful selection of the delimiters and the basic format of a particular coding scheme can therefore improve the effectiveness and usefulness of the data encoding format.

Ein Datentyp der Datenelemente kann entweder implizit durch die Aufzählungsreihenfolge von Schlüsseln und Werten angezeigt werden und/oder explizit durch eine Datentypkennung direkt vor einem Satz von Klammern um eine Untergruppe. Indexnummern einer Untergruppe von Datenelementen können mit einem Anfangswert beginnen und pro zusätzlichem Datenelement um eins ansteigen, es sei denn, eine Indexnummer gefolgt von einem Ausrufezeichen zeigt explizit etwas anderes an.In certain implementations, the string is received as a URL suffix in an HTTP request. The data elements of the string may contain tracking data for web analysis and be separated by characters that do not require an escape sequence to be used in a URL. The data elements can be separated by an asterisk, for example. A data type of the data elements may be indicated either implicitly by the enumeration order of keys and values, and / or explicitly by a data type identifier immediately preceding a set of parentheses around a subgroup. Index numbers of a subset of data elements may begin with an initial value and increase by one per additional data element, unless an index number followed by an exclamation point explicitly indicates otherwise.
