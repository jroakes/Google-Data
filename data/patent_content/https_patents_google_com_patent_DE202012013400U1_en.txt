Abstract
Ein computerimplementiertes System zur Steuerung des Zeitablaufs von Aufgaben in Frames, die Ansichten eines geographischen Informationsumfelds repräsentieren und das eingerichtet ist, das im Folgenden Aufgeführte zu umfassen: das Bestimmen einer ersten restlichen Frameperiode in einem ersten Frame, die eine erste Ansicht des geographischen Informationsumfeldes repräsentiert, bestimmt, wobei die erste restliche Frameperiode einen Zeitraum nach dem Ausführen eines ersten Satzes an feststehenden Aufgaben im ersten Frame und an einer Framegrenze des ersten Frames, darstellt. das Identifizieren einer ersten dynamischen Aufgabe in einer Zeitablaufsteuerungswarteschlange, wobei die erste dynamische Aufgabe über einen ersten nicht-bevorrechtigten Abschnitt verfügt, der sich zwischen dem Beginn der ersten Aufgabe und einem ersten Bevorrechtigungspunkt der ersten Aufgabe befindet. das Bestimmen einer ersten geschätzten Ausführungszeit für das Ausführen der ersten Aufgabe; wenn die erste geschätzte Ausführungszeit nicht größer als die erste restliche Frameperiode ausfällt, das Ausführen des ersten nicht-bevorrechtigten Abschnitts der ersten Aufgabe im ersten Frame; und wenn die erste geschätzte Ausführungszeit größer als die erste restliche Frameperiode ausfällt, das Verschieben der Ausführung der ersten Aufgabe im ersten Frame.A computer-implemented system for controlling the timing of tasks in frames that represent views of a geographic information environment and that is arranged to include the following: determining a first remaining frame period in a first frame that represents a first view of the geographic information environment, wherein the first remaining frame period represents a time period after performing a first set of fixed tasks in the first frame and at a frame boundary of the first frame. identifying a first dynamic task in a scheduling queue, the first dynamic task having a first non-privileged portion located between the beginning of the first task and a first privileging point of the first task. determining a first estimated execution time for performing the first task; if the first estimated execution time does not exceed the first remaining frame period, executing the first non-privileged portion of the first task in the first frame; and if the first estimated execution time is greater than the first remaining frame period, suspending execution of the first task in the first frame.

Ein an einer Anzeigevorrichtung gerendertes Bild wird als Frame bezeichnet. Das Rendering kann in Frameperioden gleicher Größe unterteilt werden. Eine Frameperiode kann von Faktoren wie etwa dem Betreiben der Grafikhardware und der Anzeigevorrichtungen abhängig sein. Ein vertikales Synchronisierungsereignis findet statt, wenn die Anzeigevorrichtung Daten von einem Framepuffer abliest. Dieses Ablesen ist auf die Aktualisierungsrate der Anzeigevorrichtung abgestimmt. Falls die Aktualisierungsrate einer Anzeigevorrichtung beispielsweise auf 60 Hz eingestellt ist, verfügen die Frameperioden für die Frame über eine Länge von 16,67 ms.An image rendered on a display device is called a frame. The rendering can be divided into frame periods of the same size. A frame period may depend on factors such as operating the graphics hardware and the display devices. A vertical synchronization event occurs when the display device reads data from a frame buffer. This reading is tuned to the refresh rate of the display device. For example, if the refresh rate of a display device is set to 60 Hz, the frame periods for the frames are 16.67 ms long.
Es ergibt sich keinen Vorteil aus einem Rendering, das schneller als die Aktualisierungsrate ist. Ein Rendering, das langsamer als die Aktualisierungsrate ist, kann jedoch durch das erneute Zeichnen des vorhergehenden Frames Artefakte einführen, und somit ein merkliches Framestottern erzeugen. Zeitablaufsteuerungsmechanismen in Echtzeit können verwendet werden, um das Framestottern zu reduzieren.There is no benefit of rendering faster than the refresh rate. However, a rendering that is slower than the refresh rate can introduce artifacts by redrawing the previous frame, creating noticeable frame stuttering. Timing control mechanisms in real time can be used to reduce frame stuttering.

Eine Aufgabe kann mit einem Task in Verbindung gebracht werden. Ein Task kann über einen Termin verfügen. Ein weicher Termin ist ein Termin, der verpasst werden kann. Sofern jedoch verpasst wird, sollte die Verspätung so gering wie möglich ausfallen. Das zu Ende führen eine Aufgabe nach Ablauf des weichen Termins ist ebenfalls von Bedeutung. Ein feststehender Termin ist ein Termin, nach dem eine Aufgabe nicht zu Ende geführt werden sollte, sondern stattdessen entfernt werden sollte. Das verpassen eines feststehenden Termins ist kostspielig aber nicht katastrophal.A task can be associated with a task. A task can have an appointment. Die Ausführungszeit kann auf vielfältige Art und Weise prognostiziert werden. Die geschätzte Ausführungszeit einer Aufgabe kann beispielsweise zumindest zum Teil auf vergangenes Verhalten der Aufgabe, einem Taskidentifikator der Aufgabe, einer nichtbevorrechtigten Abschnittsart der Aufgabe, einer Ausführungszeit eines nichtbevorrechtigten Abschnitts der Aufgabe, oder einer Abschnittslänge des nicht-bevorrechtigten Abschnitts (der nicht-bevorrechtigten Abschnitte) des Ersteren (z. B. längster nicht-bevorrechtigter Abschnitt einer Aufgabe) basieren. Die geschätzte Ausführungszeit kann während der Laufzeit ermittelt werden.The execution time can be predicted in many ways. The estimated execution time of a task may include, for example, at least in part a past behavior of the task, a task identifier of the task, an unprivileged portion of the task, an execution time of an unpaid portion of the task, or a portion length of the non-privileged portion (non-privileged portions). the former (eg longest non-privileged portion of a task). The estimated execution time can be determined during runtime.
Eine Zeitablaufsteuerungseinrichtung kann den Empfang einer geschätzten Ausführungszeit für das Ausführen einer Aufgabe von der Aufgabe unterstützen. Wenn die Aufgabe eine geschätzte Ausführungszeit für das Ausführen der Aufgabe bereitstellt, kann die Zeitablaufsteuerungseinrichtung das Ermitteln einer geschätzten Ausführungszeit für das Ausführen der Aufgabe fortsetzen. Die Zeitablaufsteuerungseinrichtung kann dann die zwei geschätzten Ausführungszeiten miteinander vergleichen und das pessimistischste Ergebnis auswählen. Ausgehend von diesem Ergebnis kann die Zeitablaufsteuerungseinrichtung ermitteln, ob die Aufgabe in einem Frame auszuführen ist oder ob das Ausführen der Aufgabe in einem Frame verschoben werden sollte.A scheduler may assist in receiving an estimated execution time for executing a task from the task. If the task provides an estimated execution time to perform the task, the scheduler may continue to determine an estimated execution time to complete the task. The scheduler may then compare the two estimated execution times together and select the most pessimistic outcome. Based on this result, the scheduler may determine whether the task is to be performed on a frame or whether the task should be moved in a frame.
In einer Ausführungsform kann eine geschätzte Ausführungszeit für das Ausführen einer Aufgabe zumindest teilweise auf einem Punkt während der Aufgabe basieren, ab dem das Ausführen beginnt, oder fortgesetzt wird. Ein unterschiedlicher Codepfad läuft beispielsweise zu Beginn einer Aufgabe, wenn nach einem Anruf zu ShouldContinue () ein nicht-bevorrechtigter Abschnitt ausgeführt wird. Dementsprechend können die Zeitplanungsstatistiken für diese beiden Fälle separat gehandhabt werden. Des Weiteren können die Codepfade zwischen ShouldContinue () Anrufen sich voneinander unterscheiden. In einer anderen Ausführungsform wird die schlechteste nicht-bevorrechtigte Abschnittszeit, die für jede Aufgabe nach einem ShouldContinue()-Anruf gemessen wird, als Messgröße für die nichtbevorrechtigten Abschnitte, die einem ShouldContinue () Anruf folgen, verwendet.In one embodiment, an estimated execution time for performing a task may be based, at least in part, on a point during the task from which execution begins or continues. For example, a different code path runs at the beginning of a task when a non-privileged portion is executed after a call to ShouldContinue (). Accordingly, the scheduling statistics for these two cases can be handled separately. Furthermore, the code paths between ShouldContinue () calls may be different. In another embodiment, the worst non-privileged section time measured for each task after a ShouldContinue () call is used as a measure of the non-privileged sections that follow a ShouldContinue () call.
In einer Ausführungsform wird ein Prädiktor für jeden Satz von Aufgaben, die einen einzelnen Taskidentifikator teilen, erstellt. Sämtliche Prädiktoren gleichen einander. Bei sporadischen Tasks entspricht jeder Taskidentifikator einem tatsächlichen Task. Mehrere aperiodische Tasks, die denselben Codepfad ausführen, können mit demselben Taskidentifikator ausgestattet werden, um einen nützlichen Verhaltensverlauf zu erfassen.In one embodiment, a predictor is created for each set of tasks sharing a single task identifier. All predictors are similar. For sporadic tasks, each task identifier corresponds to an actual task. Multiple aperiodic tasks running the same code path can be equipped with the same task identifier to detect a useful behavior history.
In einer Ausführungsform wird eine geschätzte Ausführungszeit von Null für jeden nichtbevorrechtigten Abschnitt einer Aufgabe geschätzt. Wenn ein Termin als solcher tatsächlich eingehalten worden ist, oder wenn eine Aufgabe eine geschätzte Ausführungszeit für die Aufgabe bereitstellt, kann einer Ausführung auf Basis der Zeitplanung verhindert werden. Ferner kann die Zeitablaufsteuerungseinrichtung die Ausführung der Aufgabe stoppen, sobald die Zeitablaufsteuerungseinrichtung prognostiziert, dass ein nicht-bevorrechtigter Abschnitt einer Aufgabe nicht vor der Framegrenze beendet werden wird. Diese Messungen können als Baseline verwendet werden.In one embodiment, an estimated execution time of zero is estimated for each non-privileged portion of a task. If an appointment as such has actually been met, or if a task provides an estimated execution time for the task, scheduling based execution may be prevented. Further, the scheduler may stop execution of the task as soon as the scheduler predicts that a non-privileged portion of a task will not be completed before the frame limit. These measurements can be used as baseline.
In einer Ausführungsform wird eine maximal beobachtete Ausführungszeit für jede Art von nicht-bevorrechtigtem Abschnitt als die geschätzte Ausführungszeit für die Aufgabe verwendet. Dies kann für die im schlimmsten Fall durchgeführte Ausführungszeit Abschätzung verwendet werden und kann empfindlich für Ausreißer sein. Falls das Betriebssystem die Entscheidung trifft, die geographische Anwendung während des Ausführens einer Aufgabe zu bevorrechtigten, könnte die gemessene nicht-bevorrechtigte Abschnittslänge sehr groß ausfallen. Die pessimistische Annahme, dass alle Aufgaben desselben Tasks eine derartig lange Ausführungszeit erfordern, könnte starke Verzögerungen und eine als lang wahrgenommene Antwortzeit für derartige Aufgaben zur Folge haben. Um diesen Effekt einzuschränken, können gemessene Werte ablaufen. Die schlimmste beobachtete Antwortzeit über die letzten 6 Sekunden kann beispielsweise seit dem Beginn der geographischen Anwendung verwendet werden.In one embodiment, a maximum observed execution time for each type of non-privileged portion is used as the estimated execution time for the task. This can be used for the worst-case execution estimation and can be sensitive to outliers. If the operating system decides to prioritize the geographic application while performing a task, the measured non-privileged section length could be very large. The pessimistic assumption that all tasks of the same task require such a long execution time could result in high delays and a perceived as long response time for such tasks. To limit this effect, measured values can expire. For example, the worst observed response time over the last 6 seconds may be used since the start of geographic application.
In einer Ausführungsform wird ein Programm vergangener nicht-bevorrechtigter Abschnittszeiten gespeichert. Dies kann die Wahrscheinlichkeit der falschen Prognose eines Wertes steuern. Wenn zum Beispiel ein 95-prozentiges Histogramm verwendet wird, werden die am weitesten links gelegenen 95% der Werte ausgewählt, und der Höchstwert im entsprechenden Bin-Bereich, kann als Ausfertigungszeitschätzer verwendet werden. Das Aufrechterhalten des Hilfsprogramms kann eine robuste Behandlung der Ausreißer zur Folge haben.In one embodiment, a program of past non-privileged section times is stored. This can control the probability of mispricing a value. In one embodiment, a bin size of 10 -3 milliseconds is used and it may be configurable percentage of the histogram can be selected. For example, if a 95% histogram is used, the leftmost 95% of the values are selected, and the maximum in the corresponding bin range can be used as the production time estimator. Maintaining the utility may result in robust treatment of the outliers.
In einer Ausführungsform werden die mittleren und standardmäßigen Abweichungswerte der nicht-bevorrechtigten Abschnittslängen einer Aufgabe gemessen. Dies kann für jede Art von nichtbevorrechtigten Abschnitt durchgeführt werden. (z. B. anfänglicher nicht-bevorrechtigter Abschnitt und nicht-bevorrechtigte Abschnitte an post-bevorrechtigten Punkten). Diese Werte können selbst dann online gemessen werden, wenn die tatsächlichen Mittelwerte und Varianzwerte unbekannt sind. Das ausschließliche Verwenden des Mittelwertes der nichtbevorrechtigten Abschnittslänge einer Aufgabe, um die Ausführungszeit zu bestimmen, kann die Hälfte der Zeit eine Untervorhersage der Ausführungszeit zur Folge haben. Durch das Verwenden des Mittelwertes plus eine gewisse Anzahl (z. B. konfigurierbare Zahl) an Standardabweichungen kann eine geschätzte Ausführungszeit mit einem ähnlichen Pessimismus für das Ausführungshistogramm ermittelt werden. Dies könnte den Aufwand verringern.In one embodiment, the mean and standard deviation values of the non-privileged section lengths of a task are measured. This can be done for any type of non-privileged section. (eg initial non-privileged section and non-privileged sections at post-privileged points). These values can be measured online even if the actual mean and variance values are unknown. Using only the average of the non-privileged section length of a task to determine the execution time may result in half the time under-predicting the execution time. By using the mean value plus a certain number (eg, configurable number) of standard deviations, an estimated execution time can be determined with a similar pessimism for the execution histogram. This could reduce the effort.
