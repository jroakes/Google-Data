Eine Anwendervorrichtung, die die nachstehend beschriebenen Systeme und Verfahren implementiert, ermöglicht das Aufnehmen eines Bildschirmfotos einer aktiven nativen Anwendung und eines einheitlichen Ressourcenbezeichners für die spezielle Umgebungsinstanz der nativen Anwendung, die in dem Bildschirmfoto aufgenommen ist. Diese Aufnahme dieser Informationen und das Teilen werden durch Verwenden eines intuitiven, einfach anzuwendenden Kontextinteraktionsmodells erreicht.A user device implementing the systems and methods described below enables capturing a screen shot of an active native application and a uniform resource identifier for the native application specific environment instance captured in the screenshot. This recording of this information and sharing is done by using an intuitive, easy-to-use context interaction model achieved.
Wie sie hier verwendet ist, erzeugt eine native Anwendung Umgebungsinstanzen zur Anzeige auf einer Anwendervorrichtung innerhalb einer Umgebung der nativen Anwendung und arbeitet unabhängig von einer Browser-Anwendung auf der Anwendervorrichtung. Eine native Anwendung ist eine Anwendung, die spezifisch konstruiert ist, um auf einem/einer speziellen Betriebssystem und Maschinen-Firmware einer Anwendervorrichtung abzulaufen. Native Anwendungen unterscheiden sich somit von browserbasierten Anwendungen und durch Browser wiedergegebenen Anwendungen. Letztere erfordern jedes Mal, wenn sie instanziiert oder wiedergegeben werden, alle oder wenigstens einige Elemente oder Anweisungen, die von einem Webserver heruntergeladen werden. Darüber hinaus können browserbasierten Anwendungen und durch Browser wiedergegebene Ressourcen durch alle webfähigen mobilen Vorrichtungen innerhalb des Browsers verarbeitet werden und sind deshalb nicht betriebssystemspezifisch. Jede spezielle Umgebungsinstanz der nativen Anwendung ist durch eine ”tiefe Verknüpfung” spezifiziert, die ansonsten als ein einheitlicher Ressourcenbezeichner (URI) bezeichnet sein kann. Der URI kann gemäß einem speziellen Standard konstruiert sein oder kann von einer Struktur sein, die für die spezielle native Anwendung eindeutig ist. Der URI bewirkt, dass die native Anwendung eine Instanz der spezifizierten nativen Anwendung instanziiert.As used herein, a native application creates environment instances for display on a user device within a native application environment and operates on the user device independently of a browser application. A native application is an application that is specifically designed to run on a particular operating system and machine firmware of a user device. Native applications are thus different from browser-based applications and browser-rendered applications. The latter, whenever instantiated or rendered, require all or at least some elements or instructions downloaded from a web server. Moreover, browser-based applications and browser-rendered resources can be processed by all web-enabled mobile devices within the browser and are therefore not OS-specific. Each native application specific environment instance is specified by a "deep link", which may otherwise be referred to as a Uniform Resource Identifier (URI). The URI may be constructed according to a particular standard or may be of a structure unique to the particular native application. The URI causes the native application to instantiate an instance of the specified native application.
Durch Verwenden eines einzigen eingegebenen Betriebssystembefehls kann ein Anwender Kontextinformationen empfangen, die sich auf eine aktive Umgebungsinstanz einer nativen Anwendung beziehen. Der Anwender kann dann einen Teilen-Befehl auswählen, der eine Nachricht erzeugt, die ein Bildschirmfoto der aktiven Umgebungsinstanz einer nativen Anwendung und eine ”tiefe Verknüpfung”, z. B. einen einheitlichen Ressourcenbezeichner, der aktiven Umgebungsinstanz enthält. Dieser Anwender kann dann das Bildschirmfoto und die tiefe Verknüpfung in einer einzigen Nachricht zu einem oder mehreren Anwendern senden. Nach dem Empfang kann jeder empfangende Anwender das Bildschirmfoto betrachten. Sollte der betrachtende Anwender wünschen, dieselbe native Anwendung auf der Empfangsvorrichtung zu instanziieren, kann der Anwender einen Befehl auswählen, z. B. durch Interagieren mit dem Bild auf eine spezielle Weise, wie z. B. durch ”langes Drücken”, um die native Anwendung auf der Empfangsvorrichtung zu instanziieren. Die native Anwendung kann dann die URI verarbeiten, um die gleiche (oder eine ähnliche) Umgebungsinstanz zu erzeugen.By using a single input operating system command, a user may receive context information related to an active environment instance of a native application. The user may then select a Share command that generates a message containing a screenshot of the native active instance of the environment and a "deep link", e.g. For example, a common resource identifier that contains the active environment instance. This user can then send the screenshot and the deep link in a single message to one or more users. Upon receipt, each receiving user can view the screenshot. Should the viewing user wish to instantiate the same native application on the receiving device, the user may select a command, e.g. B. by interacting with the image in a special way, such. By "long press" to instantiate the native application on the receiving device. The native application can then process the URI to create the same (or a similar) environment instance.
