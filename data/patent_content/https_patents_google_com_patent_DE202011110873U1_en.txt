Abstract
System zur Behandlung von Benutzerabfragen und zur Durchführung von Datenverwaltung, umfassend: Mittel zum Speichern von mit Benutzerabfragen verbundenen Daten; und ein logisch mit dem Speichermittel gekoppeltes Prozessorgerät, das betrieben werden kann zum: Empfangen von Datensätzen von einer Vielzahl von Benutzergeräten; Speichern der einzelnen Datensätze als mit einem jeweiligen Benutzer verbundene Tabelle; Generieren einer zusammengesetzten Tabelle aus einer Gruppe von Tabellen, die mit jeweiligen Benutzern verbunden sind, wobei die zusammengesetzte Tabelle alle Zeilen aus jeder Tabelle der Gruppe umfasst und jeder Eintrag der zusammengesetzten Tabelle eine Zeile einer gegebenen einen aus der Gruppe von Tabellen darstellt; Empfangen einer Vielzahl von Benutzerabfragen von einem oder mehreren der Benutzergeräte; Beantworten einer oder mehrerer aus der Vielzahl von Benutzerabfragen durch Erzeugen einer Visualisierung von ausgewählten Daten für die Anzeige auf einem oder mehreren der Benutzergeräte, wobei die Visualisierung anhand von in der zusammengesetzten Tabelle vorgefundenen Datentypen und den für die einzelnen Visualisierungen erforderlichen Datentypen bestimmt wird; und Einfügen von geographischen Merkmalen der einzelnen Datensätze in einen räumlichen Index.A system for handling user queries and performing data management, comprising: means for storing data associated with user queries; and a processor device logically coupled to the memory means and operable to: receive data records from a plurality of user devices; Storing the individual records as a table associated with a respective user; Generating a composite table from a group of tables associated with respective users, the composite table comprising all rows from each table of the group and each composite table entry representing a row of a given one of the group of tables; Receiving a plurality of user queries from one or more of the user devices; Answering one or more of the plurality of user queries by generating a visualization of selected data for display on one or more of the user devices, wherein the visualization is determined from data types found in the composite table and the types of data required for each visualization; and inserting geographic features of the individual records into a spatial index.

If so, the process fails. If no, log entries are written for each mutation and the task is marked as added. The application stage follows. Dann wird die Menge von Zellen, die die vereinfachte Linie bedecken, berechnet, wobei sich die Zellen auf einer beliebigen Ebene befinden können. Hier berechnet der Prozess die Anfangs- und Endzelle (auf der untersten Ebene) für Zelle q. Alle Merkmale im Bereich Index[Start-Ende] (zwischen der Anfangs- und Endzelle) werden dann abgerufen. Hieraus erhält man die Vereinigungsmenge zwischen den Merkmalen im Bereichsindex und den Merkmalen in Zellen, die Vorgänger von q sind.Below are several operations that the index can support. An operation is "insert point P". Here, a cell is equal to a S2CellID of point P (lowest level cell). This operation adds the point to the index [cell]. The next one is "Insert Line / Polygon". Here, the process simplifies the line by sampling to a maximum of 100 points, although other maximums can be used. Then, the amount of cells covering the simplified line is calculated, and the cells can be at any level. For every cell in the cover a line is added to the index [cell]. Another operation is "query for characteristics in cell q". Here, the process computes the start and end cells (at the lowest level) for cell q. All features in the Index [start-end] area (between the start and end cells) are then retrieved. Preferably, all features are obtained in cells that are precursors of q. In one example, the maximum number of predecessor searches is set to 30. There is a hierarchy of cells here, whereby a cell at the very bottom of the hierarchy can have a maximum of 30 predecessors. When a request arrives, the cache is first examined. If the answer is there, it will be returned. If not, the structured data store is asked for a response, which is then added to the cache. When the table is changed, all its cache entries are invalidated.
