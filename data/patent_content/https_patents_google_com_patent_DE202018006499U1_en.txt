HINTERGRUNDBACKGROUND
Einige Rechensysteme stellen elektronische Nachrichtenübertragungsdienste (z. B. E-Mail-Dienste) bereit, die eine schnelle und einfache Kommunikation zwischen Anwendern von Rechenvorrichtung ermöglichen. Ein Entwerfen und ein auf andere Weise Bearbeiten elektronischer Nachrichten kann umständlich sein und viel Zeit benötigen, was in einer verringerten Verwendung elektronischer Nachrichten und/oder einer ineffizienten Kommunikation von Informationen resultieren kann.Some computing systems provide electronic messaging services (e.g., e-mail services) that enable quick and easy communication between users of computing devices. Designing and otherwise manipulating electronic messages can be cumbersome and time consuming, which can result in reduced use of electronic messages and / or inefficient communication of information.

ZUSAMMENFASSUNGSUMMARY
Im Allgemeinen sind Techniken dieser Offenbarung auf ein Ermöglichen einer wahlweisen Textvorhersage zum Bearbeiten elektronischer Nachrichten gerichtet. Es wird ein Beispielrechensystem (wie z. B. ein E-Mail-System) beschrieben, das konfiguriert ist, einen vorgeschlagenen Text für einen Anwender, der eine elektronische Nachricht bearbeitet, bereitzustellen. Zum Beispiel kann das Rechensystem, nachdem der Anwender etwas Text eines Satzes (z. B. „Was machst Du“) getippt hat, Kandidatentext, der auf den durch den Anwender getippten Text folgen soll (z. B. „heute Abend.“), vorhersagen und anbieten. Das Rechensystem kann den Kandidatentext auf der Grundlage eines Textkorpus vorhersagen (z. B. kann das Rechensystem ein oder mehrere Modelle für maschinelles Lernen verwenden, die auf dem Textkorpus trainiert wurden). Während die Vorhersagen im Allgemeinen hilfreich und genau sein können (z. B. ist der vorhergesagte Text nahe daran, was andernfalls der Anwender getippt hätte), können einige Szenarios vorliegen, in denen es nicht wünschenswert ist, dass das Rechensystem die Vorhersagen bereitstellt. Zum Beispiel dann, wenn die elektronische Nachricht auf ein sensibles Thema gerichtet ist, kann es wünschenswert sein, dass das Rechensystem davon absieht, dem Anwender vorhergesagten Text bereitzustellen, um zu vermeiden, Text vorzuschlagen, den der Anwender als beleidigend oder unangebracht betrachten kann.In general, techniques of this disclosure are directed to enabling optional text prediction for handling electronic messages. An example computing system (such as an e-mail system) configured to provide suggested text to a user handling an electronic message is described. For example, after the user has typed some text in a sentence (e.g. "What are you doing"), the computation system can generate candidate text that is to follow the text typed by the user (e.g. "Tonight.") , predict and offer. The computing system can predict the candidate text based on a corpus of text (e.g., the computing system can use one or more machine learning models trained on the corpus of text). While the predictions can generally be helpful and accurate (e.g., the predicted text is close to what the user would otherwise have typed), there may be some scenarios where it is not desirable for the computing system to provide the predictions. For example, if the electronic message is directed to a sensitive topic, it may be desirable for the computing system to refrain from providing predicted text to the user in order to avoid suggesting text that the user may find offensive or inappropriate.
