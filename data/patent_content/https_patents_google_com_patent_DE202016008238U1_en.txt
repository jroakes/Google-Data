Abstract
System, umfassend einen oder mehrere Prozessoren und einen Speicher, der Anweisungen speichert, die den einen oder die mehreren Prozessoren dazu bringen: eine Benutzerangabe zu identifizieren, die an einem Computergerät empfangen wird; auf Grundlage der Benutzerangabe eine Vielzahl von in Frage kommenden Antwortaktionen zu identifizieren, die durch das Computergerät initiiert werden können, um die Benutzerangabe potenziell zu erfüllen; auf Grundlage der Benutzerangabe eine einzelne in Frage kommende Antwortaktion der Vielzahl von in Frage kommenden Antwortaktionen zur exklusiven Initiierung auf dem Computergerät als Reaktion auf die Benutzerangabe nichtdeterministisch auszuwählen; und die einzelne in Frage kommende Antwortaktion auf dem Computergerät exklusiv zu initiieren.A system comprising one or more processors and a memory storing instructions that cause the one or more processors: to identify a user indication received at a computing device; identify, based on the user indication, a plurality of candidate response actions that may be initiated by the computing device to potentially satisfy the user indication; based on the user indication, non-deterministically selecting a single candidate response action of the plurality of eligible response actions for exclusive initiation on the computing device in response to the user indication; and to initiate the single eligible response action on the computing device exclusively.

Hintergrundbackground
Persönliche Assistenzmodule sind Softwareprozesse, die auf Computergeräten, wie etwa Smartphones, Tablets, Fahrzeug-Navigationssystemen und Smart Watches installiert sind, um benutzerfreundliche Oberflächen für die Vornahme von natürlichen Sprachbefehlen durch Benutzer zu bieten. Benutzer nehmen oft natürliche Sprachbefehle vor, wobei sie ihre Stimme benutzen, aber persönliche Assistenzmodule können auch auf andere Modalitäten reagieren, wie etwa auf verschiedene Formen von Texteingaben. Existierenden persönlichen Assistenzmodulen kann es an Intelligenz mangeln, um auf die vorangegangenen Angaben und/oder Vorlieben des Benutzers zurückzugreifen. Dementsprechend kann es sein, dass das persönliche Assistenzmodul, wenn ein Benutzer eine mehrdeutige Angabe macht (z. B. Anfrage, Aussage, Befehl), auf die potenziell unterschiedlich reagiert werden kann, zusätzliche Information vom Benutzer anfordern muss um zu bestimmen, wie es vorgehen soll. Nehmen wir an, ein Benutzer spricht die Wörter „Bob anrufen” in sein Telefon, aber es gibt mehrere Kontakte in der Kontaktliste des Benutzers, die mit dem Namen „Bob” übereinstimmen. Ein auf dem Telefon des Benutzers laufendes persönliches Assistenzmodul kann fragen, welchen Bob der Benutzer gemeint hat und/oder ob es Bobs Telefon zu hause oder auf der Arbeit anrufen soll. Oder nehmen wir an, ein anderer Benutzer spricht die Wörter „erinnere mich daran, morgen einkaufen zu gehen” in seine Smart Watch. Ein auf der Smart Watch des Benutzers – oder auf einem mit der Smart Watch kommunizierenden Smartphone – laufendes persönliches Assistenzmodul kann fragen, welche Anwendung (z. B. Kalender, Aufgabenplaner, Anwendung eines sozialen Netzwerks) der Benutzer nutzen will, um die Erinnerung anzulegen. Die Anforderung solcher zusätzlichen Informationen kann aus einer Reihe von Gründen unerwünscht sein, nicht zuletzt aufgrund des Verbrauchs von Rechnerressourcen, der mit der Abfrage und Verarbeitung der Information einhergeht.Personal Assistance modules are software processes that are installed on computing devices such as smartphones, tablets, vehicle navigation systems, and smart watches to provide user-friendly interfaces for users to make natural language commands. Users often make natural voice commands using their voice, but personal assistance modules can also respond to other modalities, such as various forms of text input. Existing personal assistance modules may lack intelligence to access the user's previous information and / or preferences. Accordingly, when a user makes an ambiguous statement (eg, request, statement, command) that may potentially be responded differently, the personal assistant module may need to request additional information from the user to determine how it is going should. Suppose a user speaks the words "Call Bob" in his phone, but there are several contacts in the user's contact list that are named "Bob." A personal assistant module running on the user's phone may ask which Bob the user meant and / or whether to call Bob's phone at home or at work. Or suppose another user speaks the words "remember to go shopping tomorrow" into his smart watch. A personal assistant module running on the user's smart watch - or on a smartphone communicating with the smart watch - may ask which application (eg, calendar, task scheduler, social network application) the user wants to use to create the reminder. The request for such additional information may be undesirable for a number of reasons, not least due to the consumption of computational resources associated with querying and processing the information.

Eine einzelne aus der Vielzahl von in Frage kommenden Antwortaktionen kann dann auf nichtdeterministische Weise (z. B. stochastisch) ausgewählt werden (z. B. mit zumindest einem gewissen Zufallsfaktor), zumindest auf Grundlage der Wahrscheinlichkeiten. Wenn Kandidat A beispielsweise eine 51%ige Chance hat, auf die Benutzerangabe reagieren zu können, und Kandidat B eine 49%ige Chance hat, auf die Benutzerangabe reagieren zu können, wird Kandidat A wahrscheinlich in ungefähr 51% der Fälle ausgewählt werden und Kandidat B wird wahrscheinlich in ungefähr 49% der Fälle ausgewählt werden. Indem nur eine einzelne Antwortaktion ausgewählt wird, wird der Benutzer nicht mit mehreren Antwortaktionen überflutet. Stattdessen wird dem Benutzer eine einzige Wahl angeboten, die der Benutzer annehmen oder zurückweisen kann. Die Annahme oder Ablehnung der einzelnen nichtdeterministisch ausgewählten Antwortaktion kann eine wichtige Rückmeldung für eine zukünftige für diese Antwortaktion (oder für ausreichend ähnliche Antwortaktionen) bestimmte Wahrscheinlichkeit sein.A single one of the plurality of candidate response actions may then be selected in a non-deterministic manner (eg, stochastically) (eg, with at least some random factor) based at least on the probabilities. For example, if candidate A has a 51% chance of responding to the user and candidate B has a 49% chance of responding to the user's indication, candidate A will likely be selected in about 51% of the cases and candidate B will probably be selected in about 49% of cases. By selecting only a single response action, the user is not flooded with multiple response actions. Instead, the user is offered a single choice that the user can accept or reject. The acceptance or rejection of the individual non-deterministically selected response action may be an important feedback for a future probability determined for this response action (or for sufficiently similar response actions).

In den nachfolgend besprochenen Implementierungen kann ein persönliches Assistenzmodul (auch als „intelligenter persönlicher Assistent” oder „mobiler Assistent” bezeichnet) über Zugriff auf einen Verlauf von Interaktionen zwischen einem Benutzer und einem oder mehreren Computergeräten in einem Ökosystem von vom Benutzer betriebenen Geräten (z. B. sein Telefon, Tablet, Smart Watch, Auto-Navigationssytem, etc.) verfügen. Wenn ein Benutzer eine Angabe macht (normalerweise, aber nicht immer, durch Sprache), kann das persönliche Assistenzmodul einen nichtdeterministischen Aufgabeninitiierungsprozess initiieren, der eine Vielzahl von in Frage kommenden Antwortaktionen identifiziert, die auf die Angabe des Benutzers reagieren können. Auf Grundlage des Interaktionsverlaufs kann der mobile Assistent Wahrscheinlichkeiten bestimmen, dass die Vielzahl von in Frage kommenden Antwortaktionen auf die Benutzerangabe reagieren kann.In the implementations discussed below, a personal assistant module (also referred to as a "smart personal assistant" or "mobile assistant") may access accessing a history of interactions between a user and one or more computing devices in an ecosystem of user-operated devices (e.g. Phone, tablet, smartwatch, car navigation system, etc.). When a user makes an indication (usually, but not always, by voice), the personal assistant module may initiate a non-deterministic task initiation process that identifies a plurality of candidate response actions that may respond to the user's indication. Based on the interaction history, the mobile assistant can determine probabilities that the large number of eligible response actions can respond to the user's indication.
Eine einzelne aus der Vielzahl von in Frage kommenden Antwortaktionen kann dann nichtdeterministisch ausgewählt werden (z. B. stochastisch, mit zumindest einem gewissen Zufallsfaktor), in einigen Fällen zumindest teilweise auf den Wahrscheinlichkeiten basierend. Wenn die in Frage kommende Antwortaktion A beispielsweise eine 51%ige Chance hat, auf die Benutzerangabe reagieren zu können, und in Frage kommende Antwortaktion B eine 49%ige Chance hat, auf die Benutzerangabe reagieren zu können, wird die in Frage kommende Antwortaktion A wahrscheinlich in ungefähr 51% der Fälle ausgewählt werden und die in Frage kommende Antwortaktion B wird wahrscheinlich in ungefähr 49% der Fälle ausgewählt werden. In anderen Implementierungen können möglicherweise nur diejenigen in Frage kommenden Kandidaten vorläufig ausgewählt werden, die über Wahrscheinlichkeiten verfügen, die einen oder mehrere Grenzwerte einhalten, woraufhin eine Antwortaktion nichtdeterministisch (z. B. komplett zufällig oder zufällig auf der Grundlage von Wahrscheinlichkeiten) aus den verbleibenden Kandidaten ausgewählt werden kann. Indem nur eine einzelne Antwortaktion ausgewählt wird, wird der Benutzer nicht mit mehreren Antwortaktionen überflutet. Stattdessen wird dem Benutzer eine einzige Wahl angeboten, die der Benutzer annehmen oder zurückweisen kann.A single one of the plurality of candidate response actions may then be nondeterministically selected (e.g., stochastically, with at least some random factor), in some cases based at least in part on the probabilities. For example, if the responding action A in question has a 51% chance of responding to the user's indication, and a candidate response B in question has a 49% chance of responding to the user's response, the response A in question becomes likely in about 51% of the cases will be selected and the eligible response B will likely be selected in about 49% of the cases. In other implementations, only those candidate candidates who have probabilities that comply with one or more limits may be preliminarily selected, whereupon a response action may be non-deterministic (eg, completely random or random based on probabilities) from the remaining candidates can be selected. By selecting only a single response action, the user is not flooded with multiple response actions. Instead, the user is offered a single choice that the user can accept or reject.
Auf Grundlage von Benutzerinteraktion mit dem Computergerät im Anschluss an die Initiierung der nichtdeterministisch ausgewählten Antwortaktion kann eine mit der ausgewählten Antwortaktion assoziierte Wahrscheinlichkeit modifiziert werden. Wenn die Antwortaktion beispielsweise war, eine Internet-Telefonieanwendung zu öffnen und die Nummer eines bestimmten Benutzers vorzuwählen, ohne den Anruf tatsächlich durchzuführen, und der Benutzer auf die „Anrufen”-Taste drückt, um den Anruf durchzuführen, kann das dazu führen, dass die mit dieser Aktion assoziierte Wahrscheinlichkeit erhöht wird. Wenn der Benutzer hingegen die Internet-Telefonieanwendung ablehnt, indem er sie schließt, ohne den Anruf durchzuführen, kann die mit dieser Antwortaktion assoziierte Wahrscheinlichkeit vermindert werden.Based on user interaction with the computing device following the initiation of the nondeterministically selected response action, a probability associated with the selected response action may be modified. For example, if the response action was to open an Internet telephony application and dial the number of a particular user without actually making the call, and the user presses the "call" key to make the call, it may cause the user to increased probability associated with this action. If, on the other hand, the user rejects the Internet telephony application by closing it without the Calling can reduce the likelihood associated with this response action.
