Das vorliegende Dokument betrifft das Präsentieren von Inhalt.This document concerns the presentation of content.
STAND DER TECHNIKSTATE OF THE ART
Das Beliefern von Benutzern mit relevantem Werbeinhalt ist für Werbende und Service Provider im Allgemeinen wichtig. Das Implementieren einer kosteneffektiven Möglichkeit zum Bereitstellen eines derartigen relevanten Werbeinhalts kann sich in einem ständig ändernden Online-Markt als schwierig erweisen. Während relevante Informationen zum spezifischen Adressieren eines bestimmten Benutzers möglicherweise einer Entität bekannt sind, haben andere möglicherweise nicht ohne weiteres Zugang zu solchen Informationen oder sind in der Lage, sie zu verwenden, wenn beispielsweise Targeting-Entscheidungen getroffen werden.Providing users with relevant advertising content is generally important to advertisers and service providers. Implementing a cost effective way to provide such relevant advertising content can be difficult in a constantly changing online marketplace. While relevant information for specifically addressing a particular user may be known to an entity, others may not readily access or be able to use such information when making, for example, targeting decisions.

Werbende, Verlage und Service Provider wünschen möglicherweise allgemein, Daten zu dem Zweck auszutauschen, einen sinnvollen Weg zum Liefern von Informationen und/oder Dienstleistungen (zum Beispiel Werbeinhalt) an Online-Benutzer zu liefern. Falls bestimmt wird, dass für einen bestimmten Benutzer ein spezifischer Inhalt sinnvoll ist, dann wünscht der Benutzer möglicherweise, auf den Inhalt zuzugreifen, den Inhalt zu kaufen oder anderweitig mit dem Inhalt zu interagieren. Diese Interaktion kann dem Inhalt Provider (zum Beispiel Werbender, Verlag oder Service Provider) Einnahmen liefern. Falls ein bestimmter Inhalt Provider (zum Beispiel Werbender, Verlag oder Service Provider) Daten darüber sammeln kann, wie spezifischer Inhalt für Benutzer möglicherweise sinnvoll ist oder möglicherweise nicht sinnvoll ist (zum Beispiel in der Form einer Benutzerliste), können die gesammelten Daten von anderen in einer Vielzahl von Wegen verwendet werden. Eine Verwendung betrifft das Auswählen von zielgerichtetem Inhalt. Andere Verwendungen sind möglich, wie etwa beim Anpassen von Geboten bei einer Auktion beispielsweise auf der Basis von Benutzerlisten, die anzeigen, dass ein spezifischer Inhalt für einen oder mehrere Benutzer von Interesse ist. Benutzerlisten können veröffentlicht, verkauft oder lizenziert werden oder es kann anderweitig auf sie zugegriffen werden, um das Bereitstellen eines personalisierten Inhalt an die spezifischen Benutzer und das Vergrößern der Einnahme für einen Inhalt Provider zu unterstützen.Advertisers, publishers, and service providers may generally wish to share data for the purpose of providing a meaningful way to deliver information and / or services (e.g., advertising content) to online users. If it is determined that specific content is meaningful to a particular user, then the user may wish to access the content, purchase the content, or otherwise interact with the content. This interaction may provide revenue to the content providers (eg, advertisers, publishers, or service providers). If a particular content provider (eg, advertiser, publisher, or service provider) may collect data about how specific content may be useful to users or may not be useful (for example, in the form of a user list), the information collected may be collected by others be used in a variety of ways. One use involves selecting targeted content. Other uses are possible, such as adjusting bids at auction, for example, based on user lists indicating that specific content is of interest to one or more users. User lists may be published, sold or licensed, or otherwise accessed, to assist in providing personalized content to the specific users and increasing revenue for a content provider.
Benutzerlisten können spezifische Benutzerinformationen darstellen, die vordefinierte Kategorien betreffen. Die Kategorien können von den Datenbesitzern definiert werden. Beispielsweise kann eine Benutzerliste Daten über einen oder mehrere Benutzer enthalten, die den Benutzer für eine Kategorie charakterisiert (zum Beispiel Hauseigentümer, Handwerker, DVD-Ausleiher usw.), um das zielgerichtete Ansprechen der Benutzer beispielsweise durch Verlage oder Werbende zu gestatten. Bei einigen Implementierungen können die Benutzerlisten verwendet werden, um relevanten Werbeinhalt zu adressieren.User lists can represent specific user information concerning predefined categories. The categories can be defined by the data owners. For example, a user list may contain data about one or more Contain users that characterize the user for a category (for example, homeowner, craftsman, DVD lender, etc.) to allow the targeted response of users, for example, by publishers or advertisers. In some implementations, the user lists may be used to address relevant advertising content.
Benutzerlisten können gemäß einer Anzahl von Regeln generiert und ausgetauscht werden, und diese Regeln können verwendet werden, um bestimmte Benutzerlisten an spezifische Verbraucher zu vermarkten. Die Regeln können Verfahren zum Zuweisen von Benutzern zu bestimmten Benutzerlisten verwenden. Solche Regeln können eine logische Kategorisierung von Daten, Informationen oder Diensten für den Zweck bereitstellen, zu bestimmen, welcher Dateninhalt in den Benutzerlisten für eine Anzahl von Benutzern besonders relevant ist.User lists can be generated and exchanged according to a number of rules, and these rules can be used to market specific user lists to specific consumers. The rules may use methods for assigning users to particular user lists. Such rules may provide a logical categorization of data, information, or services for the purpose of determining which data content in the user lists is particularly relevant to a number of users.
Es werden Verfahren beschrieben, um benutzerspezifische Informationen mit einer oder mehreren Benutzerlisten zu assoziieren, die von einem Datenbesitzer besessen oder gepflegt werden. Eine Assoziation zwischen den benutzerspezifischen Daten und den Benutzerlisten wird hergestellt. Die Assoziation kann beispielsweise für das Echtzeitbieten als Reaktion auf Anforderungen nach Inhalt oder zum Anpassen von Inhalt, der einem spezifischen Benutzer geliefert werden soll, ausgenutzt werden. Andere Verwendungen der Benutzerlisteninformationen und -assoziationen sind möglich.Methods are described for associating user-specific information with one or more user lists that are owned or maintained by a data owner. An association between the user-specific data and the user lists is established. For example, the association may be exploited for real-time bidding in response to requests for content or for customizing content to be delivered to a specific user. Other uses of user list information and associations are possible.

Das tabellenbasierte Repository kann beispielsweise durch ein proprietäres verteiltes Speicherungssystem mit einer multidimensionalen sortierten Karte repräsentiert werden, wie in dem Referat beschrieben mit dem Titel „Bigtable: A Distributed Storage System for Structured Data” von Fay Chang, Jeffrey Dean, Sanjay Ghemawat, Wilson C. Hsieh, Deborah A. Wallach, Mike Burrows, Tushar Chandra, Andrew Fikes und Robert E. Gruber , dessen Inhalt in seiner Gänze hier durch Bezugnahme aufgenommen ist. Das tabellenbasierte Repository stellt ein verteiltes Speicherungssystem zum Verwalten strukturierter Daten dar, das ausgelegt ist, auf eine sehr große Größe zu skalieren (zum Beispiel Petabyte an Daten über Tausende von Commodity Servern). Jede Benutzerliste in der proprietären Datenbank kann in einer oder mehreren Tabellen mit mehreren Dimensionen enthalten sein (von denen eine ein Feld für Zeit sein kann, was Versionierung und Garbage Collection gestattet).During operation and for every user list generated, the system can 100 For example, in a table-based repository, store a list of user lists to which a particular user belongs. For example, the table-based repository may be represented by a proprietary distributed storage system having a multidimensional sorted map as described in the unitary title "Bigtable: A Distributed Storage System for Structured Data" by Fay Chang, Jeffrey Dean, Sanjay Ghemawat, Wilson C. Hsieh, Deborah A. Wallach, Mike Burrows, Tushar Chandra, Andrew Fikes and Robert E. Gruber , the contents of which are hereby incorporated by reference in their entirety. The table-based repository is a distributed storage system for managing structured data that is designed to scale to a very large size (for example, petabytes of data across thousands of commodity servers). Each user list in the proprietary database may be contained in one or more tables with multiple dimensions (one of which may be a field for time, allowing versioning and garbage collection).
