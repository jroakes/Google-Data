Abstract
구현들은 특정 자동화 어시스턴트와의 인터페이스를 위해 명시적으로 제공되는 기능이 없는 비-어시스턴트 애플리케이션과 상호 작용할 수 있는 자동화 어시스턴트에 관한 것이다. 비-어시스턴트 애플리케이션과 관련된 주석 데이터 및/또는 GUI 데이터와 같은 애플리케이션 데이터는 처리되어 이러한 데이터를 임베딩 공간에 매핑할 수 있다. 그런 다음 어시스턴트 입력 커맨드가 처리되고, 동일한 임베딩 공간에 매핑되고, 그리고 어시스턴트 입력 커맨드 임베딩과 비-어시스턴트 애플리케이션 데이터 임베딩으로부터의 거리가 결정될 수 있다. 어시스턴트 입력 커맨드 임베딩과 비-어시스턴트 애플리케이션 데이터 임베딩 사이의 거리가 임계값(들)을 만족할 때, 자동화 어시스턴트는 비-어시스턴트 애플리케이션 데이터에 대응하는 비-어시스턴트 애플리케이션에 대한 명령(들)을 생성할 수 있다. 예를 들어, 명령(들)은 비-어시스턴트 애플리케이션이 비-어시스턴트 애플리케이션 데이터에 의해 특징지어지거나 그와 관련된 하나 이상의 동작을 수행하게 하는 사용자 입력(들)을 시뮬레이션할 수 있다.Implementations are directed to automated assistants that can interact with non-assistant applications that do not have the functionality explicitly provided for interfacing with a particular automated assistant. Application data, such as GUI data and/or annotation data associated with non-assistant applications, may be processed to map such data to an embedding space. The assistant input command can then be processed, mapped to the same embedding space, and the distance from the assistant input command embedding and the non-assistant application data embedding can be determined. When the distance between the assistant input command embedding and the non-assistant application data embedding satisfies the threshold value(s), the automated assistant may generate command(s) for the non-assistant application corresponding to the non-assistant application data. . For example, the command(s) may simulate user input(s) that cause the non-assistant application to perform one or more actions characterized by or related to the non-assistant application data.
Description
인간은 본 명세서에서 "자동화 어시스턴트"(또한 "디지털 에이전트", "챗봇", "대화형 개인 비서", "지능형 개인 비서", "어시스턴트 애플리케이션", "대화형 에이전트" 등으로 지칭됨)라고 지칭되는 대화형 소프트웨어 애플리케이션을 사용하여 인간 대 컴퓨터 대화에 참여할 수 있다. 예를 들어 (자동화 어시스턴트와 상호 작용할 때 "사용자"라고 지칭될 수 있는) 인간은 텍스트로 변환된 다음 처리될 수 있는 음성 자연어 입력(즉, 발언)을 사용하여 및/또는 (예를 들어, 타이핑된) 텍스트 자연어 입력을 제공함으로써 자동화 어시스턴트에게 커맨드 및/또는 요청을 제공할 수 있다. 자동화 어시스턴트는 청각적 및/또는 시각적 사용자 인터페이스 출력을 포함할 수 있는 응답 사용자 인터페이스 출력을 제공함으로써 요청에 응답한다.Humans are referred to herein as "automated assistants" (also referred to as "digital agents", "chatbots", "interactive personal assistants", "intelligent personal assistants", "assistant applications", "interactive agents", etc.) can engage in human-to-computer conversations using interactive software applications that are For example, a human (who may be referred to as a "user" when interacting with an automated assistant) may use spoken natural language input (i.e., speech) that can be converted to text and then processed (i.e., utterances) and/or (i.e., typing ) can provide commands and/or requests to the automated assistant by providing text natural language input. The automated assistant responds to the request by providing responsive user interface output, which may include audio and/or visual user interface output.
위에서 언급한 바와 같이, 자동화 어시스턴트는 사용자의 음성 발언에 해당하는 오디오 데이터를 대응하는 텍스트(또는 다른 의미론적 표현)로 변환할 수 있다. 예를 들어, 오디오 데이터는 사용자가 자동화 어시스턴트와 상호 작용할 수 있도록 하는 어시스턴트 인터페이스를 포함하는 클라이언트 디바이스의 하나 이상의 마이크로폰을 통한 사용자의 음성 발언의 검출에 기초하여 생성될 수 있다. 자동화 어시스턴트는 음성 발언에 의해 생성되는 소리(예를 들어, 음소), 생성된 소리의 순서, 음성의 리듬, 억양 등과 같은, 오디오 데이터에서 캡처된 음성 발언의 다양한 특성을 인식하려고 시도하는 음성 인식 엔진을 포함할 수 있다. 또한, 음성 인식 엔진은 이러한 특성으로 표현되는 텍스트 단어 또는 구문을 식별할 수 있다. 그런 다음 텍스트는 음성 발언에 대한 응답 컨텐츠를 결정할 때 (예를 들어, 자연어 이해(NLU) 엔진 및/또는 대화 상태 엔진을 사용하여) 자동화 어시스턴트에 의해 추가로 처리될 수 있다. 음성 인식 엔진은 클라이언트 디바이스에 의해 및/또는 클라이언트 디바이스로부터 원격이지만 클라이언트 디바이스와 네트워크 통신하는 하나 이상의 자동화 어시스턴트 컴포넌트(들)에 의해 구현될 수 있다.As noted above, the automated assistant may convert audio data corresponding to the user's spoken utterances into corresponding text (or other semantic representation). For example, audio data may be generated based on detection of a user's spoken utterance through one or more microphones of a client device that includes an assistant interface that allows the user to interact with the automated assistant. The automated assistant is a speech recognition engine that attempts to recognize various characteristics of spoken utterances captured in audio data, such as the sounds (e.g., phonemes) produced by the spoken utterances, the sequence of sounds produced, the rhythm of the voice, intonation, etc. can include Also, the speech recognition engine can identify text words or phrases represented by these properties. The text may then be further processed by the automated assistant (eg, using a natural language understanding (NLU) engine and/or a dialog state engine) in determining response content to the spoken utterance. The speech recognition engine may be implemented by the client device and/or by one or more automated assistant component(s) remote from the client device but in network communication with the client device.
사용자가 자동화 어시스턴트 및 애플리케이션(예를 들어, 웹 브라우저)과 동시에 상호 작용할 때, 애플리케이션에 대한 입력으로 인해 자동화 어시스턴트가 수행하는 동작들이 중단될 수 있으며 그 반대의 경우도 마찬가지이다. 게다가, 애플리케이션에 액세스하는 동안 자동화 어시스턴트의 호출은 사용자가 애플리케이션과의 추가 상호 작용에 더 이상 관심이 없고 애플리케이션으로 하여금 특정 동작을 일시 중지하게 하거나 닫히게 한다는 표시로 많은 시스템에 의해 가정될 수 있다. 이는 사용자가 실제로 자동화 어시스턴트로 하여금 애플리케이션 및/또는 다른 애플리케이션과 관련된 동작을 수행하게 하려고 할 때 계산 리소스를 낭비할 수 있다. 이러한 경우, 애플리케이션과의 상호 작용 동안 이전에 생성된 임의의 데이터가 손실될 수 있고 이로 인해 사용자는 애플리케이션을 다시 초기화하거나 애플리케이션으로 이전 동작을 반복해야만 한다.When a user interacts with an automated assistant and an application (eg, a web browser) simultaneously, inputs to the application may interrupt actions performed by the automated assistant and vice versa. Additionally, invoking an automated assistant while accessing an application may be assumed by many systems as an indication that the user is no longer interested in further interaction with the application and is instructing the application to suspend or close certain actions. This can waste computational resources when the user is actually trying to get the automated assistant to perform actions related to the application and/or other applications. In this case, any data previously created during interaction with the application may be lost, forcing the user to reinitialize the application or repeat previous actions with the application.
자동화 어시스턴트는 사용자가 다른 애플리케이션을 작동할 때 제한된 기능을 가질 수 있다. 그 결과, 사용자는 사용자가 다른 애플리케이션(들)과 관련시키는 특정 기능(들)을 수행하기 위해 자동화 어시스턴트를 호출하려고 시도할 수 있지만(즉, 자동화 어시스턴트가 다른 애플리케이션(들)을 제어하게 하려고 시도함), 자동화 어시스턴트가 그러한 기능을 수행할 수 없을 때 궁극적으로 자동화 어시스턴트와의 대화 세션을 종료할 수 있다. 예를 들어, 자동화 어시스턴트의 제한된 가용성 또는 제한된 기능은 사용자가 자동화 어시스턴트를 통해 처리되는 음성 커맨드를 통해 다른 애플리케이션을 제어할 수 없음을 의미할 수 있다. 이것은 대화 세션 동안 음성 발언의 임의의 처리가 다른 애플리케이션과의 임의 동작의 수행으로 이어지지 않기 때문에 네트워크 및 처리 대역폭과 같은 계산 리소스를 낭비할 수 있다. 더욱이, 이러한 결점 때문에, 사용자가 자신의 개별 자동화 어시스턴트와 상호 작용할 때, 사용자는 사용자에 의해 수행되는 다양한 동작에 대해 효율성을 제공할 수 있는 다른 애플리케이션을 작동시키지 않을 수 있다.An automated assistant may have limited functionality when the user is operating other applications. As a result, the user may attempt to invoke the automated assistant to perform certain function(s) that the user associates with the other application(s) (i.e., attempt to have the automated assistant control the other application(s)). ), can ultimately end the conversation session with the automated assistant when the automated assistant is unable to perform those functions. For example, limited availability or limited functionality of an automated assistant may mean that a user cannot control other applications via voice commands processed through the automated assistant. This can waste computational resources, such as network and processing bandwidth, since any processing of voice utterances during a conversation session does not lead to the performance of any actions with other applications. Moreover, because of this drawback, when a user interacts with his/her individual automated assistant, the user may not launch other applications that may provide efficiencies for the various actions performed by the user.
본 명세서에 설명된 구현은 별도의 애플리케이션으로 하여금 특정 기능(들)을 수행하게 하기 위해 자동화 어시스턴트를 사용하는 것과 관련되며, 여기서 별도의 애플리케이션으로 하여금 특정 기능(들)을 수행하게 하는 것은 별도의 애플리케이션의 임의의 자동화 어시스턴트 애플리케이션 프로그래밍 인터페이스(API)와 독립적으로 수행된다. 다른 말로 하면, 별도의 애플리케이션의 자동화 어시스턴트 API(있는 경우)에는 특정 기능을 수행하기 위한 능력이 없을 수 있다. 그러나, 본 명세서에 개시된 구현은 그럼에도 불구하고 특정 기능(들)을 수행할 때 별도의 애플리케이션을 제어하기 위해 자동화 어시스턴트를 사용할 수 있다. 이러한 방식 및 기타 방식으로, 자동화 어시스턴트는 이러한 별도의 애플리케이션이 특정(또는 임의의) 자동화 어시스턴트 API 기능을 통합할 필요 없이 다양한 개별 애플리케이션을 제어하는데 활용될 수 있다. 자동화 어시스턴트를 통해 별도의 애플리케이션을 제어하는 것은 자동화 어시스턴트에 의해 처리되는 음성 입력을 통해 별도의 애플리케이션(들)을 제어할 수 있는 반면 별도의 애플리케이션에는 특정(또는 임의의) 음성을 처리하는 기능이 없을 수 있다. 별도의 애플리케이션 제어에 사용할 수 있는 입력 모달리티(양식)를 늘리고 손재주가 제한된 사용자에게 도움이 되는 것 외에도, 음성 입력을 통해 별도의 애플리케이션의 기능 제어를 가능하게 하면 해당 기능을 더 빠르게 수행할 수 있으며 결과적으로 개별 애플리케이션과의 전체 상호 작용 시간이 짧아질 수 있다. 예를 들어, 기능을 수행하기 위해 음성 입력을 제공하는 것은 별도의 애플리케이션 내에서 탐색하여 기능에 대한 사용자 인터페이스 요소를 찾은 다음 해당 사용자 인터페이스 요소(들)와 상호 작용하는 것보다 시간 효율적일 수 있다. 추가적으로 또는 대안적으로, 특정 자동화 어시스턴트 API 기능이 없는 별도의 애플리케이션은 결과적으로 온-디바이스 저장소를 덜 차지할 수 있고 실행 시 랜덤 액세스 메모리(RAM)를 덜 사용할 수 있다.Implementations described herein involve using an automated assistant to cause a separate application to perform a particular function(s), where the separate application performs a particular function(s). independent of any automated assistant application programming interfaces (APIs). In other words, the separate application's automation assistant API (if any) may not have the ability to perform certain functions. However, implementations disclosed herein may nonetheless use an automated assistant to control separate applications when performing certain function(s). In these and other ways, an automated assistant may be utilized to control a variety of separate applications without the need for these separate applications to incorporate specific (or any) automated assistant API functions. Controlling a separate application through an automated assistant may control the separate application(s) via voice input processed by the automated assistant, whereas the separate application may not have the ability to process specific (or arbitrary) voices. can In addition to increasing the modality (forms) of input available for controlling a separate application and benefiting users with limited dexterity, enabling control of a separate application's function via voice input allows that function to be performed more quickly and consequently As a result, the overall interaction time with individual applications can be shortened. For example, providing voice input to perform a function may be more time efficient than navigating within a separate application to find the user interface element for the function and then interacting with that user interface element(s). Additionally or alternatively, a separate application without certain automated assistant API functions may consequently occupy less on-device storage and may use less random access memory (RAM) when running.

For example, in response to the user providing the spoken statement “Assistant, send me an image using my camera,” the automated assistant may provide one or more simulated inputs to the messaging application. The one or more simulated inputs may be generated and/or determined based on interaction data, which may be based on previous interactions between the one or more users and the messaging application. In response, the messaging application may initialize the computing device's camera, use the camera to capture an image, and send the image to the recipient with whom the user is currently interacting via the messaging application. In some instances, these operations may be performed with a backend process when the messaging application is not rendered in the foreground of the computing device to further conserve computational resources.
일부 구현에서, 자동화 어시스턴트는 사용자가 외부 애플리케이션과 반드시 상호 작용하지 않을 때 수신되는 음성 발언에 응답하여 통신할 외부 애플리케이션을 식별하기 위해 하나 이상의 트레이닝된 기계 학습 모델을 사용하여 작동할 수 있다. 다시 말해서, 사용자는 컴퓨팅 디바이스 디스플레이 패널 및/또는 임의의 다른 디스플레이 인터페이스의 전경에서 실행되고 있지 않은 외부 애플리케이션을 제어하기 위해 자동화 어시스턴트로 향하는 음성 발언을 제공할 수 있다. 오히려, 외부 애플리케이션은 사용자가 음성 발언을 제공할 때 컴퓨팅 디바이스에서 실행(launch)되지 않았을 수 있다. 사용자가 참조하고 있는 외부 애플리케이션을 정확하게 식별하고 사용자의 요청을 충족시킬 하나 이상의 동작을 식별하기 위해, 하나 이상의 트레이닝된 기계 학습 모델을 사용하여 음성 발언의 자연어 컨텐츠가 처리될 수 있다.In some implementations, an automated assistant can operate using one or more trained machine learning models to identify an external application to communicate with in response to spoken utterances received when the user is not necessarily interacting with the external application. In other words, the user can provide a spoken utterance directed to the automated assistant to control an external application that is not running in the foreground of the computing device display panel and/or any other display interface. Rather, the external application may not have been launched on the computing device when the user provided the spoken utterance. The natural language content of the spoken utterance may be processed using one or more trained machine learning models to accurately identify the external application the user is referencing and identify one or more actions that will satisfy the user's request.
일부 구현에서, 트레이닝된 기계 학습 모델은 외부 애플리케이션 및 수행될 하나 이상의 동작을 식별하기 위해 음성 발언과 관련된 자연어 컨텐츠를 처리하는데 사용될 수 있다. 다른 구현에서, 제1 트레이닝된 기계 학습 모델은 외부 애플리케이션을 식별할 때 사용되고, 제2 트레이닝된 기계 학습 모델은 식별된 외부 애플리케이션이 수행할 하나 이상의 동작을 식별할 때 사용된다.In some implementations, the trained machine learning model can be used to process natural language content associated with spoken utterances to identify an external application and one or more actions to be performed. In another implementation, a first trained machine learning model is used when identifying an external application and a second trained machine learning model is used when identifying one or more actions for the identified external application to perform.
예를 들어, 사용자는 집에 있을 수 있으며 자동화 어시스턴트 및 그 자동화 어시스턴트와 별도로 사용 가능한 기타 비-어시스턴트 애플리케이션에 대한 액세스를 제공할 수 있는 자신의 태블릿 컴퓨팅 디바이스에서 책을 읽고 있을 수 있다. 사용자의 집에는 태블릿 컴퓨팅 디바이스에 설치된 제어 애플리케이션을 통해 제어할 수 있는 다양한 스마트 전원 콘센트가 있을 수 있다. 그러나, 제어 애플리케이션은 자동화 어시스턴트와의 인터페이스를 위해 명시적으로 제공된 기능 및/또는 라이브러리를 제공하지 않을 수 있다. 사용자가 제어 애플리케이션을 직접 실행할 필요 없이 자동화 어시스턴트를 통해 스마트 전원 콘센트를 제어하기 위해, 사용자는 자동화 어시스턴트에 입력을 제공할 수 있다. 입력은 제어 애플리케이션 및/또는 그 제어 애플리케이션에 의해 수행될 수 있는 하나 이상의 동작에 대한 입력을 상관시키기 위해 하나 이상의 트레이닝된 기계 학습 모델을 사용하여 처리될 수 있다.For example, a user may be at home and reading a book on their tablet computing device, which may provide access to an automated assistant and other non-assistant applications available separately from the automated assistant. A user's home may have a variety of smart power outlets that can be controlled through a control application installed on the tablet computing device. However, the control application may not provide explicitly provided functionality and/or libraries for interfacing with the automated assistant. To control the smart power outlet through the automated assistant without requiring the user to directly launch the control application, the user can provide input to the automated assistant. Inputs may be processed using one or more trained machine learning models to correlate the inputs to the control application and/or one or more actions that may be performed by the control application.
예를 들어, 자동화 어시스턴으에 대한 사용자의 입력은 "어시스턴트, 거실 창가에 있는 콘센트(outlet)를 켜줘"일 수 있다. 이에 응답하여, 자동화 어시스턴트는 사용자가 자동화 어시스턴트로 하여금 제어 애플리케이션이 관리하는 콘센트에 대해 "켜기(on)" 동작을 수행하도록 제어 애플리케이션에 지시하게 할 의도인지를 결정하기 위해 입력이 처리되도록 할 수 있다. 입력의 처리는 수행될 적절한 애플리케이션 및/또는 동작을 선택하기 위해 하나 이상의 트레이닝된 기계 학습 모델 및/또는 하나 이상의 휴리스틱 기술을 포함할 수 있다. 일부 구현에서, 자동화 어시스턴트는 다른 애플리케이션(예를 들어, 제어 애플리케이션)을 통해 제공되고 및/또는 애플리케이션과 관련하여 저장된 데이터를 사용하여 트레이닝되는 하나 이상의 기계 학습 모델을 사용할 수 있다.For example, a user's input to an automated assistant could be "Assistant, turn on the outlet in the living room window." In response, the automated assistant can cause the input to be processed to determine whether the user intends the automated assistant to instruct the controlling application to perform an “on” action on an outlet that the controlling application manages. . Processing of the input may include one or more trained machine learning models and/or one or more heuristic techniques to select appropriate applications and/or actions to be performed. In some implementations, an automated assistant may use one or more machine learning models that are trained using data provided through another application (eg, a control application) and/or stored in association with the application.
일부 구현에서는, 메타 데이터 및/또는 XML 데이터와 같은 보충 데이터가 자동화 어시스턴트에 대한 액세스도 제공하는 컴퓨팅 디바이스를 통해 액세스 가능한 다양한 다른 애플리케이션에서 사용될 수 있다. 보충 데이터는 사용자의 어시스턴트 입력을 다른 애플리케이션에 의해 수행될 수 있는 동작들과 매핑하는 것을 돕기 위해 하나 이상의 기계 학습 모델을 트레이닝하는데 사용되는 트레이닝 데이터의 일부일 수 있다. 예를 들어, 제어 애플리케이션과 관련된 메타 데이터에는 온-오프 스위치와 같은 특정 선택 가능한 GUI 요소를 기술하는 정보가 포함될 수 있다. 온-오프 스위치는 GUI 요소가 콘센트를 제어하기 위한 것임을 나타내는 해당 메타 데이터 및/또는 XML 데이터를 가질 수 있다(예를 들어, "# 스마트 콘센트 제어"). 게다가, 특정 동작에 대한 추가 컨텍스트를 제공할 수 있는 코드 및/또는 GUI 요소의 다른 부분과 같은 다른 데이터가 하나 이상의 기계 학습 모델을 트레이닝하는데 사용될 수도 있다. 예를 들어, 코드의 일부는 "result == read('Living_Room_Window(status)')를 포함할 수 있으므로 코드의 해당 부분을 포함하는 함수의 특정 부분이 "거실 창"과 관련된 디바이스를 제어할 수 있음을 나타낸다. 따라서, 사용자로부터의 입력을 처리하는 것는 온-오프 스위치 GUI 요소에도 해당하는 임베딩 공간의 위치에 매핑할 수 있으며, 이에 따라 자동화 어시스턴트가 사용자 입력에 응답하여 온-오프 스위치 GUI 요소를 토글링하기 위한 사용자 입력(예를 들어, 터치 인터페이스에 대한 터치 입력)을 시뮬레이션할 수 있게 한다.In some implementations, supplemental data such as metadata and/or XML data may be used in a variety of other applications accessible through a computing device that also provides access to an automated assistant. Supplemental data may be part of training data used to train one or more machine learning models to help map the user's assistant inputs to actions that may be performed by other applications. For example, meta data associated with a control application may include information describing specific selectable GUI elements such as an on-off switch. The on-off switch may have corresponding metadata and/or XML data indicating that the GUI element is for controlling the outlet (eg, “# control smart outlet”). Additionally, other data, such as other parts of code and/or GUI elements that may provide additional context for certain actions, may be used to train one or more machine learning models. For example, a part of code could contain "result == read('Living_Room_Window(status)') so that a specific part of the function containing that part of code could control the device associated with the "living room window". Thus, processing input from the user can map to a location in the embedding space that also corresponds to the on-off switch GUI element, whereby the automated assistant toggles the on-off switch GUI element in response to the user input. to simulate user input for ringing (eg, touch input to a touch interface).
하나 이상의 기계 학습 모델을 트레이닝하기 위한 트레이닝 데이터의 인스턴스를 생성하기 위해, 하나 이상의 연합 학습 기술이 사용될 수 있다. 예를 들어, 자동화 어시스턴트는 사용자를 위한 음성 발언에 대한 제안을 생성할 수 있으며, 사용자는 외부 애플리케이션을 제어하기 위해 자동화 어시스턴트에게 이를 반복할 수 있다. GUI 데이터, 메타 데이터, XML 데이터 및/또는 기타 데이터와 같은 외부 애플리케이션과 관련된 다양한 데이터를 사용하여 특정 제안된 커맨드가 생성될 수 있다. 예를 들어, 디바이스명 및 디바이스에 대한 선택 가능한 설정을 특징짓는 데이터가 "어시스턴트, [디바이스명]을 [선택 가능한 설정]으로 변경해줘"와 같은 제안된 음성 발언을 생성하기 위해 처리될 수 있다. 제안된 음성 발언은 특정 애플리케이션이 하나 이상의 동작을 수행하도록 하기 위해 사용자가 자동화 어시스턴트에 제공할 수 있다. 대안적으로, 사용자는 자동화 어시스턴트에게 음성 발언을 반복하지 않도록 선택할 수 있다.One or more federated learning techniques may be used to create instances of training data for training one or more machine learning models. For example, an automated assistant can generate suggestions for spoken utterances for a user, which the user can repeat to the automated assistant to control an external application. Certain suggested commands may be generated using various data associated with an external application, such as GUI data, metadata, XML data, and/or other data. For example, data characterizing the device name and selectable settings for the device may be processed to generate a suggested spoken utterance such as "Assistant, change [device name] to [selectable settings]". Suggested voice utterances can be provided by the user to the automated assistant in order to cause a particular application to perform one or more actions. Alternatively, the user may choose not to repeat the spoken utterance to the automated assistant.
트레이닝 데이터는 사용자가 제안된 각 음성 발언을 반복하는지 여부를 반영하도록 컴파일될 수 있다. 다시 말해서, 사용자에 의해 반복되는 제안된 음성 발언은 하나 이상의 애플리케이션 입력 및/또는 애플리케이션 동작과 상관될 수 있다. 더욱이, 사용자에 의해 반복되지 않는 제안된 음성 발언은 관련이 없는 것으로 간주될 수 있으며, 따라서 임의의 외부 애플리케이션 동작과 관련하여 순위가 낮아지거나 저장되지 않는다. 일부 구현에서, 제안된 음성 발언 및/또는 그에 상응하는 자연어 컨텐츠는 임베딩 공간에 매핑된 하나 이상의 외부 애플리케이션 동작 근처에 매핑될 수 있다. 그러나, 음성 발언이 제안되고 사용자에 의해 선택되지 않은 결과(예를 들어, 임계 시간 기간 내에 선택되지 않음), 음성 발언은 임베딩 공간에서 동작으로부터 더 멀리 이동될 수 있다. 즉, 특정 외부 애플리케이션 동작과 관련이 있다고 예측되는 음성 발언은 임베딩 공간에 있는 임베딩에 해당할 수 있다. 디바이스 인터페이스에서 음성 발언이 제안되었을 때 한 명 이상의 사용자가 그 음성 발언을 사용하지 않는 경우, 그 임베딩의 거리는 임베딩 공간에서 예측 동작으로부터 증가될 수 있다. 그러나, 일부 구현에서, 음성 발언은 다른 애플리케이션 및/또는 다른 동작을 위해 다시 제안될 수 있다. 사용자가 이후 제안된 음성 발언을 선택하면, 그 음성 발언에 대응하는 임베딩은 다른 애플리케이션 및/또는 다른 동작에 더 가깝게 매핑될 수 있다. 이러한 방식으로, 연합 학습은 자동화 어시스턴트에 대한 음성 발언을 자동화 어시스턴트 외부에 있는 다른 애플리케이션에서 수행할 수 있는 동작과 상관시키기 위한 임베딩 생성을 지원할 수 있다.Training data can be compiled to reflect whether the user repeats each suggested spoken utterance. In other words, suggested spoken utterances repeated by the user may be correlated with one or more application inputs and/or application actions. Furthermore, suggested spoken utterances that are not repeated by the user may be considered irrelevant and, therefore, not ranked or stored with respect to any external application operation. In some implementations, suggested spoken utterances and/or corresponding natural language content may be mapped near one or more external application actions mapped in the embedding space. However, as a result of a spoken utterance being suggested and not selected by the user (eg, not selected within a threshold time period), the spoken utterance may be moved further away from the action in the embedding space. That is, a voice utterance predicted to be related to a specific external application operation may correspond to an embedding in an embedding space. If one or more users are not using the spoken utterance when it is proposed at the device interface, the distance of the embedding may be increased from the prediction operation in the embedding space. However, in some implementations, the spoken utterance may be re-suggested for other applications and/or other actions. If the user then selects a suggested spoken utterance, the embedding corresponding to that spoken utterance may map more closely to other applications and/or other actions. In this way, federated learning can support the creation of embeddings to correlate spoken utterances to an automated assistant with actions that other applications outside of the automated assistant can perform.
