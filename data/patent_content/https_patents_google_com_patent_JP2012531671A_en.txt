Abstract
或る組織のためのメーリングリストにおけるＥメールメッセージが、そのメッセージの所定の特徴に基づいてモニター及びフィルターされる（要モデレーションの待ち行列に入れられる）。その後でメーリングリストのマネージャが、待ち行列に入れられたメッセージを承認して対象受信者に配送するか、拒絶して配送を阻止するかを決定することができる。別の方法では、メッセージを承認するか拒絶するかを、所定の特徴に基づいて自動的に判定する。所定の特徴は、メーリングリストの規模；Ｅメールメッセージのサイズ；メッセージにおける或る指定された用語（例えば、商品名や機密等の特定の用語）の有無；件名の有無；送信されたメッセージの数が多すぎるかどうか；あるリストに記載されたＥメールアドレス等である。
【選択図】 図７Email messages on a mailing list for an organization are monitored and filtered (queued in a moderation required) based on the predetermined characteristics of the message. The mailing list manager can then decide whether to approve the queued message for delivery to the intended recipient or to reject it and block delivery. Another method automatically determines whether to accept or reject the message based on predetermined characteristics. Predefined features include: mailing list size; size of email message; presence of certain specified terms in the message (eg, specific terms such as product name or confidentiality); presence of subject; number of messages sent Whether there are too many; e-mail addresses on a list, etc.

本発明は電子メーリングリストのオンラインマネジメントに関し、特に、メーリングリストの自動Ｅメール（Email: 電子メール）メッセージモデレーション（moderation: 調整、適正化）に関する。 The present invention relates to online management of electronic mailing lists, and more particularly to automatic email (Email) message moderation (moderation: adjustment, optimization) of mailing lists.
ユーザは、電子メーリングリストにより、多数のメーリングリスト加入者に対してＥメールメッセージを配送することができる。通常は、加入者を追加するか募ることによりメーリングリストを作成した人が、そのメーリングリストのオーナー・マネージャとなる。ユーザは、様々なツールやオンラインアプリケーションを利用して、このようなメーリングリストを作成することもできるし、このメーリングリストへのアクセスやメーリングリストの利用を管理することもできる。しかしながら、これらのツールは大抵、カジュアルユーザが大したトレーニングを受けることもなくスムーズにリストの管理が行えるという程簡単に使いこなせるものではない。そのため、たとえそのメーリングリストの主題に最も直接的に関わっていなくても、管理者レベルのユーザがメーリングリストを作成・管理するのが普通である。特に企業環境においては、リストの開設・運用の最適任者と思われるチームリーダーや、プロジェクトマネージャ等が既存のメーリングリストツールを使ってリストを適正に開設し、モデレートし、又は運用するだけの時間や専門知識がないという場合がある。 Users can deliver email messages to a large number of mailing list subscribers via an electronic mailing list. Usually, the person who creates a mailing list by adding or recruiting subscribers becomes the owner / manager of the mailing list. The user can create such a mailing list using various tools and online applications, and can manage access to the mailing list and use of the mailing list. However, these tools are often not as easy to use as casual users can manage their lists smoothly without much training. Therefore, it is common for an administrator-level user to create and manage a mailing list even if it is not most directly related to the subject of the mailing list. Especially in the corporate environment, team leaders who are considered to be the best managers for opening and operating lists, project managers, etc. have enough time to properly open, moderate or operate lists using existing mailing list tools. There may be no expertise.
メーリングリストは開設された時点で、いろいろな考えを伝え、議論を促進するのに利用できるオンラインフォーラムとなる。メーリングリストのメンバーは通常、互いに連絡をとりあう為に他のメンバー達にメッセージを簡単に送ることができる。それ故に、悪意を持ったユーザがこの自由を悪用して、メーリングリスト全体に迷惑な又は不快なメッセージを発信することがあり得る。更に、軽率なユーザが送信するに相応しくないメッセージを発信することもあり得る。メーリングリストのモデレーションは、リスト運用の重要なアスペクトであり、リストの目的達成の有効性を可能な限り維持できるようにする。従って、専門のリスト管理者ではなくて、リスト運用に多くの時間を充てられないごく普通のユーザによるメーリングリストのモデレーションと運用全般を容易にするシステムが必要とされる。更に、使い方が簡単で、これを使って広めた情報のセキュリティを保証する組織団体固有のメールリストシステムも必要である。 When the mailing list is established, it becomes an online forum that can be used to convey ideas and facilitate discussion. Mailing list members usually can easily send messages to other members to keep in touch with each other. Therefore, a malicious user can exploit this freedom and send annoying or unpleasant messages to the entire mailing list. Furthermore, it is possible that a low-profile user sends a message that is not suitable for transmission. Mailing list moderation is an important aspect of list operations and helps maintain the effectiveness of fulfilling the purpose of the list as much as possible. Therefore, there is a need for a system that facilitates the general moderation and operation of mailing lists by ordinary users who are not specialized list managers and cannot spend much time on list operations. Further, there is a need for an organization-specific mail list system that is easy to use and guarantees the security of the information spread using it.
メーリングリストのＥメール（電子メール）メッセージは、その所定の特徴に基づいてモニター・フィルターされ、或いはモデレーションキューに入れられる。モデレーション待ちのメッセージは、メーリングリストマネージャに伝えられ、承認して配信か拒絶の判定を受ける。この他、所定の特徴に基づいてメッセージを承認するか拒絶するかを自動的に判定する方法もある。所定の特徴とは、メーリングリストの規模；Ｅメールメッセージのサイズ；メッセージにおける指定用語（”極秘”、スパムを示唆する商標名等）の有無：無件名；送信されたメッセージの数が多すぎるかどうか；所定のリストから特定されるＥメールアドレスなどである。 Email (email) messages on the mailing list are monitored, filtered or placed in a moderation queue based on their predetermined characteristics. Messages waiting for moderation are communicated to the mailing list manager for approval and delivery or rejection. In addition, there is a method of automatically determining whether to accept or reject a message based on a predetermined characteristic. Predetermined features include: mailing list size; size of email message; presence of specified term in message ("confidential", trademark name suggesting spam, etc.): no subject; whether too many messages were sent An e-mail address specified from a predetermined list.

The system includes a first computing subsystem configured to receive a message from a client. The message is addressed to a plurality of addresses belonging to the mailing list. The system further includes a second computing subsystem configured to communicate with the first computing subsystem. The second computing subsystem is configured to examine a portion of the message and determine whether that portion passes a predetermined test. The system further includes a third computing subsystem configured to communicate with the second computing subsystem. The third computing subsystem is configured to mark the email message for processing. The process is, in one embodiment, marking the message as moderation required, and in another embodiment, deleting the message.

The method includes receiving an email message from a client by a first computing subsystem. In this step, the message is addressed to a plurality of addresses belonging to the mailing list. The method further includes examining a portion of the email message by the second computing subsystem and determining by the second computing subsystem whether the portion passes a predetermined test. The method further includes the step of marking the email message for processing by a third computing subsystem.
