Abstract
내비게이션 길안내 미리보기를 제공하기 위해, 서버 디바이스는 출발 위치로부터 목적지 위치까지의 내비게이션 길안내에 대한 요청을 수신하고 요청에 대한 응답으로 내비게이션 길안내 세트를 생성한다. 내비게이션 길안내 세트는 출발 위치로부터 목적지 위치까지 이동하기 위한 경로 세그먼트들의 세트를 포함한다. 서버 디바이스는 상기 경로 세그먼트들의 세트 내의 각 경로 세그먼트의 특성에 기초하여 상기 경로 세그먼트들의 서브세트를 선택한다. 선택된 각 경로 세그먼트에 대해, 서버 디바이스는 클라이언트 디바이스에 디스플레이될 경로 세그먼트의 미리보기를 제공한다. 경로 세그먼트의 미리보기는 경로 세그먼트를 도시하는 파노라마 거리 레벨 이미지를 포함한다.To provide a preview of navigation directions, a server device receives a request for navigation directions from a starting location to a destination location and generates a set of navigation directions in response to the request. A set of navigation directions includes a set of route segments for traveling from a starting location to a destination location. The server device selects the subset of route segments based on a characteristic of each route segment in the set of route segments. For each route segment selected, the server device provides a preview of the route segment to be displayed on the client device. The preview of the route segment includes a panoramic street level image depicting the route segment.

오늘날 많은 사용자가 다양한 지리적 위치에 대한 지도 및 내비게이션 데이터를 요청한다. 컴퓨터, 스마트폰, 임베디드 디바이스 등에서 실행되는 소프트웨어 애플리케이션은 일반적으로 시작 지점과 목적지를 지정하여 사용자로부터 입력을 수신하는 것에 대한 응답으로 단계별 내비게이션 길안내를 생성한다. 내비게이션 길안내는 일반적으로 최단 시간 내에 사용자를 목적지 위치로 안내하는 경로에 대해 생성된다.Many users today request map and navigation data for various geographic locations. Software applications running on computers, smart phones, embedded devices, etc. generally generate step-by-step navigation directions in response to receiving input from a user by specifying a starting point and a destination. Navigation directions are generally created for routes that lead the user to a destination location in the shortest amount of time.
출발 위치로부터 목적지 위치까지 경로를 이동하기 위한 내비게이션 길안내의 미리보기를 제공하기 위해 내비게이션 미리보기 생성 시스템은 내비게이션 미리보기에 포함할 경로 세그먼트들의 서브세트를 선택하며, 여기서 각 경로 세그먼트는 내비게이션 길안내 세트의 내비게이션 지시에 대응한다. 내비게이션 미리보기 생성 시스템은 경로 세그먼트에 대한 기동 유형, 기동과 이전 기동 사이의 거리 및/또는 시간, 기동에 대한 복잡도 레벨, 경로 세그먼트의 위치, 경로 세그먼트 상 트래픽 양 등과 같은 경로 세그먼트들에 대한 특성들에 기초하여 경로 세그먼트들의 서브세트를 선택할 수 있다. 내비게이션 미리보기 생성 시스템은 경로 세그먼트와의 사용자의 익숙도, 내비게이션 미리보기와의 사용자의 이전 인터렉션 등과 같은 내비게이션 길안내를 요청하는 사용자의 특성들에 기초하여 경로 세그먼트들의 서브세트를 선택할 수 있다. 그러나, 일부 실시예에서 내비게이션을 요청하는 사용자와 불가지론적인 특성들에 기초한 경로 세그먼트들의 서브세트의 선택과 관련된다. To provide a preview of navigation directions for moving a route from a starting location to a destination location, the navigation preview generation system selects a subset of route segments to include in the navigation preview, where each route segment is Corresponds to the set's navigation instructions. The navigation preview generation system may determine characteristics for route segments such as type of maneuver for that route segment, distance and/or time between maneuver and previous maneuver, level of complexity for maneuver, location of route segment, amount of traffic on route segment, etc. may select a subset of path segments based on The navigation preview generation system may select a subset of route segments based on characteristics of the user requesting navigation directions, such as the user's familiarity with the route segment, the user's previous interactions with the navigation preview, and the like. However, some embodiments involve selection of a subset of route segments based on agnostic characteristics with the user requesting navigation.
일반적으로 내비게이션 지시/길안내의 미리보기를 제공하면 사용자가 내비게이션의 기술적 작업을 수행하는데 도움이 되는 기술적 효과를 얻을 수 있다. 본 명세서에 기술된 바와 같이, 미리보기는 기동을 포함하거나 다른 특성을 갖는 특정 경로 세그먼트들에 관한 정보를 사용자에게 제공할 수 있다. 이를 통해 사용자는 경로에 더 익숙해지고, 해당 경로를 길탐색하기 전에 경로를 따라 잠재적 위험 또는 기타 지점을 인식할 수 있다. 이는 사용자가 길탐색을 수행하는데 직접적으로 도움이 되며 사용자가 보다 안전하게 길을 탐색할 수 있도록 한다. In general, if a preview of navigation instructions/route guidance is provided, a technical effect conducive to a user performing a technical task of navigation can be obtained. As described herein, a preview may provide the user with information about specific route segments that contain maneuvers or have other characteristics. This allows users to become more familiar with the route and be aware of potential hazards or other points along the route before navigating that route. This directly helps the user to perform route search and allows the user to navigate more safely.
일부 구현예에서, 내비게이션 미리보기 생성 시스템은 경로 세그먼트의 특성, 사용자 특성 또는 임의의 다른 적절한 특성에 기초하여 각각의 경로 세그먼트들에 점수를 매긴다. 그런 다음 내비게이션 미리보기 생성 시스템은 임계 점수 이상으로 점수가 매겨진 경로 세그먼트들의 서브세트를 선택한다. 추가적으로 또는 대안적으로, 내비게이션 미리보기 생성 시스템은 각각의 점수에 기초하여 경로 세그먼트의 순위를 매길 수 있다. 그런 다음 내비게이션 미리보기 생성 시스템은 임계 순위 이상으로 순위가 매겨진 경로 세그먼트들의 서브세트를 선택한다.In some implementations, the navigation preview generation system scores each of the route segments based on characteristics of the route segment, user characteristics, or any other suitable characteristic. The navigation preview generation system then selects a subset of route segments that score above a threshold score. Additionally or alternatively, the navigation preview generation system may rank route segments based on their respective scores. The navigation preview generation system then selects a subset of route segments ranked above a threshold rank.
임의의 이벤트에서, 내비게이션 미리보기 생성 시스템은 선택된 서브세트의 각 경로 세그먼트들을 포함하는 내비게이션 미리보기를 생성한다. 선택된 서브세트의 각각의 경로 세그먼트에 대해, 내비게이션 미리보기는 경로 세그먼트에 대응하는 영역의 파노라마 거리 레벨 이미지의 디스플레이를 포함할 수 있다. 일부 구현예에서, 내비게이션 미리보기는 경로 세그먼트에 대한 기동을 위한 웨이포인트 위치까지 가는, 웨이포인트 위치에서의, 웨이포인트 위치에 후속하는 지리적 영역에 대한 파노라마 거리 레벨 이미지를 도시하는 비디오 프레임을 포함하는 경로 세그먼트에 대한 비디오 디스플레이를 포함할 수 있다. 파노라마 이미지 또는 파노라마 이미지를 도시하는 비디오 디스플레이는 사용자가 내비게이션을 수행하는데 도움을 주는 기술적 효과를 달성하는데 특히 이점이 있다. 이는 사용자에게 경로를 나타내는 시각적 정보가 제공되어 단순히 내비게이션 지시가 아닌 경로 세그먼트에 대한 보다 다양한 정보에 접근할 수 있기 때문이다. 이러한 방식으로 사용자는 내비게이션을 수행하는데 도움이 되는 추가 정보를 제공받는다.At any event, the navigation preview generation system generates a navigation preview that includes each route segment in the selected subset. For each route segment in the selected subset, the navigation preview may include display of a panoramic street level image of the area corresponding to the route segment. In some implementations, the navigation preview comprises a video frame showing a panoramic street level image for a geographic area following the waypoint location, at the waypoint location, to the waypoint location for maneuvering for the route segment. Can include video displays for route segments. A panoramic image or a video display showing a panoramic image is particularly advantageous in achieving a technical effect of assisting a user in performing navigation. This is because visual information representing a route is provided to the user so that he or she can access more diverse information about a route segment than a simple navigation instruction. In this way, the user is provided with additional information to assist in navigation.
파노라마 거리 레벨 이미지 외에도, 내비게이션 미리보기는 경로 세그먼트에 대한 기동의 설명과 같은 오디오를 포함할 수 있다. 경로 세그먼트에 대한 기동은 차량 기동, 즉 경로 세그먼트에서 차량에 의해 수행되는 기동으로 간주될 수 있다. 다른 구현예에서, 기동은 사용자가 걷고 있을 때와 같이 사용자에 의해 수행될 수 있다. 내비게이션 미리보기의 오디오는 내비게이션 중에 제공되는 오디오 내비게이션 지시와 다를 수 있다. 예를 들어, 내비게이션 미리보기를 위한 오디오는 내비게이션 중에 제공되는 오디오 내비게이션 지시보다 더 상세할 수 있다. 미리 보기 중 오디오는 내비게이션 중에 제공되는 오디오에서 추가 컨텍스트를 제공할 수 있다. 보다 구체적으로, 내비게이션 미리보기를 위한 오디오는 기동에 대한 설명, 관심 지점(POI) 또는 기동을 수행할 때 사용자를 안내하는데 도움이 될 수 있는 기타 랜드마크에 대한 설명, 기동을 수행할 때 사용자를 안내하는데 도움이 될 수 있는 트래픽 신호에 대한 설명을 포함할 수 있다. 이러한 방식으로, 사용자는 내비게이션 미리보기를 볼 수 있고, 경로에서 오디오 설명 기동을 들을 수 있어 다가오는 기동을 준비하고 경로에서 무엇을 예상해야 하는지 알 수 있다. 그런 다음 사용자는 어려움이 있을 것으로 예상되는 특정 경로 세그먼트를 검토하고, 기동을 수행하기 전에 영역의 실세계 표현을 볼 수 있다. 즉, 사용자의 내비게이션 수행에 도움이 될 수 있는 추가 정보를 오디오 정보 형태로 제공한다. 추가 오디오 정보는 (내비게이션 미리보기의 일부로) 미리 제공되므로 내비게이션 중에 정보가 제공된 경우 해당 정보로 사용자의 주의가 산만해지는 위험을 방지할 수 있다. 이러한 추가 정보는 내비게이션 동안에 처음 제공되는 경우 사용자를 너무 산만하게 할 수 있으므로, 내비게이션에 미리보기로 정보를 제공하면 사용자의 주의가 산만해질 위험이 줄어들어 보다 안전한 내비게이션을 가능하게 한다.In addition to panoramic street level images, navigation previews may include audio, such as descriptions of maneuvers for route segments. A maneuver on a route segment can be considered a vehicle maneuver, i.e. a maneuver performed by a vehicle on a route segment. In another implementation, the maneuvering can be performed by the user, such as when the user is walking. The audio of the navigation preview may differ from the audio navigation instructions provided during navigation. For example, audio for a navigation preview may be more detailed than audio navigation instructions provided during navigation. Audio during preview can provide additional context from the audio provided during navigation. More specifically, audio for navigation previews includes descriptions of maneuvers, descriptions of points of interest (POIs) or other landmarks that may help guide the user as they perform the maneuver, and guide the user as they perform the maneuver. You may include descriptions of traffic signals that may help guide you. In this way, the user can see a preview of the navigation and hear the audio narrated maneuvers on the route so they can prepare for upcoming maneuvers and know what to expect along the route. Users can then review specific path segments that are expected to have difficulty, and see a real-world representation of the area before performing the maneuver. That is, additional information that can help the user perform navigation is provided in the form of audio information. Additional audio information is presented in advance (as part of the navigation preview), avoiding the risk of the user being distracted by the information if presented during navigation. This additional information can be too distracting to the user if initially presented during navigation, so providing the information as a preview to the navigation reduces the risk of user distraction, allowing for safer navigation.

I/O 모듈(14)은 예를 들어 터치 스크린일 수 있다. 다양한 구현예에서, 클라이언트 디바이스(10)는 도 1에 도시된 것보다 적은 수의 컴포넌트를 포함하거나 반대로 추가적 컴포넌트를 포함할 수 있다. 다른 실시예에서, 클라이언트 디바이스(10)는 임의의 적절한 휴대용 또는 비휴대용 컴퓨팅 디바이스일 수 있다. 예를 들어, 클라이언트 디바이스(10)는 랩톱 컴퓨터, 데스크톱 컴퓨터, 스마트 워치 또는 스마트 글래스와 같은 웨어러블 디바이스, 차량 헤드 유닛 등일 수 있다. 메모리(20)는 또한 상기 표시된 바와 같이 인터렉티브 디지털 지도를 생성하고 및/또는 다른 지리적 기능을 수행하도록 구성된 지도 애플리케이션(22)을 저장한다. 지도 애플리케이션(22)은 내비게이션 지시, 내비게이션 미리보기를 위한 경로 세그먼트들의 서브세트, 내비게이션 미리보기에 포함할 각각의 경로 세그먼트에 대한 시각적 및/또는 오디오 데이터를 수신하고, 내비게이션 디스플레이(24)를 통해 시각적 및/또는 오디오 데이터를 제공할 수 있다. 지도 애플리케이션(22)은 운전, 도보 또는 대중 교통 길안내를 디스플레이할 수 있고, 일반적으로 지리, 지리위치 내비게이션 등과 관련된 기능을 내비게이션 디스플레이(24)를 통해 제공한다.
비록 도 1가 지도 애플리케이션(22)을 독립형 애플리케이션으로서 도시하지만, 지도 애플리케이션(22)의 기능은 또한 클라이언트 디바이스(10) 상에서 실행되는 웹 브라우저를 통해 액세스 가능한 온라인 서비스의 형태로 클라이언트 디바이스(10) 상에서 실행되는 다른 소프트웨어 애플리케이션에 대한 플러그인 또는 확장으로서 제공될 수 있다. 맵핑 애플리케이션(22)은 일반적으로 상이한 각각의 운영 체제에 대해 상이한 버전으로 제공될 수 있다. 예를 들어, 클라이언트 디바이스(10)의 제조사는 Android™ 플랫폼용 맵핑 애플리케이션(22), iOS™ 플랫폼용 다른 SDK 등을 포함하는 SDK(Software Development Kit)를 제공할 수 있다. 사용자 특성은 예를 들어 사용자의 과거 내비게이션 요청 및 사용자의 집 위치에 기초하여 결정될 수 있는 사용자 익숙도 메트릭을 포함할 수 있다. 사용자 특성은 또한 사용자가 이전 내비게이션 요청 동안 특정 내비게이션 지시를 따랐는지 여부, 사용자가 이전 내비게이션 요청 동안 특정 유형의 조작을 피했는지 여부와 같은 사용자에 대한 행동 데이터에 기초하여 결정될 수 있는 사용자 호환성 메트릭을 포함할 수 있다. 사용자 호환성 메트릭은 과거에 기동이 회피되었는지 또는 부정확하게 수행되었는지 여부를 결정하기 위해 후보 경로 세그먼트에 대한 기동을 사용자에 대한 행동 데이터와 비교함으로써 결정될 수 있다. 또한, 사용자 특성은 이전 인터렉션 메트릭은 내비게이션 미리보기와의 사용자의 이전 인터렉션에 기초하여 결정될 수 있는 이전 사용자 인터렉션을 포함할 수 있다. User characteristics may include user familiarity metrics, which may be determined based on the user's past navigation requests and the user's home location, for example. User characteristics also include user compatibility metrics that can be determined based on behavioral data about the user, such as whether the user followed a particular navigation instruction during a previous navigation request, or whether the user avoided a particular type of manipulation during a previous navigation request. can do. A user compatibility metric may be determined by comparing maneuvers on candidate route segments to behavioral data for the user to determine whether maneuvers were avoided or performed incorrectly in the past. Additionally, the user characteristics may include previous user interactions, where a previous interaction metric may be determined based on the user's previous interactions with the navigation preview.
