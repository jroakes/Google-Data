HINTERGRUNDBACKGROUND
Das Internet bietet Zugang zu einer großen Vielfalt an Ressourcen. So sind zum Beispiel Video- und/oder Audiodateien sowie Webseiten für bestimmte Themen oder bestimmte Nachrichtenartikel über das Internet zugänglich. Der Zugriff auf diese Ressourcen bietet Möglichkeiten, andere Inhalte (z. B. Werbung) mit den Ressourcen bereitzustellen. Eine Webseite kann zum Beispiel Slots beinhalten, in denen Inhalt präsentiert werden kann. Diese Slots können in der Webseite definiert werden oder für eine Präsentation mit einer Webseite, zum Beispiel zusammen mit Suchergebnissen, präsentiert werden. Inhalt in diesen Beispielen kann verschiedene Formate annehmen, während die Geräte, die den Inhalt verbrauchen (z. B. präsentieren) in Bezug auf ihre Art und Fähigkeiten gleichermaßen unterschiedlich sein können.The Internet provides access to a wide variety of resources. For example, video and / or audio files as well as web pages for certain topics or certain news articles are accessible via the Internet. Accessing these resources provides opportunities to provide other content (such as advertising) with the resources. For example, a web page may contain slots in which content can be presented. These slots can be defined in the webpage or presented for presentation with a webpage, for example along with search results. Content in these examples may take on various formats, while the devices that consume (eg, present) the content may be equally diverse in terms of their nature and capabilities.
KURZDARSTELLUNGSUMMARY
Im Allgemeinen kann ein innovativer Aspekt des in dieser Spezifikation beschriebenen Gegenstands in Verfahren implementiert werden, die computerimplementierte Verfahren für die Bereitstellung von Inhalt beinhalten. Das Verfahren kann das Empfangen einer Vielzahl von Schnappschüssen, die mit der Verwendung eines Computergeräts durch einen Benutzer verbunden sind, durch ein Servergerät beinhalten, wobei jeder Schnappschuss aus der Vielzahl von Schnappschüssen auf Inhalt basiert, der dem Benutzer auf dem Computergerät präsentiert wird. Das Verfahren kann ferner die Bewertung der Vielzahl von Schnappschüsse beinhalten, einschließlich für jeden jeweiligen Schnappschuss: Identifizieren eines jeweiligen Satzes von Entitäten, der durch den jeweiligen Schnappschuss angegeben wird, und Speichern von Hinweisen des jeweiligen Satzes von Entitäten und eines jeweiligen Zeitstempels, der eine jeweilige Zeit angibt, zu der der jeweilige Schnappschuss aufgenommen wurde, wobei der jeweilige Satz von Entitäten und der jeweilige Zeitstempel im Speicher verknüpft sind. Das Verfahren kann ferner auf Basis eines ersten Schnappschusses aus der Vielzahl der Schnappschüsse die Bestimmung einer ersten Zeit für die Präsentation der einen oder mehreren Informationskarten für den Benutzer beinhalten. Das Verfahren kann ferner zum ersten Zeitpunkt das Suchen von Entitäten im Speicher mit einem Zeitstempel, der der ersten Zeit entspricht, beinhalten. Das Verfahren kann ferner das Erzeugen einer Informationskarte auf Basis der einen oder mehreren der gefundenen Entitäten beinhalten. Das Verfahren kann ferner das Bereitstellen der erzeugten Informationskarte für die Präsentation für den Benutzer beinhalten.In general, an innovative aspect of the subject matter described in this specification may be implemented in methods that include computer-implemented methods for providing content. The method may include receiving, by a server device, a plurality of snapshots associated with the use of a computing device by a user, each snapshot of the plurality of snapshots based on content presented to the user on the computing device. The method may further include evaluating the plurality of snapshots, including for each respective snapshot, identifying a respective set of entities indicated by the respective snapshot and storing indicia of the respective set of entities and a respective timestamp having a respective one Specifies the time at which the particular snapshot was taken, with the respective set of entities and the respective timestamp in memory associated. The method may further include determining a first time to present the one or more information cards to the user based on a first snapshot of the plurality of snapshots. The method may further include, at the first time, searching for entities in memory having a time stamp corresponding to the first time. The method may further include generating an information map based on the one or more found entities. The method may further include providing the generated information card for presentation to the user.

AUSFÜHRLICHE BESCHREIBUNGDETAILED DESCRIPTION
Systeme, Verfahren und Computerprogrammprodukte für die Bereitstellung einer Informationskarte oder anderer Formen der Benachrichtigung werden beschrieben, die auf Basis von einem oder mehreren bewerteten Schnappschüssen von Inhalt, der einem Benutzer präsentiert wird, bestimmt werden. Schnappschüsse können auf Basis des Inhalts, der einem oder mehreren Benutzern auf ihren jeweiligen Benutzergeräten präsentiert wird, laufend aufgenommen und bewertet werden. Inhalt kann einem Benutzer zum Beispiel in einem Browser, in einer Anwendung (z. B. einer mobilen App), einer Website, einer Werbung, einer Seite eines sozialen Netzwerks oder anderen digitalen Inhaltsumgebungen angezeigt werden. Jeder Schnappschuss kann mindestens einen Teil eines oder mehrerer der folgenden Elemente beinhalten: einen Kalendereintrag, eine Karte, eine E-Mail-Nachricht, einen Eintrag auf einer Seite eines sozialen Netzwerks, ein Webseitenelement, ein Bild oder sonstige Inhalte. Die Bewertung eines bestimmten Schnappschusses kann das Identifizieren zugehöriger Entitäten (z. B. Personen, Orte (z. B. spezifische Standorte, Adressen, Städte, Staaten, Länder, Raumnummern, Gebäude oder andere spezifische geografische Orte), Dinge (z. B. Telefonnummern), Themen, geplante Ereignisse (z. B. Mittagessen-Termine, Geburtstage, Besprechungen) oder andere identifizierbare Einheiten beinhalten. Ein mit dem Empfang eines Schnappschusses verknüpfter Zeitstempel kann ebenfalls in Verbindung mit dem Schnappschuss und/oder Entitäten gespeichert werden, auf denen der Schnappschuss basiert. Zielpräsentationszeiten können zum Beispiel auf Basis eines Zeitstempels bestimmt werden, der mit dem Empfang des Schnappschusses verknüpft ist, und/oder auf Basis von Zeiten eines oder mehrerer Ereignisse, die mithilfe des Schnappschusses identifiziert wurden. Zu Zeiten, die den Zielpräsentationszeiten entsprechen, können eine oder mehrere Informationskarten, die eine oder mehrere Entitäten identifizieren, bereitgestellt werden (z. B. für die Präsentation für den Benutzer). Jede Informationskarte kann außerdem zum Beispiel einen Kontext angeben, den der Benutzer verwenden kann, um den Grund für die Anzeige der bestimmten Informationskarte zu verstehen. Mindestens eine Handlungsaufforderung kann in der Informationskarte enthalten sein, um es dem Benutzer zum Beispiel zu ermöglichen, eine Handlung durchzuführen, die mit einer Entität verbunden ist (z. B. Wählen einer Telefonnummer, Erhalt von Wegbeschreibungen oder Empfangen zusätzlicher Informationen). Eine Informationskarte kann als Eingabeaufforderung jeglicher Art dienen (z. B. damit sich der Benutzer ein Konzept und/oder eine andere Information merken kann), oder die Informationskarte kann als Erinnerung an ein kommendes Ereignis dienen.Systems, methods, and computer program products for providing an information card or other forms of notification are determined based on one or more scored snapshots of content presented to a user. Snapshots may be continually recorded and evaluated based on the content presented to one or more users on their respective user devices. Content may be displayed to a user in, for example, a browser, an application (eg, a mobile app), a website, an advertisement, a social networking site, or other digital content environment. Each snapshot may include at least a portion of one or more of the following: a calendar entry, a card, an email message, an entry on a social network page, a web page item, an image, or other content. The evaluation of a particular snapshot may include identifying related entities (e.g., people, locations (e.g., specific locations, addresses, cities, states, countries, room numbers, buildings, or other specific geographic locations), things (eg, phone numbers), topics, scheduled events (eg, lunch appointments, birthdays, meetings) or other identifiable units. A timestamp associated with receiving a snapshot may also be stored in association with the snapshot and / or entities upon which the snapshot is based. For example, target presentation times may be determined based on a timestamp associated with receiving the snapshot and / or based on times of one or more events identified by the snapshot. At times that correspond to the target presentation times, one or more information cards that identify one or more entities may be provided (eg, for presentation to the user). For example, each information card may also indicate a context that the user may use to understand the reason for displaying the particular information card. At least one action request may be included in the information card to allow the user, for example, to perform an action associated with an entity (eg dial a telephone number, obtain directions or receive additional information). An information card may serve as a prompt of any kind (eg, to allow the user to remember a concept and / or other information), or the information card may serve as a reminder of an upcoming event.

Eine Konversion kann als stattgefunden gelten, wenn ein Benutzer eine bestimmte Transaktion oder Aktion in Verbindung mit einem Inhaltselement durchführt, das mit einer Ressource oder einer Suchergebnisseite bereitgestellt wird. Was eine Konversion darstellt, kann sich von Fall von Fall unterscheiden und kann auf eine Vielzahl von Arten bestimmt werden. Eine Konversion kann zum Beispiel erfolgen, wenn ein Benutzer auf ein Inhaltselement (z. B. eine Werbung) klickt, zu einer Webseite geleitet wird und einen Kauf abschließt, bevor er diese Webseite verlässt. Eine Konversion kann durch einen Inhaltsanbieter auch als jede beliebige messbare oder beobachtbare Benutzeraktion definiert werden, zum Beispiel Herunterladen eines Whitepaper, Navigation mindestens bis zu einer bestimmten Tiefe einer Website, Anzeigen mindestens einer bestimmten Anzahl von Webseiten, Verbringen mindestens eines vorher bestimmten Zeitraums auf einer Website oder Webseite, Registrieren auf einer Website, Erleben von Medien oder Durchführen einer sozialen Aktion in Verbindung mit einem Inhaltselement (z. B. einer Werbung), wie zum Beispiel Unterstützen, erneutes Veröffentlichen oder Teilen des Inhaltselements. Andere Aktionen, die eine Konversion darstellen, können ebenfalls verwendet werden.A conversion may be considered to have occurred when a user performs a particular transaction or action associated with a content item provided with a resource or a search results page. What constitutes a conversion may differ from case to case and may be determined in a variety of ways. For example, a conversion may occur when a user clicks on a content item (eg, an advertisement), is directed to a web page, and makes a purchase before leaving that web page. A conversion may also be defined by a content provider as any measurable or observable user action, for example, downloading a white paper, navigating at least to a certain depth of a web site, displaying at least a certain number of web pages, spanning at least a predetermined time period on a web site or web page, registering on a web site, experiencing media, or performing a social action associated with a content item (eg, an advertisement), such as supporting, republishing, or sharing the content item. Other actions that are a conversion can also be used.
