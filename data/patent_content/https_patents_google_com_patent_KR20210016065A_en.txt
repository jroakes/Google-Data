Abstract
리턴 여정 통지를 생성하기 위한 시스템 및 방법은 목표 목적지까지의 내비게이션 길안내에 대한 요청을 획득하는 것을 포함한다. 초기 위치로부터 상기 목표 목적지까지의 아웃바운드 여정 경로가 결정될 수 있고, 상기 아웃바운드 여정 경로는 추정 아웃바운드 여정 시간을 포함한다. 상기 목표 목적지로부터 리턴 목적지까지의 리턴 여정 경로가 결정될 수 있고, 상기 리턴 여정 경로는 추정 리턴 여정 시간을 포함한다. 상기 아웃바운드 여정 경로 및/또는 리턴 여정 경로는 적어도 부분적으로 현재 교통 상황 또는 이력적 교통 상황 중 하나 이상으로부터 결정될 수 있다. 상기 추정 아웃바운드 여정 시간을 상기 추정 리턴 여정 시간과 비교하는 것이 하나 이상의 미리 결정된 기준을 만족한다고 결정되는 경우 상기 리턴 여정 경로에 관한 하나 이상의 통지들이 생성될 수 있다.A system and method for generating a return journey notification includes obtaining a request for navigation directions to a target destination. An outbound itinerary path from the initial location to the target destination may be determined, and the outbound itinerary path includes an estimated outbound itinerary time. A return journey route from the target destination to the return destination may be determined, and the return journey route includes an estimated return journey time. The outbound itinerary route and/or the return itinerary route may be determined, at least in part, from one or more of a current traffic situation or a historical traffic situation. One or more notifications regarding the return journey route may be generated when it is determined that comparing the estimated outbound journey time with the estimated return journey time satisfies one or more predetermined criteria.
Description
본 발명은 일반적으로 내비게이션 시스템에 관한 것이다. 보다 구체적으로, 본 발명은 리턴 여정 경로의 양태를 고려하는 경로 정보를 생성하는 것에 관한 것이다.The present invention relates generally to a navigation system. More specifically, the present invention relates to generating route information that takes into account aspects of a return journey route.
다양한 내비게이션 시스템은 사용자에게 턴-바이-턴 길안내를 제공한다. 이러한 시스템은 핸드헬드 GPS 디바이스 또는 모바일폰, 차량 장착 디바이스 또는 지도 애플리케이션에 액세스할 수 있는 인터넷 기반 컴퓨터를 포함할 수 있다. 사용자는 하나 이상의 위치들을 입력하고, 경로 및 턴-바이-턴 길안내를 수신한다. 일반적으로, 이러한 시스템은 경로를 따라 이동하는 가장 짧은 예상 시간에 기초하여 가장 빠른 경로를 선택할 수 있다. 일부 시스템은 교통 상황(예: 정체 보고)을이 계산에 통합할 수 있다. 일부 시스템은 추정 이동 시간을 포함하는 경로 정보를 하나 이상의 위치들로 제공할 수 있다. Various navigation systems provide turn-by-turn directions to users. Such systems may include handheld GPS devices or mobile phones, vehicle-mounted devices, or Internet-based computers capable of accessing map applications. The user enters one or more locations and receives a route and turn-by-turn directions. In general, these systems can select the fastest route based on the shortest expected time to travel along the route. Some systems may incorporate traffic conditions (eg congestion reporting) into this calculation. Some systems may provide route information including estimated travel time to one or more locations.
사용자는 계획 목적으로 내비게이션 시스템 및/또는 지도 애플리케이션에 의해 제공된 정보를 평가할 수 있다. 예를 들어, 목적지 위치까지의 추정 이동 시간을 포함하는 경로 정보는 사용자가 출발 시간을 계획하게 할 수 있다. 보다 구체적으로, 사용자는 실시간 교통 정보를 고려하여 식료품점까지의 추정 이동 시간을 보기 위해 로컬 식료품점을 목적지 위치로서 선택할 수 있다. 일부 경우에, 사용자는 이 내비게이션 정보에 기초하여 원하는 목적지에 관한 결정을 할 수 있다.The user may evaluate the information provided by the navigation system and/or map application for planning purposes. For example, route information including an estimated travel time to a destination location may allow a user to plan a departure time. More specifically, the user may select a local grocery store as the destination location to view the estimated travel time to the grocery store in consideration of real-time traffic information. In some cases, the user can make a decision regarding a desired destination based on this navigation information.

The method further comprises generating, by the one or more computing devices, a notification regarding the return journey route when it is determined that comparing the estimated outbound journey time with the estimated return journey time satisfies one or more predetermined criteria. For example, an outbound itinerary route from an initial location to the target destination may be determined, and an estimated outbound itinerary time may be included. A return journey route from the target destination to the return destination may be determined, and may include an estimated return journey time. The return destination may be the initial location or may be another location ("return" is used in the sense that the outbound itinerary is going to the target destination, and the return itinerary is away from the target destination). Based on the comparison of the outbound journey time and the return journey time, one or more notifications may be provided to the map application user. Return journey notifications may include, for example, an estimated return journey time, an estimated time difference between outbound and return journey times, warnings about predetermined route factors, and the like. By providing return itinerary notifications related to the time it takes the user to return to the origin or other return destination, such users are provided with more comprehensive and useful trip planning information. This can be particularly useful in cases where the estimated time for the return journey differs significantly from the estimated time for the outbound journey, which can occur when the traffic volume is completely different in each direction of the route. This information can be very useful to the user at the time of departure, allowing more sophisticated decisions to be made about local movement.

For example, the disclosed techniques can advantageously solve the problem of comprehensiveness of travel information provided by a navigation system. In addition to providing the outbound itinerary and associated travel information, the disclosed technique may provide travel information associated with the return itinerary.
