本開示は、一般的に、位置決定(location determination)に関するものであり、より具体的には、効率のよい方式で注目点(point of interest)にあるユーザデバイスの位置(location)を決定することに関するものである。 The present disclosure relates generally to location determination, and more specifically, determining the location of a user device at a point of interest in an efficient manner. It is about.
モバイルデバイスからの位置データは、多くの用途に使用され得る。多くの用途は、たとえば、友達を探す、ゲームをする、および道案内でユーザを補助するために位置データを使用する。位置データは、ユーザが会社、施設、ランドマーク、公園、またはユーザが注目するであろう他の場所などの注目点の付近に居るときにユーザのモバイルデバイス上にアラートを出すためにも使用され得る。たとえば、ユーザは、ユーザが特定の店またはレストランに近づくたびごとにアラートを受けることを、特にアラートが注目する店またはレストランのプロモーション割引を含む場合に、選択することができる。 Location data from mobile devices can be used for many applications. Many applications use location data, for example, to find friends, play games, and assist users with directions. Location data is also used to alert on the user's mobile device when the user is near a point of interest, such as a company, facility, landmark, park, or other location that the user may be interested in. obtain. For example, the user can choose to receive an alert each time the user approaches a particular store or restaurant, especially if the alert includes promotional discounts for the store or restaurant that the attention is focused on.
位置データは、さまざまな用途に使用され得るが、ユーザデバイスから位置データを効率よく取得することは、特に正確な位置情報が必要とされるときに、困難な場合がある。従来、近くの場所に対する位置情報を取得するために、ユーザデバイスは、数分ごとに利用可能なワイヤレス信号がないかその環境を定期的にスキャンして調べる。次いで、ユーザデバイスは、位置データを位置情報サービスに伝達し、その後、位置情報サービスは、たとえば、利用可能なワイヤレス信号を認識することによってユーザデバイスの位置を決定する。 Although location data can be used in a variety of applications, it can be difficult to efficiently obtain location data from a user device, especially when accurate location information is required. Traditionally, to obtain location information for nearby locations, user devices periodically scan their environment for available wireless signals every few minutes. The user device then communicates location data to the location information service, which then determines the location of the user device, for example by recognizing available wireless signals.
しかしながら、ワイヤレス信号に対するそのような定期的スキャンは、位置情報サービスへの位置データの通信と同様、ユーザデバイスの電池寿命を浪費する。利用可能なワイヤレス信号の定期的スキャンは、たとえば、ユーザデバイスが注目点の近くにないときに、電池寿命を特に浪費する。たとえば、ユーザはユーザデバイスを携えて、荒野のへんぴな地域をハイキングしていることもあり得る。そうであるのに、ユーザデバイスは、利用可能なワイヤレス信号がないか定期的にスキャンし続け得る(その地域にはワイヤレス信号が何もないとしても)。 However, such periodic scans for wireless signals waste user device battery life, as well as communication of location data to location-based services. Regular scanning of available wireless signals particularly waste battery life, for example when the user device is not near the point of interest. For example, a user may be hiking in a deserted desert area with a user device. Even so, the user device may continue to scan periodically for available wireless signals (even if there are no wireless signals in the area).
本明細書で説明されているいくつかの例示的な態様において、ユーザデバイスの位置を決定するためのコンピュータ実装方法が提供される。位置管理システムは、たとえば、初回に、ユーザコンピューティングデバイスが特定の注目点を囲むジオフェンス境界の外側にあると決定する。それゆえ、位置管理システムは、ユーザデバイスに利用可能なワイヤレス信号を決定するためにユーザデバイスに関連付けられているワイヤレス信号検出モジュールを呼び出さない。しかしながら、初回に続く二回目に、位置管理システムは、ユーザコンピューティングデバイスが特定の注目点を囲むジオフェンス境界の内側にあると決定する。 In some exemplary aspects described herein, a computer-implemented method for determining the position of a user device is provided. The location management system, for example, determines for the first time that the user computing device is outside the geofence boundary surrounding a particular point of interest. Therefore, the location management system does not call the wireless signal detection module associated with the user device to determine the wireless signal available to the user device. However, the second time following the first time, the location management system determines that the user computing device is inside the geofence boundary surrounding the particular point of interest.
二回目に、ユーザコンピューティングデバイスが特定の注目点を囲むジオフェンス境界の内側にあると決定したことに応答して、位置管理システムは、ユーザコンピューティングデバイスに関連付けられているワイヤレス信号検出モジュールを呼び出す指令をユーザコンピューティングデバイスに伝達する。次いで、位置管理システムは、ユーザコンピューティングデバイスに利用可能である特定のワイヤレス信号に対するワイヤレス信号データを受信する。受信されたワイヤレス信号データに基づき、位置管理システムは、ユーザコンピューティングデバイスに利用可能な特定のワイヤレス信号が特定の注目点に関連付けられていると決定する。そこで、位置管理システムは--特定のワイヤレス信号が特定の注目点に関連付けられていると決定したことに応答して--ユーザコンピューティングデバイスが特定の注目点のところにあると決定する。 Second, in response to determining that the user computing device is inside the geofence boundary surrounding a particular point of interest, the location management system activates a wireless signal detection module associated with the user computing device. A command to be called is transmitted to the user computing device. The location management system then receives wireless signal data for a particular wireless signal that is available to the user computing device. Based on the received wireless signal data, the location management system determines that a particular wireless signal available to the user computing device is associated with a particular point of interest. Thus, the location management system--in response to determining that a particular wireless signal is associated with a particular point of interest--determines that the user computing device is at a particular point of interest.

本明細書に開示されているように、位置管理システムは、1人または複数のユーザの興味を引きそうな注目点を識別する。次いで、位置管理システムは、特定の注目点を囲むジオフェンス境界を定義する。いくつかの例において、位置管理システムは、特定のユーザデバイスが注目点の近くにあると決定したことに応答して注目点を囲むジオフェンス境界を定義し得る。次いで、位置管理システムは、最初に、ユーザデバイスがジオフェンス境界のところまたはジオフェンス境界内にないと決定することができる。しかし、位置管理システムが、その後、ユーザデバイスがジオフェンス境界のところまたはジオフェンス境界内にあると決定したときに、位置管理システムは、ユーザデバイスに利用可能なワイヤレス信号を決定することをユーザデバイスに指令する。次いで、位置管理システムは、利用可能なワイヤレス信号についてユーザデバイスからワイヤレス信号データを受信する。受信されたワイヤレス信号データを識別された注目点において利用可能な知られているワイヤレス信号と照合することによって、位置管理システムは、ユーザデバイス(およびしたがってユーザ)が注目点のところにあると決定する。
Overview As disclosed herein, a location management system identifies points of interest that may be of interest to one or more users. The location management system then defines a geofence boundary that surrounds the particular point of interest. In some examples, the location management system may define a geofence boundary surrounding the point of interest in response to determining that a particular user device is near the point of interest. The location management system may then first determine that the user device is not at or within the geofence boundary. However, when the location management system subsequently determines that the user device is at or within the geofence boundary, the location management system will determine the wireless signal available to the user device. To The location management system then receives wireless signal data from the user device for available wireless signals. By matching the received wireless signal data with known wireless signals available at the identified point of interest, the location management system determines that the user device (and hence the user) is at the point of interest. .
より具体的に、位置管理システムは、複数の注目点に対する位置データを取得する。たとえば、位置管理システムは、1人または複数のユーザの興味を引きそうなさまざまな注目点について位置情報を決定し得る。いくつかの例では、位置マーケティングシステム(location marketing system)は、ユーザの興味を引きそうなさまざまな商店店頭(merchant storefront)などについて、位置管理システムに位置情報を提供し得る。位置情報は、住所、陸地測量部グリッド参照(Ordnance Survey Grid Reference)情報、ならびに/または注目点に対する緯度および経度座標などの、注目点の位置を確立する任意の情報であってよい。位置情報は、注目点のところで利用可能なワイヤレス信号の指示などの、位置に関するコンテンツも含み得る。たとえば、位置情報は、特定の注目点に関連付けられているワイヤレスアクセスポイント(WAP)に対する媒体アクセス制御(MAC)アドレスを含み得る。次いで、位置管理システムは、位置情報を--注目点に対する任意のワイヤレス信号情報とともに--各識別された注目点に対する注目点レコードに記憶することができる。 More specifically, the position management system acquires position data for a plurality of attention points. For example, the location management system may determine location information for various points of interest that may be of interest to one or more users. In some examples, the location marketing system may provide location information to the location management system, such as various merchant storefronts that may be of interest to the user. The location information may be any information that establishes the location of the point of interest, such as an address, land surveying grid reference information, and / or latitude and longitude coordinates for the point of interest. The location information may also include content related to the location, such as an indication of a wireless signal available at the point of interest. For example, the location information may include a medium access control (MAC) address for a wireless access point (WAP) that is associated with a particular point of interest. The location management system can then store the location information--with any wireless signal information for the point of interest--in the point of interest record for each identified point of interest.
さまざまな注目点に対する位置情報を取得した後、位置管理システムは、注目点に関連付けられているジオフェンス境界を決定する。たとえば、位置管理システムは、注目点に関連付けられているマーケティングシステムからジオフェンス境界データを受信し得る。それに加えて、または代替的に、位置管理システムは、1つまたは複数の注目点に対するジオフェンス境界を定義し得る。たとえば、位置管理システムが、特定のユーザデバイスが注目点の近くにあると決定した場合に、位置管理システムは、注目点を囲むジオフェンス境界を定義し得る。次いで、位置管理システムは、いくつかの例において、注目点の近くにあるユーザデバイスにジオフェンス境界データを伝達することができる。 After obtaining location information for various points of interest, the location management system determines a geofence boundary associated with the points of interest. For example, the location management system may receive geofence boundary data from a marketing system associated with the point of interest. In addition or alternatively, the location management system may define a geofence boundary for one or more points of interest. For example, if the location management system determines that a particular user device is near the point of interest, the location management system may define a geofence boundary that surrounds the point of interest. The location management system may then communicate geofence boundary data to user devices that are near the point of interest in some examples.
位置管理システムが、特定の注目点に対するジオフェンス境界を決定した後、位置管理システムは、ジオフェンス境界に相対的な1つまたは複数のユーザデバイスの位置を決定する。すなわち、位置管理システムは、特定のユーザデバイスがジオフェンス境界のところまたはジオフェンス境界の外側にあるかどうかを決定する。ユーザデバイスの位置に基づき、位置管理システムは、ユーザデバイスに利用可能であるローカルワイヤレス信号を決定することをユーザデバイスに指令する。ワイヤレス信号は、Wi-Fiワイヤレスローカルエリアネットワーク(WLAN)信号、Bluetooth（登録商標）信号、近距離無線通信(NFC)信号、または他の信号などの、ユーザデバイスに利用可能な任意の種類の無線信号を含み得る。 After the location management system determines a geofence boundary for a particular point of interest, the location management system determines the location of one or more user devices relative to the geofence boundary. That is, the location management system determines whether a particular user device is at or outside the geofence boundary. Based on the location of the user device, the location management system instructs the user device to determine a local wireless signal that is available to the user device. The wireless signal can be any type of radio available to the user device, such as a Wi-Fi wireless local area network (WLAN) signal, a Bluetooth® signal, a near field communication (NFC) signal, or other signal. A signal may be included.

That is, if the user device is not within the geofence boundary, the location management system cannot accurately determine the wireless signal available from the point of interest because, for example, the user device is not sufficiently close to the point of interest. And decide. Accordingly, the location management system does not instruct the user device to search for wireless signals, and thus reduces power consumption by the user device as described herein. A wireless signal can be any type of signal available to a user device, such as a Wi-Fi wireless local area network (WLAN) signal, a Bluetooth® signal, a near field communication (NFC) signal, or other signal. Can be included.

The user device then determines wireless signal data for the wireless signal available to the user device. The user device communicates wireless signal data to the location management system, which receives the wireless signal data.
ワイヤレス信号データを受信した後に、位置管理システムは、ワイヤレス信号データを読み取り、ユーザデバイスに利用可能なワイヤレス信号を決定する。次いで、位置管理システムは、利用可能なワイヤレス信号を注目点レコードに関連付けられているワイヤレス信号のレコードと比較する。位置管理システムが、レコード内の利用可能なワイヤレス信号に対する一致--一致する媒体アクセス制御(MAC)アドレス--を識別した場合、位置管理システムは、利用可能なワイヤレス信号がレコードにおいて対応する注目点に関連付けられていると決定する。したがって、ユーザデバイスが注目点に十分接近しており注目点に関連付けられているワイヤレス信号がユーザデバイスに利用可能であると決定できるので、位置管理システムは、ユーザデバイスが特定の注目点のところにあると決定する。 After receiving the wireless signal data, the location management system reads the wireless signal data and determines a wireless signal available to the user device. The location management system then compares the available wireless signal to the wireless signal record associated with the point of interest record. If the location management system identifies a match to an available wireless signal in the record--a matching medium access control (MAC) address--the location management system will note that the available wireless signal corresponds to the record. Determine that it is associated with. Thus, the location management system can determine that the user device is at a particular point of interest because the user device is sufficiently close to the point of interest and the wireless signal associated with the point of interest can be determined to be available to the user device. ユーザデバイスがジオフェンス境界のところまたはジオフェンス境界の内側にあると決定した後に利用可能なワイヤレス信号を決定することをユーザデバイスに指令することによって、本明細書で説明されている方法およびシステムは、デバイスの電池電力消費量を低減する。すなわち、本明細書で説明されている方法およびシステムは、ユーザデバイスがデバイスに利用可能なワイヤレス信号を主に決定するときからユーザデバイスが注目点に実際に近いときまでタイミングをシフトする。したがって、本明細書で説明されている方法およびシステムを採用することによって、ユーザデバイスは、ユーザデバイスが注目点についてジオフェンス境界の外側にある(したがって注目点の近くにない)ときに利用可能なワイヤレス信号を決定することを試みることでエネルギーを無駄にする(および不必要に電池電力を消費する)ことをしない。 By instructing a user device to determine an available wireless signal after determining that the user device is at or inside the geofence boundary, the methods and systems described herein are , Reduce the battery power consumption of the device. That is, the methods and systems described herein shift timing from when the user device primarily determines the wireless signal available to the device until when the user device is actually close to the point of interest. Thus, by employing the methods and systems described herein, a user device can be utilized when the user device is outside the geofence boundary for the point of interest (and thus not near the point of interest) Don't waste energy (and unnecessarily consume battery power) by trying to determine the wireless signal.
