Hintergrundbackground
Persönliche Assistenzmodule sind Software-Prozesse, die auf Computergeräten, wie etwa Smartphones, Tablets, Fahrzeugnavigationssysteme und Smartwatches installiert sind, um benutzerfreundliche Schnittstellen für Benutzer bereitzustellen, um natürliche Sprachbefehle zu erteilen. Benutzer erteilen oft natürliche Sprachbefehle unter Verwendung ihrer Stimme, aber persönliche Assistenzmodule können auch auf andere Modalitäten, wie etwa auf verschiedene Formen textlicher und/oder bildlicher Eingaben, antworten. Inhalte, die zwischen einem Benutzer und einem persönlichen Assistenzmodul ausgetauscht werden, beschränken sich in der Regel auf Texte (z. B. eingetippt oder von einer Spracheingabe generiert) oder sonstige Daten, wie etwa Bilder, und werden in der Regel nicht für einen späteren Gebrauch gespeichert. Folglich muss ein Benutzer, der externe Informationen sucht, die für die die Konversation maßgebend sind, auf eine andere Anwendung, wie etwa einem Webbrowser umschalten, um die erwünschten Informationen einzuholen. Das Umschalten von Anwendungen, insbesondere unter Verwendung von mobilen Geräten, die oft eine einzige Anwendung auf einmal anzeigen, können ablenkend und lästig sein, insbesondere für diejenigen Benutzer, die über eine verringerte Geschicklichkeit verfügen und können den Fluss der Konversation stören.Personal Assistance modules are software processes that are installed on computing devices such as smart phones, tablets, car navigation systems, and smart watches to provide user-friendly interfaces for users to issue natural voice commands. Users often issue natural voice commands using their voice, but personal assistance modules can also respond to other modalities, such as various forms of textual and / or pictorial input. Content exchanged between a user and a personal assistant module is typically limited to texts (eg, typed or generated by a voice input) or other data, such as images, and typically will not be for later use saved. Consequently, a user searching for external information for which the conversation is authorizing must switch to another application, such as a web browser, to obtain the desired information. Switching applications, especially using mobile devices that often display a single application at a time, can be distracting and annoying, especially for those users who have reduced skill and can disrupt the flow of conversation.
KurzdarstellungSummary
Die vorliegende Offenbarung ist im Allgemeinen an Verfahren, Vorrichtungen, computerlesbare Medien (flüchtige oder nicht-flüchtige) gerichtet, um „auswählbare Anwendungslinks” – grafische Elemente, die in einen Nachrichtenaustausch-Thread zwischen einem Benutzer und einem persönlichen Assistenzmodul einbezogen sind, um dem Benutzer eine Schnittstelle, die einer anderen Anwendung zugewiesen ist, zu offenbaren – direkt in eine Abschrift des Nachrichtenaustausch-Threads einzubeziehen, der auf einem von einem Benutzer betriebenen Client-Computergerät (oder einfach „Clientgerät”) wiedergegeben wird. In verschiedenen Implementierungen können die auswählbaren Anwendungslinks, die in eine Abschrift einbezogen werden, von verschiedenen Ressourcen des Clientgeräts, welches die Abschrift wiedergibt, diktiert werden. Zum Beispiel können ein oder mehrere „Kandidaten-Anwendungen”, die auf irgendeine Weise hinsichtlich des Inhalts des Nachrichtenaustausch-Threads relevant sind – und die auch auf dem Client-Computergerät installiert werden, das vom Benutzer betrieben wird, der mit dem persönlichen Assistenzmodul interagiert identifiziert werden. In einigen Implementierungen können nur ausführbare Anwendungslinks zu installierten Anwendungen präsentiert werden. Falls in anderen Implementierungen keine anwendbaren Anwendungen installiert werden, kann ein Link zu einer Installationsanwendung (z. B. in einem App-Store oder auf einer Webseite) oder zu einer Webseite, die über eine entsprechende Funktionalität verfügt, stattdessen einbezogen werden. Die Miteinbeziehung auswählbarer Anwendungslinks in den Nachrichtenaustausch-Thread kann die Anzahl der Benutzereingaben verringern, die ein Benutzer an einem Gerät tätigen muss, um auf die Inhalte, die über die Links zur Verfügung stehen, zuzugreifen. Dies kann von besonderem Vorteil für Anwender sein, die über eine verringerte Geschicklichkeit verfügen, oder für Benutzer, die auf sonstige Weise Schwierigkeiten damit haben, Benutzereingaben in ein Gerät zu tätigen. Die Miteinbeziehung der auswählbaren Vorschläge in den Nachrichtenaustausch-Thread kann wenigstens für diese Benutzer die Zuverlässigkeit und Geschwindigkeit der Verwendung des Geräts erhöhen, z. B. indem der Benutzer erst gar nicht in die Situation gebracht wird, aufgrund seiner verringerten Geschicklichkeit Fehler zu machen.The present disclosure is generally directed to methods, apparatuses, computer-readable media (volatile or non-volatile), "user-selectable application links" - graphical elements included in a message exchange thread between a user and a personal assistance module to disclose an interface assigned to another application - to include directly in a transcription of the messaging thread rendered on a user-operated client computing device (or simply "client device"). In various implementations, the selectable application links included in a transcript may be dictated by various resources of the client device that reproduces the transcript. For example, one or more "candidate applications" that are somehow relevant to the content of the messaging thread - and that are also installed on the client computing device operated by the user interacting with the personal assistance module become. In some implementations, only executable application links to installed applications can be presented. In other implementations, if no applicable applications are installed, a link to an installation application (for example, in an app store or on a web page) or to a web page that has such functionality may be included instead. The inclusion of selectable application links in the messaging thread can reduce the number of user inputs a user must make to a device in order to access the content available through the links. This may be particularly beneficial for users who have reduced skill or for users who otherwise have difficulty placing user input in a device. The inclusion of the selectable suggestions in the messaging thread, at least for these users, can increase the reliability and speed of use of the device, e.g. B. by the user is not even put into the situation, due to his reduced skill to make mistakes.
Die Miteinbeziehung der auswählbaren Anwendungslinks in eine Abschrift kann aufgrund verschiedener Signale ausgelöst werden. In einigen Implementierungen können die Signale von Inhalten ausgehen, die von einem oder mehreren vom Benutzer betriebenen Client-Computergeräten in den Nachrichtenaustausch-Thread einbezogen werden. Zum Beispiel kann ein Benutzer eine Instanz eines persönlichen Assistenzmoduls, der auf einer Smartwatch betrieben wird, fragen: „Was steht für heute Abend in meinen Terminkalender?”. Das persönliche Assistenzmodul kann nach dem Kalender des Benutzers und/oder sonstigen Datenquellen (die sich entweder online oder lokal auf dem Clientgerät befinden) suchen und antworten, dass der Benutzer für heute Abend in seinem Terminkalender nichts stehen hat. Später kann der Benutzer mit einer zweiten Instanz des persönlichen Assistenzmoduls interagieren, die auf einem unterschiedlichen Clientgerät betrieben wird, wie etwa ein Smartphone oder ein Tablet. Der Benutzer kann etwas sagen wie „Mach eine Reservierung im XYZ Café”. Der Benutzer war sich nicht darüber im Klaren, wann die Reservierung gemacht werden sollte, weswegen die zweite Instanz des persönlichen Assistenzmoduls sich an den persistenten Nachrichtenaustausch-Thread wenden kann, um – auf Basis der vorhergehenden Frage an die erste Instanz des persönlichen Assistenzmoduls, die auf der Smartwatch betrieben wird und/oder die Antwort darauf – daraus zu folgern, dass der Benutzer diesen Abend gemeint hat. Durch die Ziehung derartiger Rückschlüsse ist es möglich, die Anzahl der Eingaben, die von einem Benutzer erfordert werden, ferner zu verringern.The involvement of the selectable application links in a transcript can be triggered by various signals. In some implementations, the signals may emanate from content that is included in the message interchange thread by one or more user-operated client computing devices. For example, a user may ask an instance of a personal assistance module that operates on a smartwatch, "what's in my diary tonight?". The personal assistant module can search for the user's calendar and / or other data sources (located either online or locally on the client device) and respond that the user has nothing in their schedule for tonight. Later, the user may interact with a second instance of the personal assistance module that operates on a different client device, such as a smartphone or a tablet. The user can say something like "Make a reservation at XYZ Café". The user was not aware of when the reservation should be made, which is why the second instance of the personal assistance module can contact the persistent messaging thread, on the basis of the previous question to the first instance of the personal assistance module the smartwatch is operated and / or the answer to it - to deduce that the user meant that evening. By drawing such conclusions, it is possible to further reduce the number of inputs required by a user.
