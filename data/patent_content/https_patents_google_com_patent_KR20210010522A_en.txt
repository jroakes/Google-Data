Abstract
본 명세서에서 논의되는 구현들은 휴대용 전자 디바이스의 그래픽 카메라 인터페이스를 통해 하나 이상의 사물 인터넷(IoT) 디바이스를 제어하기 위한 시스템, 방법 및 장치에 관한 것이다. 그래픽 카메라 인터페이스는 사용자가 휴대용 전자 디바이스를 통해 카메라 애플리케이션에 액세스하고 IoT 디바이스에서 휴대용 전자 디바이스의 카메라를 지향하는 것에 응답하여 휴대용 전자 디바이스에 제공될 수 있다. 카메라 출력에서 생성된 하나 이상의 이미지는 처리되어 사용자가 제어하려고 하는 IoT 디바이스를 식별할 수 있다. 사용자가 자신의 카메라를 IoT 디바이스로 겨냥한 상황과 관련된 정보는 사용자 및/또는 카메라에 의해 캡처된 이미지와 관련될 수 있는 다수의 상이한 IoT 디바이스로부터 IoT 디바이스의 선택 범위를 좁히는데 도움이 될 수 있다.Implementations discussed herein relate to a system, method, and apparatus for controlling one or more Internet of Things (IoT) devices through a graphic camera interface of a portable electronic device. The graphical camera interface may be provided to the portable electronic device in response to a user accessing the camera application through the portable electronic device and pointing the camera of the portable electronic device at the IoT device. One or more images generated from the camera output can be processed to identify the IoT device that the user wants to control. Information related to the context of a user aiming his or her camera at the IoT device may help narrow the selection of IoT devices from a number of different IoT devices that may be associated with images captured by the user and/or the camera.

개개인은 가정 내에서 다양한 애플리케이션으로 제어될 수 있는 다수의 연결 (된) 디바이스를 가질 수 있다. 이러한 연결 디바이스는 예를 들어 스마트 전구, 스마트 플러그, 스마트 잠금 장치, 스마트 온도 조절기, 스마트 보안 시스템, 스마트 기기 등을 포함할 수 있다. 일부 경우, 특정 연결 디바이스를 제어하기 위해 특정 애플리케이션이 제공될 수 있다. 그러나, 특정 애플리케이션은 클라이언트 디바이스에 설치되어야 한다. 더욱이, 특정 연결 디바이스를 제어하기 위해 사용자는 클라이언트 디바이스에 설치된 수 많은 애플리케이션 중에서 특정 애플리케이션을 식별하고, 클라이언트 디바이스에서 실행될 애플리케이션을 선택한 다음 특정 애플리케이션 내에서 탐색하여 특정 연결 디바이스를 식별하고 제어해야 한다. 특정 연결 디바이스를 제어하기 위해 상대적으로 많은 양의 사용자 입력을 요구하는 것 외에도, 특정 애플리케이션을 식별하는 사용자에 의해, 특정 애플리케이션을 실행함으로써 및/또는 사용자가 특정 디바이스를 식별하고 제어하기 위해 특정 애플리케이션 내에서 탐색할 때 특정 애플리케이션의 컨텐츠를 렌더링함으로써 중요한 클라이언트 디바이스 자원(예를 들어, 프로세서 및/또는 배터리)가 소비될 수 있다. 추가적으로, 이러한 특정 애플리케이션은 클라이언트 디바이스의 메모리를 소비할 수 있다. 이러한 메모리 소비는 클라이언트 디바이스가 제한된 메모리 자원을 갖는 많은 상황 및/또는 클라이언트 디바이스가 개별 연결 디바이스의 제어를 위한 개별 특정 애플리케이션을 각각 갖는 많은 이질적인 연결 디바이스를 제어하는데 사용되는 상황에서 악화될 수 있다.Individuals can have multiple connected (connected) devices within the home that can be controlled by various applications. Such connection devices may include, for example, smart light bulbs, smart plugs, smart locks, smart thermostats, smart security systems, smart devices, and the like. In some cases, specific applications may be provided to control specific connectivity devices. However, certain applications must be installed on the client device. Moreover, in order to control a specific connection device, a user must identify a specific application from among the numerous applications installed on the client device, select an application to be run on the client device, and then search within the specific application to identify and control the specific connection device. In addition to requiring a relatively large amount of user input to control a specific connected device, by a user identifying a specific application, by running a specific application, and/or within a specific application for the user to identify and control a specific device. Important client device resources (eg, processor and/or battery) can be consumed by rendering the content of a particular application when browsing in. Additionally, these specific applications can consume the memory of the client device. This memory consumption can be exacerbated in many situations in which the client device has limited memory resources and/or in situations in which the client device is used to control many disparate connected devices, each having a separate specific application for control of individual connected devices.

Specifically, the user may activate a graphic camera interface to initialize an interface for controlling a specific connection device and point the camera of the portable client device to the specific connection device. The specific connection device may be controlled via the interface based at least in part on the processing of image data from the camera and the determination that the image data captures the specific connection device and/or the installation environment of the specific connection device. A particular connection device may be, for example, a smart light bulb, a smart television and/or other smart device, and a user may control and/or configure the device using the camera of the portable client device and one or more interfaces of the portable client device. . In this way, the user does not necessarily have to rely on a variety of individual applications to control the various devices in the home, but a camera can be used to indicate to a portable client device that he or she wishes to control a particular controllable (one) device. have. This allows the user to free up space and processing bandwidth on a portable client device since it is not necessary to install, open or switch all applications provided by manufacturers of various controllable devices to control the devices. For example, in a scenario where a user wants to control two or more controllable devices, the user does not necessarily need to rely on two or more separate applications installed on their client devices to control the controllable devices. Instead, the user can be provided with the ability to control two or more controllable devices from the client device through a camera-related user interface, whereby the client device needs to switch between different applications related to each controllable device. none. It will be appreciated that this can reduce the processing burden and reduce power usage on the client device. Moreover, various implementations disclosed herein reduce the amount of user input required to control the controllable device. For example, some of these implementations include a single input (e.g., tap, long tap, tap) in a graphical camera interface that captures a graphical element overlaid on the graphical camera interface or the installation environment of the user-controllable device and/or devices. And drag), and allows controllable devices to be controlled in response to a single input. This allows the automation assistant to distinguish between other controllable devices that may be candidates to be controlled according to the voice utterance. For example, a user working in a scientific laboratory may have multiple lamps that he can use to provide specific lighting conditions in the laboratory, which provide EM radiation of specific wavelengths to test samples. Voice utterances such as "Assistant, turn on the lamp" may not provide a lot of context (context) to help identify the lamp the user is referring to. However, if the user points the camera of the portable client device to a specific lamp while saying "Assistant, turn on the lamp", the automation assistant may determine that the user wants to turn on the specific lamp based on the context data and voice utterance. In another example, since a user may have multiple televisions at home, voice utterances such as “Turn on Assistant Television” may not provide much context to help identify the television the user is referring to. However, if the user points the camera of his portable client device to a specific television while saying "Assistant, turn on the television", the automation assistant can determine that the user wants to turn on the specific TV based on the context data and voice utterance. have. The context data can be characterized by the location of the portable client device, the orientation of the portable client device as indicated by one or more sensors of the portable client device, the time the user provided the voice utterance, and any other context in which the user provided the command. 일부 구현에서, 본 명세서에서 논의된 이미지 처리 기술과 함께 컨텍스트 데이터는 사용자가 자신의 카메라를 제어 가능 디바이스로 지향함과 동시에 자신의 터치 스크린 디스플레이를 통해 제어하고자 하는 제어 가능 디바이스를 결정하는데 사용될 수 있다. 이러한 컨텍스트 데이터는 사용자와 관련된 위치(예를 들어, 집) 내에서 방들의 레이아웃을 특징짓는 방 인접성(adjacency) 데이터를 포함할 수 있지만 이에 제한되지는 않는다. 방 인접성 데이터는 사용자가 자신의 집(또는 기타 위치)내에서 활동을 수행하고, 방 데이터(예를 들어, "거실", "침실"등과 같은 방의 라벨 또는 "실험실 1", "실험실 2"등)를 제공하고 및/또는 집(또는 다른 위치)내에 있는 특정 방에 관한 추론을 제공하도록 행동할 때 시간이 지남에 따라 생성될 수 있다. 일부 구현에서, 자동화 어시스턴트가 특정 디바이스가 그 특정 디바이스에 이용 가능한 신호 강도에 기초하는 위치를 결정할 수 있도록 네트워크 라우터의 위치가 룸 인접 데이터와 관련하여 저장될 수 있다. 예를 들어, 네트워크 라우터와 동일한 층(예를 들어, 1 층)에 있는 디바이스는 그 디바이스가 다른 층(예를 들어, 지하 층)에 있을 때보다 더 많은 네트워크 연결성을 가질 수 있다. 또한 이에 한정되지 않지만 글로벌 위치 시스템(GPS) 데이터, 자이로스코프 센서 데이터, 이미지 데이터, (예를 들어, 디바이스의 온도를 하나 이상의 방의 알려진 온도와 비교하기 위한) 온도 데이터, 궤적 데이터, 및/또는 특정 방내의 휴대용 클라이언트 디바이스의 위치를 결정하는데 사용될 수 있는 임의의 다른 위치 데이터와 같은 다른 컨텍스트 데이터가, 사용자가 카메라를 지향하고 있는 제어 가능 디바이스를 결정하는데 사용될 수 있다. 일부 구현에서, 컨텍스트 데이터는 특정 제어 가능 디바이스가 다수의 후보 디바이스의 타겟 디바이스임을 결정하는데 사용될 수 있으며, 이에 따라 메트릭들이 계산될 때 나타날 수 있는 "동점(tie)"을 깨뜨릴 수 있다. 예를 들어, 두 개 이상의 후보 디바이스에 동일한 메트릭 또는 모두 허용 범위 내에 있는 메트릭들이 할당될 수 있다. 후보 디바이스들을 단일 후보 디바이스로 더 제한하기 위해, 컨텍스트 데이터가 여러 후보 디바이스를 단일 후보 디바이스로 줄이는데 사용될 수 있다.In some implementations, context data, along with the image processing techniques discussed herein, can be used to determine the controllable device that the user wishes to control via his touch screen display while pointing his or her camera to the controllable device. . Such context data may include, but is not limited to, room adjacency data that characterizes the layout of rooms within a location (eg, home) associated with the user. Room adjacency data means that the user performs an activity within his or her home (or other location), and the room data (eg, the label of a room, such as "living room", "bedroom", etc., or "lab 1", "lab 2", etc. ) And/or acting to provide inferences about a particular room within a house (or other location) over time.
